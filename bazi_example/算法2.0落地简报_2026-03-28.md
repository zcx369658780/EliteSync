# 玄学算法 2.0 落地简报（2026-03-28）

## 已完成
- A1 统一模块输出契约：`score/confidence/reason_short/reason_detail/risk_detail/evidence_tags/evidence/degraded/degrade_reason` 已落地并有契约测试。
- A2 八字主逻辑重构：八字评分已改为“特征层优先（结构/关系/时间）”，字符串相似仅兜底。
- A3 属相来源统一：已改为“八字年柱优先，生日兜底”并接入回填命令。
- A4 星盘计算抽象层：`AstroEngineAdapter` 已落地，支持 `default/standard` 切换。
- A5 边界样本：已新增属相边界向量与单测。
- A6 开源复核台账：已生成并标记 `approved/conditional/unverified`。
- B1 八字特征工程：`BaziFeatureExtractor` 已接入主评分。
- B2 合盘特征拆分：`pair_chart` 已输出结构化 `features`，解释可追溯。
- B3 MBTI 题库版本化：后端已支持 `lite3_v1/lite5_v1`，兼容旧版。
- B4 风险标签标准化：新增 evidence tag 字典，API 返回可读标签。
- C1 展示分/排序分分离：`display_score` 与 `rank_score` 已区分并返回。
- C2 离线评估管线雏形：新增 `app:eval:astro-v2` 命令。
- C3 发布门禁算法回归：release gate 已加入 Astro 回归测试。

## 当前未完全落地（需后续推进）
- 八字高级特征：
  - 十神、格局、用神/忌神、夫妻宫细化（当前为可运行基础版）。
  - 真太阳时精确换算链路（当前未引入标准天文时间校正）。
- 西占深度模块：
  - Synastry/Composite 的标准化独立引擎（当前为过程层近似特征版）。
  - 行星相位/宫位完整标准计算（需接入 Swiss Ephemeris 或等价引擎）。
- 人格模块：
  - Big Five 连续维度与 MBTI-like 映射（当前仍以 MBTI 维度为主）。
- 业务校准：
  - `rank_score` 仍未接入真实行为数据训练，仅完成结构分离。
  - 离线评估命令依赖数据库驱动和样本数据（当前环境缺少 PDO MySQL 时无法产出报告）。

## 建议下一阶段优先级
1. 接入标准占星计算引擎（先本命盘，再 synastry/composite）。
2. 八字特征升级到十神/夫妻宫/岁运联动。
3. 增加算法回归基准集（八字/星盘边界样本统一放入 `test_vectors`）。
4. 用小规模真实行为数据校准 `rank_score`。
