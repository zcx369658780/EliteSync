# 算法 2.3 落地简报（2026-03-30）

## 1. 当前进度

2.3 版本已从“口径整理”推进到“解释可信、前后端一致、可回归验收”的工程阶段。当前整体完成度约 80% 左右。

### 已完成
- 2.3-0 基线冻结：已固定 2.2 线上口径，建立对照基线。
- 显示保护层：已接入 `display_guard`、`confidence_tier`、`engine_source`、`engine_mode`、`data_quality` 等元字段。
- 解释模板化：`explanation_blocks` 已进入匹配输出链路，支持模板化解释。
- 置信度与标签映射：用户端不再直接暴露 raw evidence tags，改为可读展示标签。
- 西占路线策略化：已形成 ADR 与过渡策略，避免生产口径与实现状态不一致。
- 低样本调权守门：已建立权重变更约束与守门脚本。
- 灰度/回滚/验收文档：已建立发布、回滚、灰度与回归文档。
- Flutter 端收口：匹配详情页已补齐摘要兜底，问卷链路改为选中即跳转，性格测试文案已替换 MBTI。

### 当前仍需收尾
- 匹配详情页的解释展示还可进一步优化，但已不再出现纯灰空页。
- 西占 canonical 仍处于受控过渡态，后续需按 ADR 再决定是否启用更高精度引擎。
- 仓库中仍有历史文档与兼容字段，需要后续继续精简与收口。

## 2. 2.3 的主要算法思路

### 2.1 总体原则
- 不再继续扩写旧近似算法。
- 后端成为唯一真源，客户端只做展示、缓存与降级提示。
- 任何低置信或退化路径都必须明确降级，不允许误导性高口径展示。
- 所有解释都要可回归、可比对、可灰度。

### 2.2 八字与玄学分层
- 八字仍是核心权重之一，但强调结构互补、节律同步、风险点与建议，而不是简单打分。
- 属相继续作为低复杂度辅助模块，保持低权重、低措辞强度。
- 星座作为轻量展示层，不能与完整星盘混用。
- 星盘 / 合盘分层处理，避免把不同层级的关系解释混为一谈。

### 2.3 性格模块
- 原 `MBTI` 已在前台统一改名为 `性格测试 / 性格特征`。
- 性格模块只承担沟通风格参考，不作为最终排序主因。
- 结果强调“辅助判断”，不使用过强结论口径。

### 2.4 解释层
- 所有模块统一输出结论、过程、风险、建议四层结构。
- 解释模板已模板化，支持回归样例与快照 diff。
- 模块说明、风险标签、证据标签都已可读化，避免 raw tag 直接暴露。

### 2.5 置信度与展示门禁
- `engine_mode = canonical / legacy / fallback` 直接影响前端措辞。
- `precision_level` 和 `confidence` 决定 badge 与文案强度。
- `date_only`、`partial_unknown`、`legacy_estimate` 等路径必须降级展示。

## 3. 版本级风险
- 西占高精引擎尚未完全收口，当前仍按过渡策略控制风险。
- 样本不足时不允许激进调权。
- 所有新增依赖必须先过许可证门禁。

## 4. 下一步建议
1. 继续收紧匹配详情页的解释展示，确保所有结果都有稳定摘要与可展开内容。
2. 按 ADR 继续推进西占 canonical 的最终路线决策。
3. 继续清理历史文档和兼容残留，减少仓库噪音。
4. 后续如进入 2.3-1 / 2.3-2 的收尾，可优先做解释 diff 与回归样例的自动化。
