# EliteSync 当前版本全算法报告（八字与匹配）

- 文档日期：2026-03-29
- 适用版本：`0.02.02`（当前主线）
- 目的：给顾问 Agent 做二次算法设计输入，说明“当前已落地实现”而非目标方案。

## 1. 总览

当前系统采用两层结构：
1. **画像生成层**：主要在客户端生成八字/星盘结构化数据，再写入后端。
2. **匹配评估层**：后端统一读取用户画像与问卷数据，输出综合分、模块分、解释、风险与证据标签。

其中：
- 结果层偏重：八字、属相。
- 过程层偏重：星座、星盘、合盘、MBTI、人格。
- 当前工程已支持“模块化解释 + 证据标签 + 降级原因”。

## 2. 画像生成算法（当前实现）

## 2.1 八字/大运/流年/五行（客户端）

- 主要实现文件：
  - `apps/android/app/src/main/java/com/elitesync/astro/AstroCalculator.kt`
  - `apps/android/app/src/main/java/com/elitesync/astro/AstroModels.kt`
- 核心依赖：`com.nlf.calendar`（Lunar/Solar 八字库）

### 生成流程
1. 输入：生日、出生时刻、性别、经纬度（可缺省）。
2. 太阳/月亮/上升：
   - 太阳黄经 -> 太阳星座。
   - 月亮黄经近似算法 -> 月亮星座。
   - 当有经纬度时计算上升点经度 -> 上升星座。
3. 真太阳时修正：
   - 按中国 120E 基准，`(birthLng - 120) * 4min` 修正出生本地时。
4. 八字：
   - `Solar.fromYmdHms(...).lunar.eightChar` 得到年/月/日/时柱文本。
5. 大运：
   - `eight.getYun(gender).getDaYun(8)`，过滤 `index > 0`。
6. 流年：
   - 在命中大运中取当前阶段，并取 `getLiuNian(10)`。
7. 五行：
   - 四柱五行计数 + 藏干（权重 0.5）累计，最终映射到木火土金水。

### 当前特征
- 有经纬度时可产出真太阳时与更稳的八字时柱。
- 无经纬度时仍可生成（但标记 notes，精度降级）。

3. 结果写入 `users.public_mbti`（匹配引擎唯一 MBTI 来源）。

## 3. 匹配算法（当前实现）

## 3.1 引擎入口与数据流

- 主要实现文件：
  - `services/backend-laravel/app/Services/MatchingEngineService.php`

### 数据流
1. 读取候选用户画像、问卷、行为信号。
2. 先过硬过滤（同城、异性等）。
3. 计算 pair detail（人格、MBTI、玄学模块）。
4. 叠加软惩罚/加成、分数守卫、曝光公平性。
5. 输出配对、分数、模块解释、证据标签。

---

## 3.2 硬过滤（Hard Filters）

配置文件：`services/backend-laravel/config/matching.php`

当前开启：
- `same_city_only = true`：仅同城。
- `opposite_gender_only = true`：仅男女。
- `reject_casual_vs_marriage = true`：婚姻目标硬冲突直接剔除。
- `exclude_recent_pair_days = 14`：近14天配对去重。

---

## 3.3 核心总分结构

配置文件：`services/backend-laravel/config/matching.php`

当前核心权重：
- `personality`: 0.50
- `mbti`: 0.15
- `astro`: 0.35

公式（简化）：
- `coreTotal = personality*0.50 + mbti*0.15 + astroTotal*0.35`
- `base = coreTotal / 100`
- `final = base * product(soft_penalty_factors)`
- `fairAdjusted = final * fairnessMultiplier(exposure_count)`

并有 guard：防止某单模块极端主导。

## 3.5 MBTI 匹配模块（MbtiCompatibilityService）

- 文件：`services/backend-laravel/app/Services/MbtiCompatibilityService.php`
- 配置：`services/backend-laravel/config/match_rules.php` -> `mbti`

### 结构
1. 维度评分（EI/SN/TF/JP）
2. 认知栈评分（dominant/auxiliary 关系）
3. 生活节律修正（JP、TF 风险）

### 总分（代码中的组合）
- `raw = base + dimension*0.55 + stack*0.35 + lifestyle*0.10`
- 缺失 MBTI 时降级到中性分 55。

输出含：
- 维度证据、栈证据、风险标签（如 strong-strong conflict）。

---

## 3.6 玄学总模块（AstroCompatibilityService）

- 文件：`services/backend-laravel/app/Services/AstroCompatibilityService.php`
- 配置：`services/backend-laravel/config/match_rules.php`

当前玄学子项权重：
- 八字 `bazi = 0.45`
- 属相 `zodiac = 0.25`
- 星座 `constellation = 0.08`
- 星盘 `natal_chart = 0.07`
- 合盘 `pair_chart = 0.15`

`astroTotal = Σ(模块分 * 权重)`

### 3.6.1 八字模块（Bazi）
- 特征抽取器：`BaziFeatureExtractor.php`
- 三层特征：
  - 结构（五行互补/平衡）
  - 关系（地支六合/三合/冲刑害）
  - 时序（流年/大运年份重叠）
- 特征层权重：
  - structure 0.50 / relationship 0.30 / timing 0.20
- 降级链路：
  - 若缺五行 -> 用八字文本相似度估算；
  - 再缺 -> 中性保守分 55。

### 3.6.2 属相模块（Zodiac）
- 规则表：六合、三合、冲、刑、害。
- 分数映射：
  - 六合 100、三合 85、同属相 60、普通 55、冲 20、刑 30、害 35。
- 输出关系类型、短因子、细节说明、风险说明。

### 3.6.3 星座模块（Constellation）
- 元素映射（火土风水）
- 规则：同元素 / 互补元素 / 张力元素
- 降级：信息不足时中性估算并降低 confidence。

### 3.6.4 星盘模块（Natal Chart）
- 通过 western engine 计算 process 指标：
  - `direction_sync`
  - `emotion_sync`
  - `expression_sync`
- 缺失上升/月亮时会部分降级。

### 3.6.5 男女合盘模块（Pair Chart）
- 特征抽取器：`PairChartFeatureExtractor.php`
- 关键指标：
  - `sun_moon_harmony`（0.35）
  - `asc_interaction`（0.20）
  - `emotion_rhythm`（0.25）
  - `long_term_stability`（0.20）
- 输出 high/medium/low 档位解释与风险。

## 3.7 软惩罚、加成、公平性

配置：`services/backend-laravel/config/matching.php`

### 软项
- 目标部分不一致惩罚
- 生活方式冲突惩罚
- 沟通风格冲突惩罚
- 兴趣重叠过低惩罚

### 信号修正
- 同城加成（当前 1.12）
- 年龄差分桶修正
- MBTI per-letter 旧修正当前关闭（避免与核心 MBTI 模块重复计分）

### 公平性
- 按近期曝光量分桶乘以 multiplier，抑制头部过曝，提升长尾机会。

## 6. 当前局限（供顾问 Agent 继续优化）

1. 八字仍是工程化近似，不是完整命理推演（尤其神煞/格局深层规则未全量接入）。
2. 星盘与合盘使用简化指标，不等价于专业占星全盘相位系统。
3. MBTI 题库是轻量版，置信度上限受题量限制。
4. 目前真实用户量不足，匹配参数更多依赖离线与合成样本调优。
