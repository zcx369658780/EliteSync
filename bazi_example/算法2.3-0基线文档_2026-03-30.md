# 算法 2.3-0 基线文档（2026-03-30）

## 1. 目标与边界
- 目标：启动 2.3 版本算法迭代，建立可追踪基线，后续所有改动以此为对照。
- 边界：本基线记录当前已上线的 `2.2` 能力现状，不在本文做新算法实现。

## 3. 2.3-0 阶段交付物（计划内）
- 许可证门禁与依赖审计持续更新（根目录许可证总表 + bazi_example 细化文档）
- Bazi canonical 继续稳态化（边界样本、置信度与解释一致性）
- Western canonical 进入可灰度路线（先对拍、后小流量）
- 匹配解释层与前端展示契约收口（字段稳定、降级提示一致）

## 4. 风险与约束
- AGPL/商业授权链（Swiss 相关）未完成合规前，不得作为闭源生产默认路径。
- 客户端不可再作为 canonical 计算源，只承担输入/展示/缓存。
- 任何“生肖=公历年”的回退逻辑必须标记为低置信 legacy。

## 5. 验收基线（后续每轮必须对照）
- 接口契约不破坏：匹配详情、画像详情、解释字段结构稳定。
- 分数可复现：同输入在同配置下结果稳定。
- 置信度可解释：缺失出生时刻/经纬度时有明确降级说明。
- 许可证可审计：新增依赖必须先入许可证总表再上线。
