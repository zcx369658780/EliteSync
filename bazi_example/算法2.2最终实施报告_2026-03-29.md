# 算法2.2最终实施报告（2026-03-29）

## 1. 目标与范围
本轮目标是把玄学算法从“客户端近似 + 多入口混算”收敛到“服务端 canonical 真源 + 可灰度 + 可对拍 + 可调权 + 可回滚”的工程形态。

## 2. 已落地能力

### 2.1 canonical 服务化
- 八字 canonical：服务端 `BaziEngine` 接管，客户端仅输入/展示。
- 西占 natal canonical：服务端 `WesternNatalEngine` 接管，客户端近似逻辑降级为 legacy 展示路径。
- 关键配置：
  - `services/backend-laravel/config/astro_canonical.php`
  - `services/backend-laravel/config/astro_rollout.php`
  - `services/backend-laravel/config/confidence_policy.php`

### 2.2 生肖口径修正
- 生肖由年柱地支派生，公历年回退仅作为低置信 legacy 兜底。

### 2.3 匹配解释 2.2
- 匹配模块输出统一结构：分数、置信度、证据、风险与建议。
- Flutter 匹配详情页支持模块卡片与标签化说明。

### 2.4 对拍与灰度
- 对拍命令：`app:dev:astro-shadow-compare`
- 灰度控制：平台覆盖、白名单、全局开关。
- 产出：`docs/devlogs/ASTRO_SHADOW_COMPARE.md/json`

### 2.5 调权与校准闭环
- 指标漏斗：`app:dev:pair-outcome-metrics`
- 校准数据导出：`app:dev:export-match-calibration`
- 评估报告：`app:eval:astro-v2`
- 自动化脚本：
  - `scripts/run_astro_calibration_cycle.ps1`
  - `scripts/generate_weight_candidates.ps1`
  - `scripts/generate_calibration_weekly_report.ps1`
  - `scripts/generate_calibration_wechat_brief.ps1`
  - `scripts/generate_calibration_reports_bundle.ps1`

### 2.6 校准注入安全网
- 注入命令加入显式开关、生产保护、`--force`、注入标记。
- 清理命令：`app:dev:cleanup-calibration-injected`
- 指标/导出默认排除注入样本（可显式包含）。

### 2.7 许可证与依赖门禁
- 新增许可证跟踪入口：`LICENSE_DEPENDENCY_STATUS.md`
- 依赖审计文档：`docs/dependency_audit_astrology.md`
- 服务端门禁配置：`config/astrology_dependency_gate.php`
