# 玄学算法 2.2 执行清单（2026-03-29）

## 目标
从“旧近似算法扩写”切换到“成熟开源底层 + 服务端 canonical + 灰度迁移 + 可验收”。

## 硬性执行规则（锁定）
1. 停止自研生产级天文/历法底层（黄经/ASC/宫位/节气/干支）。
2. Android 不再做 canonical 星盘计算；仅输入/展示/缓存/降级提示。
3. 生肖仅由年柱地支派生；公历年仅 legacy 回退。
4. MBTI 不再担任核心排序模块，仅保留展示+轻因子。
5. 第三方依赖先过许可证门禁（Swiss/AGPL 高风险）。
6. 服务端成为唯一真源（八字/星盘/关系盘）。

## 分阶段计划

### Phase 0 依赖与许可证审计
- 交付：`docs/dependency_audit_astrology.md`
- 内容：依赖、版本、许可证、闭源影响、商业授权需求、替代方案。
- 验收：Swiss / AGPL 依赖高亮；未审不允许默认上线。

### Phase 1 八字 canonical 服务化
- 新增 `BaziEngine` 服务接口（Laravel）。
- 首选外部能力：`lunar-java`/`lunar-python`；PHP 可先桥接。
- 统一输出：四柱、节气口径、生肖、大运、流年、五行、十神摘要、accuracy/confidence。
- 验收：新画像写入后端 canonical；生肖不再公历硬算。

### Phase 2 Tyme 评估
- 交付：`docs/tyme_vs_lunar_eval.md`
- 对比：API、接入成本、边界一致性、维护性。
- 验收：明确继续 Lunar 还是新模块改 Tyme。

### Phase 3 西占高精引擎接管
- 新增 `WesternEngine` canonical 服务。
- 生产候选：Swiss 体系（许可证未过仅研究）。
- 原型/对拍：`pyswisseph`/`Flatlib`/`Kerykeion`/`Astronomy Engine`。
- 验收：服务端独立生成 natal；返回 engine/precision/confidence。

### Phase 4 关系盘结构拆分
- 拆分接口层：
  - `natal_compatibility`
  - `synastry`
  - `composite_like`
- 拆分特征层：`SynastryFeatureExtractor`、`CompositeLikeFeatureExtractor`。

### Phase 5 对拍与灰度
- 增加 old/new 对拍脚本；输出 Sun/Moon/ASC/House/Aspect 差异。
- 灰度开关：global flag + whitelist + profile version + platform override。

### Phase 6 confidence policy
- 新增：`config/confidence_policy.php`
- 规则：无时间/无经纬度/仅日期/生肖回退/八字缺失/轻量 MBTI 等降级策略。

### Phase 7 八字深化 + 人格降权 + 埋点
- 八字证据：十神摘要、用忌神初版、夫妻宫标签。
- MBTI 降权并限定置信上限。
- 埋点：互喜欢/首条消息/24h回复/7天互动/解释查看率。
