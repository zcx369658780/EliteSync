## 6. Staging branch / staging clone 方案

未来真正执行 Laravel 12 dry-run 时，建议使用隔离 staging branch 与 staging clone。

建议 branch：

```text
staging/6.0-a1-laravel12-dry-run-20260514
```

建议 clone 原则：

- 使用独立 staging clone 目录，不使用 production 工作树直接 dry-run。
- production 默认继续 v1。
- 不使用 production DB。
- 不使用 production Redis keyspace。
- 不使用 production storage / OSS / CDN。
- 不使用真实用户账号。
- 不让 production traffic 进入 staging。
- Flutter 默认不切 v2。
- v2 只能 staging / controlled backend 验证。
- staging 必须有独立 `APP_ENV`、`APP_URL`、DB、Redis prefix、cache prefix、session prefix、queue prefix、storage path。
- staging dry-run 之前必须完成 backup / rollback / monitoring / load-test gate，或至少明确其未完成状态并作为 blocker/observation。

## 8. Rollback Plan

未来 dry-run rollback 要求：

- git rollback：保留 dry-run branch，可直接废弃或 reset staging branch。
- composer.json / composer.lock rollback：回到 dry-run 前 commit。
- vendor rollback：删除 staging clone vendor 并重新按原 lock 安装；不得污染 production vendor。
- staging branch 删除或废弃：由用户确认后执行。
- staging clone 删除或归档：由用户确认后执行。
- staging config rollback：恢复 staging `.env` / config snapshot。
- staging DB snapshot restore：仅在用户已授权 staging DB clone 后使用。
- staging Redis keyspace cleanup：清理 dry-run prefix，不碰 production keyspace。
- storage cleanup：仅清理 staging storage path / bucket prefix。
- no production rollback required，因为 production 不参与。
- 如果误触 production，应立即停止并生成 blocker report。

## 11. 与 env/cache/session plan 的衔接

承接 `6.0_A1_ENV_CACHE_SESSION_CONFIG_UNIFICATION_PLAN.md`：

- `APP_CACHE_STORE` 作为 cache 主口径。
- `APP_SESSION_DRIVER` 作为 session 主口径。
- `APP_SESSION_CONNECTION` 作为 session connection 主口径。
- `QUEUE_CONNECTION` 作为 queue 主口径。
- cache staging / production 目标为 Redis。
- session A1 目标建议 Redis session，但 production v1 可短期保留 database session 过渡。
- queue 目标为 Redis queue。
- queue worker 未常驻仍是 P1。
- production health 返回 `environment=local` 仍为 observation，后续需纳入 `APP_ENV` / config cache / environment split 检查。

Laravel 12 staging dry-run 前，必须先确认：

- `.env.example` 与 config 实际读取的 key 不再互相误导。
- test env key 口径能反映 app-scoped config key。
- Redis DB / prefix 按 local / staging / production 隔离。
- session driver 与 Sanctum 行为有独立回归计划。
