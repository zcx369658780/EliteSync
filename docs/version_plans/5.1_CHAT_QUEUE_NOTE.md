# EliteSync 5.1B Chat Queue Note

更新时间：2026-05-01

## 目标

5.1B 将 Chat 页的首聊 / 回聊 / 冷场恢复从静态破冰提示，升级为轻量关系回流队列。

本轮只做 Flutter presentation 层增强，不改消息 API，不新增本地待发送消息队列，不自动代发消息。

## 修改范围

- `apps/flutter_elitesync_module/lib/features/chat/presentation/widgets/icebreaker_card.dart`
- `apps/flutter_elitesync_module/lib/features/chat/presentation/pages/chat_room_page.dart`

## 当前实现

### 1. 建议卡增强

`IcebreakerCard` 现在支持：

- `lane`：建议所属队列，例如 `继续聊`、`回聊`、`冷场恢复`
- `source`：建议来源，例如 `匹配解释 / 共同点`、`最近互动`、`状态内容`
- `description`：使用场景说明
- `prompt`：写入输入框的草稿内容

点击建议后仍然只调用现有 `_applyIcebreakerSuggestion`，把文本放入输入框，不触发 `sendMessageUseCase`。

### 2. 队列内容

当前最小队列包括：

- `继续聊：从周末聊起`
- `继续聊：问最近状态`
- `回聊：接住话题`
- `回聊：从共同点延展`
- `稍后再回：低压回归`
- `冷场恢复：共鸣接起`
- `冷场恢复：周末安排`

这些提示保持慢约会口径，避免高压催聊。

## 保留主链

- 消息发送仍走 `POST /api/v1/messages`
- 附件发送仍走既有媒体主链
- 建议层不会创建本地消息
- 建议层不会写数据库
- 建议层不会绕过用户确认

## 风险与观察项

- 当前建议来源是 presentation 派生标注，不是后端真值字段。
- 如果后续要接入真实 match / status source metadata，必须先验证 payload，不得让缓存抢真值。
- 当前建议队列是本地 UI 队列，换设备不会同步；这是 5.1B 可接受 observation。

## 验收口径

- Chat 页可见 `继续聊 / 回聊 / 冷场恢复` 分层。
- Chat 页可见 `稍后再回` 队列，不只作为隐藏来源。
- 点击建议只写入草稿。
- 用户仍可编辑或清空草稿。
- 不出现自动发送、伪成功、工程 slug。
