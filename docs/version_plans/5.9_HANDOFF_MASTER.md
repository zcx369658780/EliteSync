## 1. 版本定位

5.9 是 **Chat 轻追问与低压开场版**。

本版本只在 Flutter Chat presentation 层新增 `低压开场建议`，把 Match / Me 的解释层能力轻量衔接到聊天开场点。建议内容是本地 deterministic 草稿候选，用户点击后只填入输入框，必须由用户自己确认发送。

## 5. Chat 低压开场说明

5.9 建议层为本地 deterministic presentation：

- 只使用固定本地文案和当前 Chat 页面已公开展示的关系推进语境。
- 点击建议只调用既有 `_applyIcebreakerSuggestion()` 写入 `TextEditingController`。
- 不调用 `_sendMessage()`。
- 不调用 `SendMessageUseCase`。
- 不触碰 `ChatRepository` / `ChatRemoteDataSource`。
- 草稿会进入本机输入框 / 本机草稿缓存，仍需用户手动点击发送。

固定免责声明：

```text
以上内容仅为开场参考，不会读取私密聊天，不会写入资料，也不会自动发送消息。草稿只保存在本机输入框 / 草稿缓存中。
```

## 6. no-auto-send / no-private-read contract

5.9 只允许：

- 本地固定建议。
- 点击填入可编辑草稿。
- 用户自行确认发送。

5.9 不允许：

- 自动发送消息。
- 自动创建消息。
- 自动通知对方。
- 读取真实聊天历史生成建议。
- 调用真实 AI。
- 写 `chat_messages`。
- 反写 profile / astro / match 真值。
