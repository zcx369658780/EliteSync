# EliteSync 3.1 风险评审

更新时间：2026-04-09

## 高风险模块

### 1. 账号分层与角色字段
- 风险点：
  - `role` 与 `account_type` 混用
  - 旧账号默认值回填错误
  - 普通用户 / 测试账号 / 管理员权限串线
- 风险等级：高

### 2. 测试账号生成 / 重建
- 风险点：
  - 批量生成污染真实指标
  - 重建脚本非幂等导致重复账号 / 重复状态
  - 画像补齐不完整导致匹配与状态发布依赖半成品数据
- 风险等级：高

### 3. 管理员 APP 内控制面
- 风险点：
  - 管理入口泄露给普通用户
  - 封禁 / 解封 / 删状态误操作
  - 发起每周匹配缺少审计或回查
- 风险等级：高

### 4. 状态发布与广场流
- 风险点：
  - 写入链路误走前端缓存
  - 测试内容污染普通广场
  - 首页 feed 首进被拖慢
- 风险等级：高

### 5. 匹配资格 / 可参与性控制
- 风险点：
  - `is_match_eligible` 默认值错误
  - 测试账号误入真实指标
  - 资格与封禁状态不同步
- 风险等级：高

### 6. 指标隔离
- 风险点：
  - `exclude_from_metrics` 缺失或默认错误
  - Beta 指标被测试流污染
  - 统计口径与业务口径不一致
- 风险等级：高

## 回滚建议

1. 所有新字段必须向后兼容，默认回退到正常用户语义。
2. 测试账号脚本必须可停用、可重建、可恢复。
3. 管理员高风险动作必须可通过开关迅速关闭。
4. 状态发布链路必须支持暂停写入并回退只读。
5. 匹配资格控制必须能回到现有默认逻辑。

## 兼容性注意

- 旧账号必须默认落到正常用户语义。
- 服务端 canonical 真值必须优先于前端缓存。
- 测试账号默认应隔离出运营指标。
- 管理员入口必须严格受控。

## 结论

3.1 是一个明显跨后端、数据层、权限层、前台 UI 的版本，必须按分层拆单推进，不能按单纯 UI 任务处理。
