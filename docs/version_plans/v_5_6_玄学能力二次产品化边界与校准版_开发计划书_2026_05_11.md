# 3. 总体产品原则

## 3.1 5.6+ 主线原则

5.6+ 主线不是继续堆工具，也不是复刻测测。

5.6+ 的目标是把 EliteSync 已有的：

- 星盘；
- 画像；
- 匹配解释；
- 关系理解；
- 慢约会表达建议；
- 低压开场；

转译为 Match / Me / Chat / Settings 主链中用户可理解、可忽略、可关闭的解释层。

## 3.2 Soul 与测测的分工

Soul 继续作为社交主链参考：

- Discover 复合入口；
- Chat 关系推进；
- Me 个人经营；
- Settings / Appearance 分层；
- 稍后再聊；
- 冷场恢复；
- 关系上下文；
- AI 破冰；
- 低压私聊转化。

测测只作为玄学解释层参考：

- 入口层；
- 上下文层；
- 解释对象层；
- 解释 / 建议层；
- 轻追问入口的克制参考。

不得把测测参考扩展成：

- 真人服务市场；
- 玄学咨询平台；
- 付费报告商店；
- 上传截图主路径；
- 娱乐化测试主路径；
- 强出生资料收集；
- 测试结果反写画像；
- 大规模社区讨论；
- 测测术语体系复制。

## 3.3 全路线硬约束

所有玄学解释、合盘解释、AI 追问必须保持：

```text
derived-only / display-only / explanation layer
```

不得反写：

```text
profile/basic
profile/astro/summary
profile/astro/chart
user_astro_profiles
```

不得改变：

- canonical truth；
- profile contract；
- astro contract；
- match contract；
- release contract；
- API contract；
- 数据库 schema；
- migration；
- version check；
- APK release chain。

## 6.3 5.6C：测测可吸收 / 不吸收清单固化

### 目标

把测测 Stage 1-5 的研究结果整理成 5.6+ 可执行边界。

### 必须保留的可吸收项

| 层级 | 可吸收结构 | EliteSync 转译 |
|---|---|---|
| 入口层 | 解释能力入口嵌入主链 | Match / Me / Chat / Settings 中的轻入口 |
| 上下文层 | 单人 / 双人上下文 | 个人资料上下文、匹配双方上下文 |
| 解释对象层 | 星盘、测试、合盘 | 关系理解、表达偏好、匹配理由 |
| 解释 / 建议层 | 摘要句、维度、建议 / 避免 | 关系摘要、慢约会适配、低压开场建议 |
| 轻追问层 | 问题入口可见 | disabled / coming soon / 用户可忽略占位 |

### 必须固化的不吸收项

| 禁止吸收项 | 原因 |
|---|---|
| 真人玄学咨询市场 | 与 EliteSync 慢约会主链不一致，且会引入服务市场和支付链 |
| 付费报告商店 | 当前不做商业化报告，不进入交易链 |
| 上传截图主路径 | 会引入隐私、权限、内容审核和非主线流程 |
| 娱乐化测试主路径 | 容易削弱资料真值和慢约会定位 |
| 强出生资料收集 | 与低压资料原则冲突 |
| 测试结果反写画像 | 违反 canonical truth 边界 |
| 测测术语体系复制 | EliteSync 应做自有产品语言，不复制竞品术语 |
| 大规模社区讨论 | 当前不是重社区平台 |
| 从竞品 UI 推断后端 contract | UI 研究不得推断接口、算法、DB、服务实现 |

### 输出要求

这些清单默认写入 `5.6_HANDOFF_MASTER.md`，作为后续 5.7-5.10 共同边界。
