## 3. Token 临时注入方式

建议方式：

- token 由用户在执行 smoke 当次通过 shell session 临时注入；
- PowerShell 示例：

```powershell
$env:ELITESYNC_TOKEN_A = "<TOKEN_FROM_USER_AT_RUNTIME>"
```

安全规则：

- token 不写入 Markdown 文档；
- token 不写入日志；
- token 不进入 git；
- token 不截图；
- Codex 不在报告中回显 token；
- Codex 不主动请求、生成、保存或粘贴 token；
- 如果任何输出中出现 token，立即停止并标记 security blocker；
- LiveKit token / secret 与 Sanctum token 同等级保护。

## 4. 建议第一批 readonly smoke 范围

根据 readiness check，建议第一批只允许以下低风险 readonly smoke：

1. `GET /api/v1/app/health`
2. `GET /api/v1/app/version/check`
3. release chain local readonly file check

如果用户确认 smoke token 与测试账号，再允许：

4. `GET /api/v1/profile/basic`
5. `GET /api/v1/profile/astro/summary`
6. `GET /api/v1/questionnaire/history`
7. `GET /api/v1/notifications/unread-count`

以下项目必须单独标为 conditional，不得默认执行：

- `GET /api/v1/profile/astro/chart`：可能写 chart cache。
- `GET /api/v1/messages`：可能写 read receipt。
- `GET /api/v1/media/{assetId}`：依赖 smoke asset id。
- `GET /api/v1/rtc/calls/{callId}/livekit`：依赖 smoke call id，且 LiveKit token 不得入报告。

## 5. 明确禁止本轮执行的 smoke

以下全部属于 write smoke 或有明确副作用，本轮不得执行：

- `POST /api/v1/profile/basic`
- `POST /api/v1/messages`
- `POST /api/v1/media`
- `POST /api/v1/questionnaire/answers`
- `POST /api/v1/rtc/calls`
- `POST /api/v1/rtc/calls/{callId}/heartbeat`
- `POST /api/v1/notifications/{id}/read`
- `POST /api/v1/notifications/read-all`

禁止原因：

- 这些接口会写 DB、改变通知 / 消息 / profile / RTC 状态，或可能触发 storage / notification side effect。
- 当前 write smoke 仍未获得用户逐项确认。
- 当前 cleanup / recovery 仍未按账号、数据、环境逐项确认。
- 本轮授权请求只覆盖 readonly smoke，不覆盖任何写入 smoke。

## 8. 停止条件

任一条件出现，必须停止：

- base URL 未确认，停止。
- APP_ENV 未确认，停止。
- DB 是否 production 未确认，停止。
- Redis / storage 是否 production 未确认且相关 smoke 需要它们，停止。
- token 处理未确认，停止。
- smoke 账号未确认，停止需要 auth 的 smoke。
- 任意输出包含 token，停止。
- 任意输出包含手机号 / 私聊正文 / 精确经纬度，停止。
- 任意输出包含 `.env` secret、DB host、Redis host、OSS key、SSH key，停止。
- git status 不干净且无法解释，停止。
- API 返回 5xx，停止后续同类 smoke。
- release chain 版本不一致，停止。
- 任何 POST 请求准备执行，停止。
- 任何 DB / migration / production 写操作准备执行，停止。
- 任何 v2 skeleton route / controller / DTO / Resource 准备新增，停止。
