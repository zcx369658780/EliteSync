当前特别观察：

- `GET /api/v1/profile/basic`、`GET /api/v1/profile/astro/summary`、`GET /api/v1/profile/astro/chart` 当前 contract 仍会返回 `birth_lat` / `birth_lng`。A1 文档设计阶段不得修改；后续位置展示清理必须优先从前端 display 层和 v2 display contract 规避普通用户可见坐标。
- `POST /api/v1/profile/basic` 保存后会重算并返回 `astro_profile` 快照，任何 v2 包装都不得跳过该重算链。
- `AstroProfileController@showChart` 可能写入 `users.private_natal_chart` 缓存；它不是纯 GET，无计划改动会影响 chart cache。
