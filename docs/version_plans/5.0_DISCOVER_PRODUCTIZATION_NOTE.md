# EliteSync 5.0B Discover 发现页产品化说明

更新时间：2026-05-01

## 结论

5.0B 的 Discover 目标不是把发现页做成重社区平台，而是把当前内容流骨架收敛成一个更清晰的**复合入口层**。

当前基线已在模拟器上补到以下结构：

- 顶部搜索入口；
- 顶部分栏：`热门 / 同城 / 活动 / 话题 / 直播`；
- 横向快捷入口：`同城活动 / 热点话题 / 语音房 / 兴趣圈`；
- Feed 卡片：热点话题、热门征集、同城活动复盘、你可能喜欢；
- 底部导航保持与首页一致。

这说明 Discover 已具备“发现、分流、引导、轻治理”的产品壳层，但还需要把结构语义继续做实，避免停留在单一内容流。

## 5.0B 的产品化方向

本轮只确认最小产品化方向，不扩写实现：

1. 分栏语义保持清晰；
2. 搜索入口不再像空壳；
3. 同城入口与内容卡片保持连通；
4. 卡片轻动作保持统一；
5. 不把 Discover 扩成重推荐平台或重社区。

## 当前基线观察

### 顶部搜索

- 占位文案：`搜索话题、活动、兴趣圈`
- 作用：作为 Discover 的统一检索入口
- 当前状态：已可聚焦，属于低风险入口

### 顶部分栏

- `热门`
- `同城`
- `活动`
- `话题`
- `直播`

建议后续产品化时保持“入口语义优先”，避免分栏变成纯视觉标签。

### 复合快捷入口

- `同城活动`
- `热点话题`
- `语音房`
- `兴趣圈`

这些入口说明 Discover 不是单一 feed，而是带有多种发现路径的复合面。

### Feed 卡片

当前可见卡片主要是：

- `热点话题：异地恋最难的不是距离`
- `热门征集：你最想参加哪类第一次约会活动？`
- `热门：同城活动后 24 小时内复盘最能提升二次见面率`
- `你可能喜欢`

这些卡片适合后续补：

- 分享
- 不喜欢
- 举报
- 复制链接或等价轻动作

## 低风险分支观察

### 搜索页

当前已确认搜索入口可聚焦，适合作为 Discover 的低风险检索壳层。

建议后续继续补：

- 热词 / 热门主题
- 空态 / 无结果态
- 返回路径

### 同城页

当前可见同城内容已形成独立内容流雏形，适合作为 Discover 的第二分支。

建议后续继续补：

- 按需定位提示
- 拒绝权限后的降级提示
- 默认占位说明

## 非目标

本轮不做：

- 重推荐算法改造
- 重社区平台化
- 复杂 moderation 中心拆分
- 商业化会员入口扩散

## 对后续实现的约束

Discover 的增强只应做 additive 承接：

- 不回头重开 4.x 已归档链路；
- 不破坏 UI protected surfaces；
- 不把分栏做成空视觉标签；
- 不把搜索页做成空页面。

## 交接口径

本文件作为 5.0B Discover 产品化最小增强的起点说明。
后续若补充实现，应继续配套：

- Discover UI evidence index
- 回归检查点
- 必要时的 handoff master 收口
