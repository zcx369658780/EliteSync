# EliteSync 5.1 单文件主交接

更新时间：2026-05-02

## 1. 版本定位

5.1 是 EliteSync 的 **关系转化与内容回流增强版**。

当前门禁基线：

- 当前发布基线：`0.04.09 / 40900`
- 4.9 状态：`pass with observations`
- 5.0 状态：`pass with observations`
- 当前工作分支：`feature/5.0-alpha-readiness-20260501`
- 当前 HEAD：`16a87090706a9c2990499c187411a781dc382b77`

5.1 的目标不是重开 4.x 主链，也不是重复 5.0 的 Discover / Chat / Me 最小壳层，而是在 4.9 稳定门禁与 5.0 产品化起点之上，补齐关系继续推进、内容自然回流、通知可处理、语音节奏更自然的产品链路。

当前验收口径：`pass with observations`。

GPT 顾问验收结论：`pass with observations`。

## 2. 当前已完成内容

### 2.1 5.1A 基线确认

- 已承接 4.9 与 5.0 的稳定基线；
- 已确认 5.1 范围、非目标、风险闸门和回滚边界；
- 已完成 plan-first 只读评审；
- 已固定本轮默认不改范围：
  - Laravel 后端业务逻辑；
  - 数据库 schema / migrations / seed；
  - notification / message / RTC API contract；
  - LiveKit join-info、heartbeat、RTC 状态机；
  - 匹配算法、分数、explanation composer；
  - profile canonical truth；
  - Android 初始化、Manifest 权限、第三方 SDK 初始化；
  - 版本发布链、下载链、version check；
  - repo 级回滚。

### 2.2 5.1B Chat 首聊 / 回聊 / 冷场恢复队列

- Chat 页的破冰提示已升级为轻量关系回流队列；
- `IcebreakerCard` 已支持 `lane`、`source`、`description`、`prompt`；
- 当前最小队列包含：
  - `继续聊`；
  - `回聊`；
  - `稍后再回`；
  - `冷场恢复`；
- 点击建议只写入输入框草稿，不触发真实发送。

保留主链：

- 消息发送仍走 `POST /api/v1/messages`；
- 附件发送仍走既有媒体主链；
- 建议层不会创建本地消息；
- 建议层不会写数据库；
- 建议层不会绕过用户确认。

### 2.3 5.1C 匹配解释到聊天建议联动

- `MatchDetailPage` 已新增 `解释到聊天` 行动卡；
- 完整解释、共同点、行动建议、关系节奏可转成可编辑聊天草稿；
- 若已有不同草稿，会提示用户选择 `取消 / 替换 / 追加`；
- 写入草稿后跳转到 `${AppRouteNames.chatRoom}/$peerId`；
- 未拿到有效 `partnerId` 时不伪成功。

保留主链：

- 不改匹配算法；
- 不改服务端 explanation composer；
- 不改 profile canonical truth；
- 不改 `/api/v1/matches/*`；
- 不改 `/api/v1/messages`；
- 不新增 route contract。

### 2.4 5.1D 状态 / 动态到私聊的低压转化

- `StatusPostCard` 已新增低压动作：
  - `继续了解`；
  - `看看资料`；
  - `稍后再聊`；
  - `低压私聊`；
- `StatusSquarePage` 已把低压动作接入状态卡；
- `StatusAuthorPage` 已新增 `低压回流` 卡；
- `低压私聊` 不直接伪造 ChatRoom，只提示匹配关系限制并进入作者页；
- `稍后再聊` 仅为本地 UI 提示，不写服务端队列。

保留主链：

- 不改 `StatusRemoteDataSource`；
- 不改 `StatusPostController`；
- 不新增 `status_posts` / `status_post_likes` schema；
- 不绕过消息主链的 matched + released + not blocked 规则；
- 不创建本地待发送消息；
- 不伪造私聊成功态。

### 2.5 5.1E 通知中心回流产品化

- 通知中心已从提醒列表增强为关系回流中枢；
- 通知卡新增分类 chip：
  - `消息回流`；
  - `动态互动`；
  - `匹配推进`；
  - `语音邀请`；
  - `低噪声提醒`；
- 通知卡新增显式动作：
  - `继续回复`；
  - `去看看`；
  - `查看匹配`；
  - `处理语音`；
  - `查看结果`；
  - `打开`；
  - `稍后处理`；
  - `标记已读`；
- 有跳转目标的主动作会先尝试打开页面，打开成功后才 `markRead`；
- 未知或缺失 route 不会被错误标记已读；
- 无 route 通知主动作显示 `仅标记已读`。

保留主链：

- 不改 `NotificationRemoteDataSource`；
- 不改 `NotificationItemEntity`；
- 不改 Laravel `NotificationController` / `NotificationService`；
- 不改 notification payload 语义；
- 不把站内通知写成厂商推送能力；
- 不改消息 / 动态 / 匹配 / RTC 后端主链。

### 2.6 5.1F 关系推进中的语音联动

- 聊天页 `关系摘要` 内已新增 `语音节奏` 区块；
- 顶部语音入口发起前会显示轻量确认弹窗；
- 用户选择 `现在语音` 后才继续既有权限检查与 `createCall` 流程；
- RTC 结果页已新增：
  - `通话后的继续建议`；
  - `未接通后的回聊建议`；
- 未接通、拒绝、忙线、失败时给出低压回聊文案；
- 结果页建议只展示，不自动生成或发送消息。

保留主链：

- 不改 `RtcRemoteDataSource`；
- 不改 `RtcLiveKitService`；
- 不改 `rtc_providers.dart`；
- 不改 RTC 后端 API；
- 不改 LiveKit token / join / leave 主链；
- 不改路由定义；
- 不改权限服务；
- 不扩多人语音、视频或直播能力。

### 2.7 5.1G 验收与单文件交接

- 已将 5.1 文字验收材料收敛为本文件；

## 5. 当前观察项

1. 5.1 当前仍维持 `0.04.09 / 40900`，未进入发包升级。
2. 5.1 实现主要是 Flutter presentation additive 增强，不代表后端 contract 已扩展。
3. `conversationId` 在聊天链路中仍按 `peerUserId` 使用，延续 5.1A 观察项。
4. Message 通知 payload 中 `conversation_id` / `peer_user_id` 方向仍需在真实通知数据下继续核验。
5. Chat / Match / Status 的正式截图证据仍不完整，当前由 widget tests 与局部截图共同支撑。
6. 聊天页进入已有消息会话后会自动滚到消息底部；`语音节奏` 区块需要手动回到顶部后采集。
7. `稍后再聊` 仍是本地 UI 表达，不是服务端队列。
8. RTC success result 的 `通话后的继续建议` 尚未采集正式截图。
9. 当前工作区不是 clean 状态，存在大量前序未提交改动与未跟踪文件；后续不得擅自 repo 级回滚。

5.2 承接判断：

- 适合进入 5.2 小单：Match / Status / Notification 证据补齐、RTC success result 截图、真实通知 payload 跳转方向复核、`match_detail_page_test.dart`。
- 不需要在 5.2 重复建设：Chat 轻队列、解释到聊天草稿链、状态低压 CTA、通知中心动作语义、语音前确认、RTC 未接通后回聊建议。
- 不应进入 5.2 的误扩展：后端 notification payload contract 扩展、RTC 状态机重写、自动代发消息、服务端 `稍后再聊` 队列、匹配算法重构。

## 9. 后续建议

- 继续保持 5.x 的 additive 产品化策略；
- 不把 5.1 扩成重社区、自动代聊、重通知系统或 RTC 重构；
- 若进入 5.2，应优先基于 5.1 observation 拆分为独立小单；
- 如果要发包，应另走版本链、Android build、安装验证、版本检查与发布证据流程。
- 5.2 更适合推进个人经营页与表达层增强，并吸收 5.1 observations 中的证据补采集和真实 payload 复核；不要重复建设 5.1 已通过的关系推进、通知回流和语音节奏能力。
