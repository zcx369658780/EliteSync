# EliteSync 3.4 保护面清单

更新时间：2026-04-11  
对应版本：3.4

## 1. 必须保护的核心链路

- 注册 / 登录
- 基础资料保存
- 出生时间 / 出生地点编辑
- 地点搜索与地理编码
- `POST /api/v1/profile/basic`
- `GET /api/v1/profile/astro/summary`
- `GET /api/v1/profile/astro/chart`
- 星盘主页面
- 星盘设置页
- 设置中心
- 版本中心页
- 隐私设置页
- 版本检查链路
- AdminUsersPage 统计显示

## 2. 必须保护的产品边界

- 服务端 canonical 真值仍是唯一真源
- 前端本地绘制与本地偏好只影响显示，不改真值
- 盘面设置页不得变成算法调参面板
- 版本中心与 health / version/check 只做可观测性

## 3. 必须保护的既有能力

- 3.2 已通过能力
- 3.2a 星盘本地绘制与盘面设置
- 3.3 资料重算链路与设置中心正式化
- synthetic / test / normal 分层
- Admin 客观治理视图
- 匹配 / 首聊 / 广场的已通过链路

## 4. 必须保护的显示规则

- 设置中心分组不要随意打散
- 版本中心、关于与更新、更新历史口径必须一致
- 星盘图层开关只影响本地 SVG 绘制
- 资料页和星盘页文案必须区分“显示偏好”和“真值”

## 5. 不能擅自触碰的模块

- 数据库 schema / migration / 初始化
- 地点搜索 / 地理编码服务
- 版本检查与发布脚本
- canonical 算法层
- 任何后端真值快照字段契约

## 6. 回归时必须检查的边界

- 保存资料后是否仍会触发服务端重算
- 星盘本地绘制是否仍正常
- 版本中心是否仍可查看版本与服务状态
- 设置中心是否仍能到达所有入口
- 隐私设置和性能偏好是否仍只影响本地偏好
