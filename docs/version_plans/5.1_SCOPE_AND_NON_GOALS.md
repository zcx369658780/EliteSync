# EliteSync 5.1 范围与非目标

更新时间：2026-05-01

## 本轮版本定位

5.1 是 **关系转化与内容回流增强版**。

本版本不重开 4.x 已归档主链，也不重复做 5.0 的 Discover / Chat / Me 最小壳层。5.1 只在 4.9 稳定门禁与 5.0 产品化起点之上，补齐关系继续推进、内容自然回流、通知可处理、语音节奏更自然的产品链路。

## 本轮范围

5.1 的正式范围分为以下工作包：

1. `5.1A`：基线确认与 5.0 结果承接；
2. `5.1B`：首聊 / 回聊 / 冷场恢复队列增强；
3. `5.1C`：匹配解释到聊天建议联动增强；
4. `5.1D`：动态 / 状态到私聊的低压转化增强；
5. `5.1E`：通知中心回流产品化增强；
6. `5.1F`：关系推进中的语音联动增强；
7. `5.1G`：证据、回归与单文件交接。

## 本轮允许改动

默认允许：

- Flutter presentation 层；
- 轻量 UI 文案；
- CTA 排布；
- 建议卡和关系摘要展示；
- 现有路由回流；
- 现有聊天草稿链；
- 截图证据索引；
- 5.1 版本计划、执行、回归、验收、handoff 文档。

谨慎允许：

- `app_router.dart` 与 route args，仅限补齐现有页面间跳转；
- 通知中心 route 分发，仅限向后兼容的 additive 分流；
- 本地缓存，仅限草稿、轻量偏好、稍后再看类兜底，不作为主数据源。

## 本轮默认不改

默认不改：

1. 不改 Laravel 后端业务逻辑；
2. 不改数据库 schema / migrations / seed；
3. 不改远端数据库、备份、恢复、生产配置；
4. 不改 notification / message / RTC API contract；
5. 不改 LiveKit join-info、heartbeat、RTC 状态机；
6. 不改匹配算法、分数、explanation composer；
7. 不改 profile canonical truth；
8. 不改 Android 初始化、Manifest 权限、第三方 SDK 初始化；
9. 不改版本发布链、下载链、version check；
10. 不做 repo 级回滚。

若后续发现确实必须触碰上述范围，必须暂停当前工作包，先写 blocker report，再进入新的 plan-first 流程。

## 明确非目标

5.1 明确不做：

1. 不做大模型首聊平台；
2. 不做自动代发消息；
3. 不做本地待发送消息队列替代服务端消息；
4. 不做重社区化动态平台；
5. 不做评论 / 关注 / 复杂广场运营体系；
6. 不做多人 RTC、直播、视频通话增强；
7. 不做会员、商城、币体系、装扮商业化；
8. 不做大推荐算法重构；
9. 不做复杂活动系统；
10. 不做本地写库型后端改造；
11. 不伪造通知、聊天、状态、语音的成功态；
12. 不沿用旧包或旧截图冒充 5.1 证据。

## 工作包边界

### 5.1B + 5.1C：Chat / Match 表达层

允许：

- 扩展聊天页建议卡；
- 将匹配解释、共同点、主页标签转成可编辑草稿；
- 补 `继续聊 / 稍后再回 / 冷场后重新接起` 的轻队列；
- 补建议来源标注。

禁止：

- 自动发送消息；
- 改 `/api/v1/messages`；
- 改匹配算法或服务端解释链；
- 将缓存或本地快照当成真值。

### 5.1D：Status / Content 回流

允许：

- 增加低压 CTA；
- 将状态作者页和状态卡片的动作语义产品化；
- 增加本地的 `稍后再聊` 表达或提示。

禁止：

- 新增 schema；
- 把状态流做成重社区；
- 对未匹配对象伪造私聊成功；
- 新增定位 / 地图 SDK 能力。

### 5.1E：Notification 回流

允许：

- 通知分类和文案重排；
- 显式化 `继续回复 / 去看看 / 回到资料 / 稍后处理 / 标记已读`；
- 复核现有 `route_name` 与 `route_args`。

禁止：

- 改后端 payload 语义；
- 删除旧 route 兼容；
- 把站内通知写成厂商推送能力；
- 用已读成功掩盖跳转失败。

### 5.1F：Voice rhythm

允许：

- 增加语音邀请前提示；
- 增加未接通或结束后的回聊建议；
- 结束后回到聊天页或消息页时保留关系节奏文案。

禁止：

- 改 RTC 状态机；
- 改 LiveKit；
- 改 heartbeat；
- 绕过麦克风权限；
- 扩多人 / 视频 / 直播。

## 手工验收重点

后续 5.1B-F 的用户面验收必须覆盖：

1. Chat 首聊 / 回聊 / 冷场恢复队列可见，点击建议只填充草稿，不自动发送；
2. 匹配解释或关系摘要能自然进入聊天建议；
3. 状态 / 动态卡片的 `继续了解 / 看看资料 / 低压私聊 / 稍后再聊` 不高压、不伪成功；
4. 通知中心不同类型通知能回到正确页面；
5. 通知标记已读不掩盖跳转失败；
6. 语音邀请前权限链路仍正常；
7. RTC 未接通 / 结束后能自然回到消息或聊天；
8. 版本中心仍显示当前基线，除非明确发包；
9. 截图文件名、截图内容、页面实际内容三者一致。

## 回滚与恢复

- 禁止 repo 级回滚；
- 只允许 path-level / file-level / smallest-scope recovery；
- 修改 UI protected surfaces 前必须记录改动范围；
- Chat 建议层的最小回滚面是 `chat_room_page.dart` 与相关 widget；
- Notification 的最小回滚面是 `notification_center_page.dart`；
- Status CTA 的最小回滚面是 `status_post_card.dart` 与 `status_square_page.dart`；
- Voice rhythm 的最小回滚面应优先限制在 Flutter UI，不触碰 `RtcSessionService`、`RtcRemoteDataSource`、LiveKit 与 heartbeat。

## 5.1A 结论

5.1A 只确认起点、范围、风险与非目标，不进入业务实现。

当前允许继续进入 `5.1B + 5.1C`，但前提是：

- 先复核 Chat / Match 当前 diff；
- 明确拟修改文件；
- 不改后端和数据库；
- 不自动发送消息；
- 不使用缓存抢真值；
- 实现后运行 Chat 相关最小测试并记录结果。
