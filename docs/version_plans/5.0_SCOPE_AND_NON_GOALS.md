# EliteSync 5.0 范围与非目标

更新时间：2026-05-01

## 本轮范围

5.0A 只做以下最小工作：

1. 确认 5.0 从 `4.9` 稳定门禁基线承接；
2. 建立安全起点说明；
3. 固定 5.0 的范围与非目标；
4. 为后续 Discover / Chat / Me 产品化补强预留统一口径。

## 本轮非目标

本轮不做：

1. 不改业务代码；
2. 不改后端逻辑；
3. 不改数据库 schema；
4. 不重开 4.x 已归档主链；
5. 不扩写 Discover / Chat / Me 的具体实现；
6. 不重写 RTC、媒体、通知、版本链；
7. 不做 repo 级回滚；
8. 不做商业化、多人 RTC、重社区化改造。

## 5.0A 的工作原则

- 只确认起点，不扩散实现；
- 只冻结边界，不顺手改别的内容；
- 只保留当前稳定门禁基线，不回头重开旧版本；
- 如果后续进入实现，必须先补对应的计划、风险、测试与验收材料。
