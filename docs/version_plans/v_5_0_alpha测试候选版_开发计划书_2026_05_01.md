# 0. 一句话结论

5.0 不重开 4.x 已归档主链，不再继续做纯稳定化壳层，也不提前做重商业化、重娱乐化或重微服务化。

本版本目标是：

> 在 4.9 稳定门禁基线之上，完成 Discover / Chat / Me 三大模块的**最小产品化补强**，让 EliteSync 从“功能骨架已齐”进入“可用于 Alpha 试用准备”的产品状态。

5.0 的核心成果不是“再加一批零散功能”，而是：

1. Discover 从单一内容骨架升级成复合入口层；
2. Chat 从消息容器升级成关系推进层；
3. Me 从资料页升级成个人经营中枢；
4. AI 从孤立能力改为嵌入首聊、回聊、个人经营与展示建议；
5. 所有新增能力都在 4.9 release gate / UI baseline / DB safety / RTC / media / notification 主链上 additive 承接；
6. 为后续小样本测试准备更像产品的核心路径，而不是继续堆治理文档。

# 1. 版本定位

## 1.1 版本名称

**5.0 Alpha 测试候选版 / 产品化补强最小集**

## 1.2 版本类型

本版本属于：

- 5.x 正式起点版本；
- 产品化补强首版；
- Alpha 测试候选版；
- Discover / Chat / Me 三主模块增强版；
- AI 轻嵌入与关系推进增强版。

它不是：

- 继续治理壳层版；
- 重做主链版；
- 运营后台平台化版；
- 商业化变现版；
- 重娱乐玩法版；
- 微服务重构版；
- 多人 RTC / 直播 / 群聊房版本；
- 大推荐算法改造版。

## 1.3 当前承接基线

默认承接以下已完成或已冻结基线：

- `3.9`：高级时法与解释边界已归档；
- `4.0`：媒体基础设施、队列、缓存、worker 最小闭环已归档；
- `4.1`：非官方四维人格问卷主链已归档；
- `4.2`：图片消息正式接入版已归档；
- `4.3`：动态流基础版已归档；
- `4.4 / 4.4S`：视频消息与媒体链稳定性修正版已归档；
- `4.5 / 4.5E`：站内通知与社交转化增强版已归档；
- `4.6P`：1v1 真语音闭环已归档；
- `4.7`：UI protected surfaces、rollback / recovery policy 已归档；
- `4.8`：Alpha smoke 与真实路径复验已归档；
- `4.9`：测试前治理、限流、监控、发布链强化版已按 `pass with observations` 冻结，作为 5.x 稳定门禁基线。

5.0 的第一原则是：

> 只从 4.9 additive baseline 承接，不回头重开旧主链，不以补功能为由破坏 4.9 的稳定门禁资产。

# 2. 背景与问题来源

## 2.1 为什么需要 5.0

当前项目已经完成：

- 动态、图片、视频、通知、1v1 真语音、版本中心、release gate、health、数据库正式演练、UI baseline 保护等主干能力；
- 4.9 已把测试前最容易出事故的治理、门禁、可观测性问题收口；
- 继续沿“纯稳定化 / 纯治理”推进，边际收益正在下降。

但当前仍存在一个核心现实：

> 产品主链虽已齐，但 Discover / Chat / Me 三个模块整体仍更像“已可用骨架”，尚未完全进入“真实 Alpha 用户会觉得顺手”的状态。

因此 5.0 的任务不是“再补一个基础设施块”，而是：

- 让 Discover 真正承担内容发现、同城、搜索和轻治理入口；
- 让 Chat 真正承担首聊、回聊、关系推进和 AI 续话；
- 让 Me 真正承担资料展示、标签表达、内容经营和 AI 助理入口；
- 把 Soul 拆解中真正有价值的结构能力改写成 EliteSync 的慢约会语言。

## 2.2 5.0 要解决的核心问题

本版本重点解决以下问题：

1. 发现页目前仍偏骨架，缺少更明确的分栏、搜索与同城结构；
2. 聊天页仍偏消息容器，缺少首聊 / 回聊辅助和关系摘要；
3. 个人页仍偏资料归档，缺少“个人经营中枢”的产品结构；
4. AI 能力尚未足够自然地嵌入慢约会主链；
5. Discover / Chat / Me 的产品化增强必须在 4.9 门禁基线之上完成，而不是破坏现有稳定资产。

# 5.2 5.0B：Discover 发现页产品化最小增强

## 目标

把 Discover 从“内容流骨架”升级成“复合入口层”，承担推荐 / 同城 / 搜索 / 轻治理动作的统一发现体验。

## 建议范围

### A. 分栏正式化

建议至少形成以下三分结构之一：

- `推荐 / 同城 / 关注`
- 或 `推荐 / 同城 / 话题`

要求：
- 结构清晰；
- tab 之间语义明确；
- 不做大推荐算法改造；
- 不把 Discover 扩成重社区。

### B. 搜索页产品化

建议补齐：
- 搜索框；
- 热词 / 热门主题；
- 提示卡；
- 空态与无结果态；
- 返回路径清晰。

### C. 同城内容分支

建议补齐：
- 同城入口；
- 按需申请定位权限；
- 拒绝权限时的降级提示；
- 不依赖定位时的默认占位说明。

### D. 卡片 action sheet

建议统一承载：
- 分享；
- 不喜欢；
- 举报；
- 复制链接或等价轻动作。

## 任务

1. 梳理当前 Discover 页面结构；
2. 最小增强分栏与搜索页；
3. 补 action sheet；
4. 补同城路径最小壳层；
5. 输出：
   - `docs/version_plans/5.0_DISCOVER_PRODUCTIZATION_NOTE.md`
   - `docs/version_plans/5.0_DISCOVER_UI_EVIDENCE_INDEX.md`

## 禁止事项

- 不做重社区平台；
- 不新增复杂推荐平台；
- 不把治理动作拆成多个重按钮；
- 不先弹定位权限再解释价值。

## 验收标准

- Discover 顶部分栏语义明确；
- 搜索页不再是空入口；
- 同城路径可解释、可降级；

- `GET /api/v1/messages`

- 日志中不记录明文密码、完整 token、私钥、LiveKit secret、OSS secret；
- 聊天与个人页增强涉及的用户内容应做脱敏；
- 截图若包含敏感会话内容，应按需打码。

# 8. 跨层 Blocker 处理条款

5.0 虽然是产品化补强版，但仍可能涉及：
- Discover 数据源；
- 搜索；
- 同城权限；
- 消息 / 通知 / 媒体 / RTC 联动；
- 版本与发布链。

因此继续强制执行跨层 blocker 规则。

## 8.1 触发条件

出现以下任一情况，立刻进入 blocker 流程：

- 页面显示成功，但真实效果失败；
- Discover tab / 搜索页看似正常，但数据回流错误；
- 聊天 AI 入口存在，但实际建议不可用；
- 首聊 / 回聊路径存在，但不能回读；
- 语音入口存在，但 RTC 失败；
- 媒体 / 通知 / version check 行为与 UI 矛盾；
- 构建、端口、协议、网络、云端配置异常。

## 8.2 标准流程

1. 冻结现象；
2. 先写 Markdown blocker report；
3. 请 Claude 做根因缩层；
4. Codex 只做最小修复；
5. 用机器证据 + 用户面证据双重复验；
6. 写 fix note / verification note / acceptance summary。

## 8.3 禁止事项

- 禁止 Codex 连续多轮只在 UI 层盲修；
- 禁止没有 blocker report 就继续试；
- 禁止用假成功掩盖真实失败；
- 禁止根因未定前大范围重构。

# 9. Rollback / Recovery Policy

## 9.1 禁止 repo 级恢复

默认禁止：

```bash
git reset --hard <old_commit>
git checkout <old_commit> .
git restore --source=<old_commit> .
git clean -fdx
```

也禁止：
- 用旧 zip 覆盖当前项目；
- 为修 Discover / Chat / Me 局部问题恢复整仓；
- 恢复旧 UI 覆盖 protected surfaces。

## 9.2 允许的恢复方式

只允许：
- path-level restore；
- file-level restore；
- smallest-scope restore。

## 9.3 恢复前必须 checkpoint

恢复前必须执行：

```bash
git status --short
git branch --show-current
git rev-parse HEAD
git diff --stat
git diff --name-only
```

并至少执行一种：

```bash
git switch -c safety/pre-recovery-<date>-<topic>
```

或：

```bash
git add -A
git commit -m "safety: checkpoint before <topic> recovery"
```

或：

```bash
git stash push -u -m "safety: checkpoint before <topic> recovery"
```
