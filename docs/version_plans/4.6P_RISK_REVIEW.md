# 4.6P 风险评审

更新时间：2026-04-24

## 高风险点

1. **把播放链问题继续当成连接层问题**
   - 风险：反复回到旧连接排障，错过真正的 publish / subscribe / playback 问题
   - 处理：本版只盯音频阶段证据与播放链

2. **把音频频谱当成播放证明**
   - 风险：有帧不代表有声，波形不动也不代表订阅失败
   - 处理：频谱只做诊断辅助，不能代替实际听觉验证

3. **模拟器表现误导真机判断**
   - 风险：模拟器有帧、真机没帧，或者反过来，结论会被带偏
   - 处理：真机 / 模拟器双向最小复验

4. **音频路由改动误伤通话返回与终态**
   - 风险：为了修声音把返回消息页、挂断、结果页链路弄坏
   - 处理：只做音频路由的最小修复

5. **调试串继续污染用户面**
   - 风险：把 heartbeat / ended / token / track id 等调试语义直接暴露给用户
   - 处理：仅保留诊断层输出，用户面保持最小可读

## 回滚点

- 保留现有 RTC 状态机独立运行
- LiveKit 接入失败时仍可退回状态机 + 通知骨架
- 不动 4.0 / 4.2 / 4.4 / 4.5 主链
