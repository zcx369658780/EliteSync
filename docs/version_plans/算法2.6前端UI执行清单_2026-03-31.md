# 算法 2.6 前端 UI 执行清单

日期：2026-03-31

> 顺序：我的页重构 -> 玄学可视化 -> 资料输入交互 -> 状态提示 -> 发布联动

---

## P0：页面定位收口

- “我的”页升级为：
  - 基础资料中心
  - 玄学资料录入中心
  - 玄学画像查看中心
  - 资料更新状态中心
- 不在客户端本地做 canonical 计算。

### 验收
- “我的”页不再只是静态资料页。

---

## P1：我的页面重构

### 必做区块
- 基础资料
- 玄学资料
- 玄学画像入口
- 系统与安全

### 推荐入口
- 八字
- 星盘
- 紫微
- 生肖 / 星座
- 玄学总览

### 验收
- 用户能从“我的”页进入各画像详情。

---

## P2：玄学可视化

### 八字
- 总览卡
- 五行雷达 / 柱状图
- 四柱表
- 关系与标签摘要

### 星盘
- 简化轮盘
- 三层摘要
- 降级提示

### 紫微
- 命宫 / 身宫入口卡
- 简化宫盘
- 宫位摘要

### 生肖
- 轻量桥接模块
- 与八字联动展示

### 玄学总览
- 模块状态
- 最近更新时间
- 一句话摘要

### 验收
- 八字 / 星盘 / 紫微至少各有一个稳定详情页。

---

## P3：资料输入与实时更新

### 目标
- 用户修改资料后，前端要明确显示“正在更新画像”。

### 必做交互
- 修改生日 / 出生时间 / 出生地 / 经纬度后：
  - 显示更新中状态
  - 自动刷新摘要
  - 失败可重试

### 文案
- `保存后将自动更新八字 / 星盘 / 紫微画像`
- `画像更新中，请稍后刷新`
- `部分画像更新失败，可重试`

### 验收
- 前端不再出现“修改了资料但页面看起来没变”的错觉。

---

## P4：版本更新与备份入口提示

### 目标
- 在前端保留轻量安全提示，不把备份做成复杂功能页。

### 内容
- 数据安全说明
- 版本更新入口
- 当前画像状态说明

### 验收
- 前端只做提示，不替代后端备份流程。
