## 2. Base URL / APP_ENV / DB Readiness

当前 `<BASE_URL>` 尚未确认。执行 smoke 前必须确认该 base URL 属于以下之一：

- local controlled backend；
- staging backend；
- 用户明确授权的 controlled backend。

默认禁止使用 production 真实用户链路作为 smoke 环境。若用户后续要求 production readonly smoke，必须单独确认目标、范围、账号、隐私脱敏和停止条件；本阶段不默认允许。

当前 APP_ENV 尚未确认。若无法确认 APP_ENV 是 local / staging / controlled，必须标记为 blocked，不得执行 smoke。

当前 DB 指向尚未确认。执行 smoke 前必须确认：

- DB 不是 production DB；
- write smoke 使用 staging / controlled DB；
- DB 写入有可恢复数据集、smoke 账号或 cleanup 方案；
- 报告只记录 DB 环境脱敏摘要，不输出真实 DB host、账号或 secret。

当前 Redis 指向尚未确认。执行 smoke 前必须确认：

- Redis 不使用 production keyspace；
- cache / queue / session / rate limit 的 DB / prefix 不与 production 混用；
- 报告不输出真实 Redis host、密码或 secret。

当前 storage / media 指向尚未确认。执行 media smoke 前必须确认：

- media 写入不会落到 production OSS/CDN；
- smoke asset 属于隔离 storage 或可清理路径；
- 报告不输出 OSS key、真实 bucket secret 或隐私文件名。

只要 base URL、APP_ENV、DB、Redis、storage 任一项无法确认，就必须标记 blocked，不得执行 smoke。

## 4. Token Handling Readiness

- token 只能由用户或授权流程在执行 smoke 时临时提供。
- token 只允许保存在当前 shell session / 临时变量。
- token 不得写入 Markdown 文档。
- token 不得进入日志附件。
- token 不得进入截图。
- token 不得进入 git。
- token 不得由 Codex 主动请求、生成、保存或粘贴到报告。
- 任何包含 token 的输出必须立即停止，并标记为 security blocker。
- LiveKit token / secret 与普通 Sanctum token 一样受保护，不得输出到报告。

## 9. Evidence / Report Redaction Rules

smoke 报告不得包含：

- token；
- 手机号；
- 私聊正文；
- 完整出生时间；
- 精确经纬度；
- DB host；
- Redis host；
- OSS key；
- SSH key；
- `.env` secret；
- LiveKit token / secret；
- 真实用户身份信息；
- 真实生产 URL；
- 真实生产账号。

smoke 报告允许包含：

- HTTP status；
- pass / fail / skipped / blocked；
- 响应字段摘要；
- 脱敏 base URL；
- smoke ID；
- 是否写 DB；
- 是否已 cleanup；
- observation；
- 脱敏环境标签，如 local / staging / controlled backend。

任一敏感字段进入输出、日志、截图或 git，必须立即停止 smoke，并标记为 security blocker。
