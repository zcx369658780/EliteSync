## 2. 本轮拟授权的 authenticated readonly smoke 范围

本轮只允许规划以下 authenticated readonly smoke。所有接口都需要 token，但默认应只读、不得写 DB。本阶段只做授权计划，不执行这些接口。

| Smoke ID | Endpoint | Method | Auth | 默认写 DB | 前置条件 | 报告摘要允许项 | 停止条件 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| AUTH-RO-01 | `/api/v1/profile/basic` | GET | Sanctum token | no | 用户确认 SMOKE_USER_A 是测试账号，token 临时注入 | HTTP status、字段存在性、是否有 profile baseline，不输出隐私字段 | 401/403、5xx、响应含不应入报告的隐私字段、任何写入迹象 |
| AUTH-RO-02 | `/api/v1/profile/astro/summary` | GET | Sanctum token | no | SMOKE_USER_A 已有 astro summary / canonical truth | HTTP status、`exists`、summary 字段摘要，不输出完整出生时间、经纬度或 chart | 401/403、5xx、canonical truth 异常、敏感字段无法脱敏 |
| AUTH-RO-03 | `/api/v1/questionnaire/history` | GET | Sanctum token | no | SMOKE_USER_A 已有或允许为空的 questionnaire history | HTTP status、items/count 摘要，不输出完整答案 | 401/403、5xx、结构破坏、完整答案进入报告 |
| AUTH-RO-04 | `/api/v1/notifications/unread-count` | GET | Sanctum token | no | SMOKE_USER_A 可用于 notification unread-count 读取 | HTTP status、unread count 摘要，不输出 notification body | 401/403、5xx、真实用户通知信息进入报告 |

本轮不授权任何需要 peer、asset id、call id、storage 访问或可能触发 read receipt / cache write 的接口。

## 3. 本轮仍不建议授权的 conditional / side-effect smoke

以下项目不进入本轮默认授权，只能后续单独授权：

| Endpoint | Method | 当前结论 | 原因 | 后续授权前置 |
| --- | --- | --- | --- | --- |
| `/api/v1/profile/astro/chart` | GET | 暂不默认授权 | 可能写 chart cache，不是严格纯读 | 用户确认 chart cache side effect 可接受，SMOKE_USER_A astro truth 已确认，报告脱敏规则已确认 |
| `/api/v1/messages` | GET | 暂不默认授权 | 可能写 read receipt，且涉及私聊正文隐私 | 用户确认 peer smoke user、conversation、read receipt side effect 与正文脱敏 |
| `/api/v1/media/{assetId}` | GET | 暂不默认授权 | 依赖 smoke asset id，且需要确认 storage / asset 来源 | 用户确认 asset 属于 smoke 数据，storage 不触碰 production 隐私资源 |
| `/api/v1/rtc/calls/{callId}/livekit` | GET | 暂不默认授权 | 依赖 smoke call id，且 LiveKit token 不得进入报告 | 用户确认 smoke call id，报告只输出字段存在性，不输出 LiveKit token / secret |

这些接口不得被包含在下一批默认 authenticated readonly smoke 中，除非用户逐项勾选并确认对应 side effect 与脱敏规则。

## 6. Token 临时注入规则

- token 只允许由用户在执行 smoke 当次注入 shell session。
- 推荐 PowerShell：

```powershell
$env:ELITESYNC_TOKEN_A = "<TOKEN_FROM_USER_AT_RUNTIME>"
```

- token 不写入 Markdown 文档。
- token 不写入日志。
- token 不进入 git。
- token 不截图。
- Codex 不在报告中回显 token。
- Codex 不主动请求、生成、保存、粘贴或转存 token。
- 如果任何输出中出现 token，立即停止并标记 security blocker。
- 如果后续涉及 LiveKit token / secret，应按同等级别保护，但本轮不授权 RTC LiveKit smoke。

## 7. Authenticated readonly smoke 执行顺序建议

如下一阶段获得用户明确授权，建议按以下顺序执行：

1. profile basic GET
2. astro summary GET
3. questionnaire history GET
4. notifications unread-count GET

执行停止规则：

- 任一接口返回 `401` / `403`：停止后续 auth smoke，标记 token/auth blocker。
- 任一接口返回 `5xx`：停止后续同类 smoke，标记 backend blocker。
- 响应中包含不应进入报告的敏感字段：停止并脱敏，必要时标记 security blocker。
- 任何迹象显示写 DB：停止，并把该接口从 authenticated readonly 范围移出。
- 任一接口需要 POST、read-all、profile write、message send、media upload、RTC heartbeat：停止，本轮不授权。

## 8. 报告脱敏要求

后续 authenticated smoke 报告不得包含：

- token；
- 手机号；
- 密码；
- 私聊正文；
- 完整出生时间；
- 精确经纬度；
- DB host；
- Redis host；
- OSS key；
- SSH key；
- `.env` secret；
- LiveKit token / secret；
- 真实用户身份信息。

允许包含：

- HTTP status；
- pass / fail / skipped / blocked；
- 响应字段摘要；
- 是否 auth required；
- 是否写 DB：应为 no；
- observation；
- 脱敏 base URL 或已授权可显示的 BASE_URL；
- smoke ID 与 endpoint。

若报告生成过程中无法可靠脱敏，应停止并把对应 smoke 标记为 blocked。
