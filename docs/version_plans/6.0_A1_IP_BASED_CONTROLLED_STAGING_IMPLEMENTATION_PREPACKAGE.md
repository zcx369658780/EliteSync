## 2. 推荐实施路线

Candidate options:

- Option A: same server + separate port + security group allowlist。
- Option B: same server + SSH tunnel only。
- Option C: separate lightweight staging server。

Codex recommendation:

- 默认推荐 `Option A: same server + separate port + security group allowlist`，前提是用户明确允许 Nginx / 安全组 / staging env / staging DB / Redis / storage 边界配置，并且可以在执行前备份、执行中 `nginx -t`、失败时回滚。
- 如果用户希望最大限度避免公网暴露，推荐 `Option B: same server + SSH tunnel only`，它更适合极窄 readonly metadata verification，但 Codex / 外部工具访问流程会更复杂。
- `Option C: separate lightweight staging server` 边界最清晰，但对当前 A1 readonly metadata verification 来说成本和配置工作量更高，可作为长期 staging / QA 方向。
- 不推荐使用 `101.133.161.203` 根路径。
- 不推荐使用 `slowdate.top`。
- 不推荐继续等待 `staging.slowdate.top`，除非未来备案完成。

Decision guidance:

| Option | Recommended use | Main reason | Main risk |
|---|---|---|---|
| A | operational default if controlled server changes are allowed | repeatable IP staging URL with allowlisted access | same-server production impact if isolation or rollback is weak |
| B | safest exposure profile | no public staging port required | harder for external tools and evidence replay |
| C | long-term clean staging | strongest environment boundary | higher cost and setup work |

## 3. 最小服务器变更清单草案

This section is a draft only. No command is executed by this prepackage.

### Option A future change draft

If Option A is selected, future execution may need:

- 新增 staging 端口，例如 `8081` 或其他用户确认端口。
- 新增 Nginx server block / location。
- 新增独立 Laravel `APP_ENV=staging`。
- 新增独立 staging env 文件。
- 新增独立 staging DB 或测试库。
- 新增独立 Redis prefix / DB index。
- 新增独立 storage path。
- 新增 environment marker。
- 安全组仅对白名单 IP 开放 staging 端口。
- 不影响 `slowdate.top`。
- 不影响 Android release/default base URL。
- 不影响 Flutter default base URL。
- 不影响 release chain。

### Option B future change draft

If Option B is selected, future execution may need:

- staging 服务仅绑定 localhost。
- 通过 SSH tunnel 暴露给用户本机。
- 不开放公网端口。
- 记录 tunnel endpoint as redacted local endpoint only, for example `<LOCAL_TUNNEL_ENDPOINT>`.
- 只通过 tunnel 对 R1 readonly metadata endpoints 做 verification。
- 局限：Codex / 外部工具访问不如公网端口直接。

## 4. 数据隔离要求

- 不得使用 production DB 写链。
- 优先使用独立 staging DB / 测试库。
- 若临时只读连接 production DB，必须另行授权，且只能读 health / metadata，不得读用户数据。
- Redis 必须使用独立 prefix / DB index。
- storage 必须使用独立路径。
- `app_release_versions` production release chain 不得受影响。
- 不允许真实用户写操作。
- 不允许 migration。
- 不允许 seed / import production data。
- 不允许清库。

## 5. 配置隔离要求

- `APP_ENV=staging`。
- `APP_DEBUG=false` 或受控。
- `APP_URL` 指向 IP staging endpoint 或 localhost tunnel endpoint。
- `LOG_CHANNEL` / `LOG_LEVEL` 不得泄露 secrets。
- `CACHE_PREFIX` / `SESSION_DOMAIN` / `QUEUE_CONNECTION` / `REDIS` prefix 必须与 production 区分。
- `FILESYSTEM_DISK` / storage path 必须与 production 区分。
- 不读取或输出真实 `.env` / `.env.*` 内容。
- 所有 env value 在报告中必须 redacted。

## 6. Nginx / 安全组 / 回滚要求

未来执行前必须满足：

- 修改 Nginx 前备份现有配置。
- 修改安全组前记录原规则。
- staging 端口只对白名单开放，或仅本机 / SSH tunnel。
- Nginx reload 前必须 `nginx -t`。
- 若 Nginx test failed，停止，不 reload。
- 若 reload 后异常，立即回滚。
- staging 端口必须可关闭。
- 不影响 80 / 443 production/default 服务。
- 不影响 `slowdate.top`。

## 10. Risks and mitigations

| Risk | Mitigation |
|---|---|
| 同机 staging 误影响 production | 先备份 Nginx / config；独立端口；独立 env；`nginx -t` pass 后才 reload；保留 rollback plan |
| Nginx reload 风险 | reload 前必须 `nginx -t`；test failed 即停止；reload 后立刻做 readonly boundary check |
| 安全组误开放风险 | staging 端口只对白名单 IP 开放；记录原规则；保留 close-port rollback |
| env 混用风险 | 独立 staging env；`APP_ENV=staging`；所有 env value redacted；不读取真实 `.env` 内容 |
| DB / Redis / storage 混用风险 | 独立 staging DB / 测试库；独立 Redis prefix / DB index；独立 storage path |
| release chain 误改风险 | 不运行 release scripts；不改 `app_release_versions` production release chain；不改 Android / Flutter default base URL |
| IP 暴露和备案 / 合规误解风险 | 标注 IP staging 仅为受控测试环境；不作为公开上线入口；优先 allowlist / SSH tunnel |
| Codex 误把 plan 当 implementation 的风险 | 当前文档反复标注 prepackage only；未来 implementation 必须用户逐项授权后才执行 |
