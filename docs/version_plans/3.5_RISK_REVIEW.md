# 3.5 风险评审

更新时间：2026-04-11  
适用对象：Codex / Gemini-MCP / Claude-mcp / 顾问归档  
版本定位：`Astro Engine MVP / 参数化基础版`

---

## 1. 结论

`3.5` 的风险等级高于普通 UI 小修，但可控。  
关键原因是它会触及：

- 星盘响应 schema
- 服务端计算输出
- Flutter 本地绘制输入
- 引擎抽象与参数化能力

只要坚持“增量字段、兼容旧结构、先 schema 后 UI、先测试后扩展”，风险就可被压住。

---

## 2. 高风险点

### 2.1 canonical 真值与显示层混淆

风险：

- 把参数面板写成业务真值调参面板
- 把本地显示偏好误当作服务端 canonical 结果
- 让前端缓存覆盖 canonical 快照

回滚建议：

- 所有新增字段先标明角色
- 任何参数切换都不得回写真值层
- `astro_chart_preferences_v1` 继续只做显示层

### 2.2 响应结构漂移

风险：

- `chart_data` 被重构导致 Flutter builder 读不到旧字段
- `private_natal_chart` 缓存和新 schema 不兼容
- Python / Laravel / Flutter 三端对字段名理解不一致

回滚建议：

- 只做 additive change，不删旧字段
- 保留旧 key 兼容读取逻辑
- 测试中显式覆盖旧缓存与新响应并存

### 2.3 引擎抽象过早复杂化

风险：

- 把 MVP 一次性设计成过重的多引擎框架
- 过早引入无法验证的引擎分叉
- 让 adapter 层压住交付节奏

回滚建议：

- 第一阶段只补 schema / metadata / version tag
- 引擎切换先做可观测性，不做真实切换
- 保留 `kerykeion` 作为当前稳定计算路径

### 2.4 UI 参数化扩大面过快

风险：

- 参数面板过早开放太多开关
- 用户无法理解哪些是计算参数，哪些只是显示参数
- 页面过度拥挤，反而比 `3.4` 更难读

回滚建议：

- 参数面板先做最小集合
- 显示分组优先于深度参数
- Gemini-MCP 必须参与阶段验收

### 2.5 旧功能误伤

风险：

- 资料编辑、保存、重算链路被改坏
- 星盘页面无法刷新或空白
- 设置中心 / 版本中心入口回退
- 匹配、首聊、广场、治理系统被间接影响

回滚建议：

- 不改 `POST /api/v1/profile/basic` 行为
- 不改现有页面路由
- 每次修改后跑回归清单

## 3. 潜在回归点

1. `GET /api/v1/profile/astro/summary` 返回结构
2. `GET /api/v1/profile/astro/chart` 返回结构
3. `POST /api/v1/profile/basic` 保存后重算链路
4. Flutter 星盘 SVG builder 的读取字段
5. 星盘设置页的本地偏好缓存
6. `private_natal_chart` 旧缓存读取
7. `AstroCompatibilityService` 和 matcher 的 adapter 读取
8. 旧版本 / 旧设备 / 旧缓存兼容

---

## 4. 兼容性提示

### 4.1 对 API 的要求

- 新增 `engine_info`、`metadata` 必须向后兼容
- 不删除旧的 `chart_data`、`planets_data`、`houses_data`、`aspects_data`
- 不更改现有 JSON 路径语义

### 4.2 对 Flutter 的要求

- Flutter 仍然只消费局部字段
- 未识别的元信息必须忽略
- 不能让新参数面阻塞旧星盘页加载

### 4.3 对缓存的要求

- `private_natal_chart` 仍然可作为兜底缓存
- 旧缓存没有 `engine_info` 时必须能继续显示
- 新缓存写入时不能覆盖掉旧有稳定字段

---

## 5. 回滚建议

如果 `3.5` 第一阶段出现问题，优先回滚到：

1. 只保留原始 `chart_data` 输出
2. 移除新增的 `engine_info` / `metadata` 使用
3. 保留新增字段但不在 UI 读取
4. 恢复到 `3.4` 的稳定显示基线

如果参数面板或盘面绘制出现问题：

1. 先关掉新增参数入口
2. 保留原有星盘页
3. 保留设置中心旧入口
4. 只保留显示层增强，不碰计算层

---

## 6. 需要停止的信号

只要出现以下任一情况，必须暂停并回到计划阶段：

- 发现要改 `POST /api/v1/profile/basic` 的核心保存语义
- 发现要重写当前 canonical 引擎
- 发现 `chart_data` 结构必须大改才继续
- 发现参数面板需要一次性开放太多高级项
- 发现 UI 调整会把旧入口打断
- 发现固定样例对照无法重复
