# 2. 版本定位

## 2.1 本版本是什么

5.8 是 Me / Profile 个人解释层的最小产品化版本。

它把 5.6 冻结的解释层原则落到个人经营场景：

```text
用户进入我的 / 个人资料页时，可以看到一组克制、可忽略、display-only 的表达建议，帮助他理解自己当前资料如何被看见，并获得更适合慢约会的自我介绍和资料补充方向。
```

5.8 只做最小可验证闭环：

- 在 Me / Profile 用户经营点中新增一个个人解释层区块；
- 区块标题建议为 `我的表达说明` 或 `我的慢约会表达建议`；
- 包含一句个人表达摘要；
- 包含 2-4 个解释维度；
- 包含资料展示建议；
- 包含慢约会友好表达建议；
- 包含“可以补充什么”；
- 包含 AI 草稿 / 轻追问 disabled 或 coming soon 占位；
- 包含 display-only 免责声明；
- 保持用户可忽略，且原 Me / Profile 主流程不被替代。

## 2.2 本版本不是什么

5.8 不是：

- 重做个人资料页；
- 重做星盘图谱；
- 深度人格诊断；
- 强出生资料收集；
- 自动编辑资料；
- 自动写入 profile；
- 自动写入 astro；
- 真实 AI 服务接入；
- 付费报告；
- 真人咨询；
- 测测复刻；
- 商城 / 币 / 会员 / 装扮中心扩张；
- release / APK 发布任务。

# 3. 产品目标

## 3.1 用户目标

5.8 要帮助用户解决四个问题：

1. **我现在给人的第一印象是什么？**
   - 用一句克制摘要说明当前资料表达方向。

2. **我的资料是否支持慢约会？**
   - 从关系意图、表达清晰度、资料完整度、互动安全感等角度给出提示。

3. **我可以怎样更自然地介绍自己？**
   - 给出可复制但不自动写入的本地草稿建议。

4. **我还可以补充什么？**
   - 给出资料补充方向，不强制、不保存、不写真值。

## 3.2 产品目标

5.8 要完成从“个人资料展示”到“个人表达可理解”的最小升级。

它应增强 Me / Profile 的个人经营价值，但不能压过资料真值链、星盘真值链和原资料编辑主流程。

建议产品结构：

```text
Me / Profile
  ├─ 原有身份区 / 资料区 / 标签区
  ├─ 原有个人经营区 / AI 展示建议 / 草稿助手
  ├─ 新增：我的慢约会表达建议
  │   ├─ 一句个人表达摘要
  │   ├─ 2-4 个解释维度
  │   ├─ 资料展示建议
  │   ├─ 慢约会友好表达
  │   ├─ 可以补充什么
  │   ├─ AI 草稿 / 轻追问 disabled 或本地说明
  │   └─ display-only 免责声明
  └─ 原有编辑资料 / 查看状态 / 设置入口
```

## 3.3 设计语言

5.8 应使用 EliteSync 自有语言，不复制测测术语，也不把个人解释写成诊断。

推荐用词：

- `我的慢约会表达建议`
- `我的表达说明`
- `别人可能先看到什么`
- `资料展示建议`
- `慢约会友好表达`
- `可以补充什么`
- `先从真实、轻松的信息开始`
- `仅供自我表达参考，不会自动修改资料`

避免用词：

- `人格诊断`
- `命定人设`
- `吸引力改命`
- `低分用户`
- `必须补充`
- `系统已判定`
- `自动优化资料`
- `付费解锁报告`
- `大师建议`
- `真人咨询`
- `上传截图`

# 4. 功能范围

## 4.1 必做功能

5.8 必须完成以下最小功能：

### 4.1.1 个人解释卡

在 Me / Profile 的个人经营区域新增一张解释卡。

建议标题二选一：

```text
我的慢约会表达建议
```

或：

```text
我的表达说明
```

建议副标题：

```text
这些提示只帮助你整理表达方向，不会自动修改资料。
```

### 4.1.2 一句个人表达摘要

摘要应短、克制、非诊断化。

示例：

```text
你的资料更适合用真诚、轻松的方式开始，让对方先看到你的生活状态和关系节奏。
```

或：

```text
当前资料已经能表达一部分真实状态，可以再补充一点互动偏好，让开场更自然。
```

### 4.1.3 解释维度

首版建议维度为 4 个：

1. `真实感`
2. `表达清晰度`
3. `慢约会适配度`
4. `开场友好度`

每个维度建议包含：

- 维度标题；
- 一句解释；
- 可选轻标签；
- 不使用黑箱分数；
- 不使用羞辱性评价；
- 不写成系统判定。

### 4.1.4 资料展示建议

新增轻量区块：

```text
资料展示建议
```

示例建议：

- `可以补一句你希望怎样慢慢认识别人。`
- `可以把兴趣写得更具体，方便别人找到开场点。`
- `如果暂时不想透露太多，保留轻松但真实的表达即可。`

### 4.1.5 慢约会友好表达

新增轻量区块：

```text
慢约会友好表达
```

示例建议：

- `用“可以先聊聊共同兴趣”代替强推进关系。`
- `用具体生活片段代替空泛标签。`
- `让对方知道你愿意慢慢了解，而不是立刻给出承诺。`

### 4.1.6 可以补充什么

新增轻量区块：

```text
可以补充什么
```

示例建议：

- `补充一个最近喜欢的事物。`
- `补充一个你舒服的聊天节奏。`
- `补充一个不涉及隐私的生活状态。`

### 4.1.7 AI 草稿 / 轻追问占位

5.8 首版不接真实 AI。

可做两种方式之一：

1. disabled / coming soon：

```text
帮我整理一句表达 · 敬请期待
```

2. 本地说明 dialog：

```text
后续会支持把这些建议整理成可编辑草稿；当前不会自动修改资料。
```

不得调用网络、不得生成真实 AI、不得写资料、不得保存草稿到服务端。

### 4.1.8 display-only 免责声明

必须清楚可见：

```text
以上内容仅为自我表达参考，不会写入资料，不会改变星盘或匹配算法，也不会自动修改个人资料。
```

# 7. 实现原则

## 7.1 数据来源原则

5.8 不新增 API，只能使用当前 Me / Profile 页面已有数据上下文。

解释层数据可以来自：

- 已有用户基本资料；
- 已有 profile summary；
- 已有标签 / 关系目标 / 生活状态；
- 已有问卷或人格展示摘要；
- 已有星盘 / astro summary 展示字段；
- 本地 deterministic presentation helper；
- 静态 fallback copy。

如果当前 Me / Profile 数据不足，宁可展示克制 fallback，不得新增接口。

示例 fallback：

```text
当前资料还不够完整，可以先用一句轻松真实的自我介绍开始。
```

## 7.2 deterministic helper 原则

首版解释层可使用本地 deterministic helper，根据已有展示字段生成文案。

但 helper 必须：

- 只在 presentation 层；
- 不写入缓存；
- 不写入 server；
- 不改变 repository / API contract；
- 不伪装成 AI；
- 不输出绝对判断；
- 不自动进入编辑资料流程。

## 7.3 UI 原则

解释层必须：

- 可读；
- 克制；
- 不像工程面板；
- 不堆玄学术语；
- 不制造焦虑；
- 不羞辱用户；
- 不干扰原 Me / Profile 主行动；
- 不让失败态比成功态更重；
- 不出现强警告压过主流程。

建议视觉结构：

- 卡片式区块；
- 标题 + 简短说明；
- 3-4 个维度 chip / row；
- 资料展示建议；
- 慢约会友好表达；
- 可以补充什么；
- disabled / coming soon 草稿按钮；
- 免责声明小字。

## 7.4 可访问性与截图原则

必须确保关键文本能在 Android UI hierarchy 中被读取。

至少应能采集到：

- `我的慢约会表达建议` 或 `我的表达说明`
- `真实感`
- `表达清晰度`
- `慢约会适配度`
- `开场友好度`
