# EliteSync 2.9 Scope Boundary

更新时间：2026-04-08

## 1. 本版目标

2.9 的定位是 Beta 上线准备版，目标是把现有产品推进到可小规模接入真实用户、可监控、可回滚、可灰度、可验收的状态。

## 2. 本版明确包含

- 2.6.4 已建立的发布门禁收口
- 2.7 已完成的匹配主体验链路
- 2.8 已完成的信任安全与运营后台
- 首页、资料、匹配、消息、about 健康检查的稳定性收口
- 最小观测、最小灰度、最小运维文档

## 3. 本版明确不包含

- 不做大规模算法重写
- 不重做星盘 / 八字 / 紫微核心规则
- 不新增复杂 A/B 平台
- 不新增多媒体消息
- 不重构认证体系
- 不扩展数据库核心 schema

## 4. 不可触碰的保护面

- `POST /api/v1/profile/basic`
- `GET /api/v1/profile/astro/summary`
- `GET /api/v1/profile/astro/chart`
- `GET /api/v1/geo/places`
- `MatchPortalPage`
- `MatchResultPage`
- `MatchFeedbackPage`
- `ChatRoomPage`
- `AdminDashboardPage`
- `AdminModerationPage`
- `AdminVerificationPage`
- `AdminUsersPage`

## 5. 版本边界原则

- 前端缓存只能兜底，不能成为第二真源
- 任何高风险改动都必须先写验证与回滚
- 任何 Beta 发布都必须先过 smoke / regression / release gate
