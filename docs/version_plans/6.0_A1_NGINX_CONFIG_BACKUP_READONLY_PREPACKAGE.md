## 6. Redaction and Git safety policy

- backup report 只能记录文件名、路径、大小、hash、时间戳。
- 不记录 secret values。
- 不记录 `.env` 内容。
- 不记录 DB / Redis / OSS / LiveKit / Baidu / API keys。
- 不把 server backup 文件纳入 Git。
- 不把任何私有配置复制到 repo。
- Git 中只允许保存 redacted backup report / plan。
- 服务器本地 backup 目录不得被上传到 GitHub。

## 7. Future rollback plan outline

未来回滚思路：

- 回滚前先确认目标文件。
- 从 timestamped backup copy restore。
- restore 前备份当前文件。
- `nginx -t` 必须先通过。
- `nginx reload` 需要用户另行授权。
- reload 后检查服务。
- 若失败，恢复上一份配置。
- 所有命令必须记录但不得输出 secrets。

注意：本轮不执行 rollback。
