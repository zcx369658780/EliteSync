## 1. UI 目标

5.6A Match 关系解释层 UI 是 Match detail 内的轻量解释模块，用于帮助用户理解“为什么值得聊”以及如何低压开场。

本文件只描述 UI 方案，不进入代码实现。

核心目标：

- 在 Match detail 中补充可读的关系解释。
- 不替代现有匹配页。
- 不改变匹配算法。
- 不改变 Match 推荐结果。
- 不阻塞聊天入口、查看资料、返回等原流程。
- 不把解释层写成事实判断、预测、诊断或强承诺。
- 不引入真人咨询、付费报告、上传截图或服务转化入口。

## 2. 推荐页面位置

推荐放置位置：

- Match detail / 匹配解释区域下方。
- 与现有匹配理由、共同点、聊天入口保持同页承接。
- 放在核心资料和主行动之后，不抢主流程。

展示方式：

- 作为可折叠卡片。
- 默认展示短摘要。
- 不默认展开长内容。
- 展开后显示维度、建议 / 避免、轻追问占位和免责声明。

页面关系：

- 不替代现有匹配结果页。
- 不遮挡资料、聊天按钮、返回按钮。
- 不弹窗打断用户。
- 不要求用户必须阅读或操作。

## 3. 卡片结构

### 卡片标题

推荐标题：

- `关系理解`
- `为什么值得聊`

首版建议使用：`为什么值得聊`

原因：更贴近 Match 场景，不显得玄学，也比“关系理解”更行动导向。

### 一句关系摘要

位置：标题下方第一行。

作用：

- 用一句克制文案说明这次推荐的可聊方向。
- 默认态只展示这一句摘要和折叠箭头。

### 解释维度

首版展示 3 个维度：

- `共同点`
- `表达节奏`
- `慢约会适配度`

每个维度包含：

- 维度名。
- 一句说明。
- 可选轻量标签，不使用数字评分。

### 建议 / 避免

建议 / 避免作为一组并列小块：

- `可以尝试`
- `先避免`

`开场友好度` 不单独上屏，吸收到建议 / 避免区域。

### 轻追问入口占位

首版只做 disabled / coming soon / 本地占位说明。

建议表现：

- 小号文本按钮或禁用 chip。
- 文案如 `稍后可继续追问开场方式`。
- 不接真实 AI。
- 不跳转。
- 不生成消息。

### Display-only 免责声明

固定放在卡片底部。

推荐文案：

`仅用于帮助理解互动节奏和开场方向，不代表事实判断，也不会改写你的资料。`

## 4. 首版推荐维度

首版从 PRD 候选维度中选择 3 个。

### 共同点

用途：

- 帮用户看到可聊起点。
- 把匹配理由转成开场素材。

不做：

- 不列过多资料。
- 不暴露未授权信息。
- 不暗示系统掌握隐私。

### 表达节奏

用途：

- 帮用户判断开场应该轻松、具体、慢一点还是直接一点。
- 服务首聊压力降低。

不做：

- 不判断对方心理。
- 不做“对方一定喜欢这种方式”的断言。

### 慢约会适配度

用途：

- 对齐 EliteSync 的慢约会产品定位。
- 引导用户从低压、持续、尊重边界的方式开始。

不做：

- 不做强匹配结论。
- 不说“适合长期关系”这类强承诺。

### 暂不使用

暂不单独上屏：

- `开场友好度`：吸收到建议 / 避免区域。
- `关系安全感`：文案风险更高，后续再评估。
- `互动互补性`：容易写成性格判断，后续再评估。

暂不使用复杂评分、雷达图或强玄学维度。

## 5. UI 状态

### 默认态

显示：

- 卡片标题 `为什么值得聊`
- 一句关系摘要
- 折叠 / 展开 affordance
- 简短 display-only 提示可选露出一行

默认态不显示长解释，不挤压主流程。

### 展开态

显示：

- 关系摘要。
- 3 个解释维度。
- 建议 / 避免。
- 轻追问入口占位。
- 固定免责声明。

展开态仍保持短内容，不做长文报告。

### 折叠态

显示：

- 标题。
- 摘要。
- 展开入口。

折叠态不隐藏主流程，不影响聊天入口。

### 无可用解释态

当没有足够匹配上下文时显示：

- 标题：`为什么值得聊`
- 文案：`暂时没有足够信息生成关系提示，可以先从资料里的公开兴趣开始。`

不显示空白卡片。

### 加载态

第一版如为静态文案可不需要加载态。

未来如接 AI 或远端解释，可使用：

- 骨架行。
- `正在整理可聊方向...`

加载态不得阻塞 Match detail 主内容。

### 错误态

第一版如为静态文案可不需要错误态。

未来如接 AI 或远端解释，可使用：

- `关系提示暂时不可用，不影响继续查看匹配。`

错误态不能影响聊天、返回或查看资料。

### AI 入口禁用 / 占位态

首版轻追问入口必须是禁用或本地占位。

建议文案：

- `轻追问稍后开放`
- `之后可以继续问怎么开场`

不得：

- 接真实 AI。
- 跳转 AI 页面。
- 生成聊天消息。
- 自动发送。

## 6. 文案样例

### 关系摘要样例

1. `这次推荐更适合从共同兴趣开始，先轻松聊几句。`
2. `你们的公开资料里有一些可接上的话题，适合低压开场。`
3. `这组匹配更适合慢一点推进，先从具体细节建立熟悉感。`

### 维度说明样例

共同点：

- `你们都有可以展开的话题点，适合从一个具体兴趣切入。`

表达节奏：

- `更适合轻松、具体的开场，不需要一开始就聊得太深入。`

慢约会适配度：

- `这次推荐更适合慢慢建立熟悉感，先看对话是否自然。`

### 建议样例

1. `可以从对方资料里一个具体兴趣开始问。`
2. `先用轻松问题开场，给对方留出回应空间。`
3. `表达好奇就够了，不需要一开始说明太多意图。`

### 避免样例

1. `避免一上来追问隐私或关系期待。`
2. `避免过早下结论，比如“我们很适合”。`
3. `避免连续发很多问题，先等对方接住一个话题。`

### 轻追问入口样例

1. `稍后可继续追问：怎么开场更自然`
2. `稍后可继续追问：为什么推荐这个人`

### 固定免责声明

`仅用于帮助理解互动节奏和开场方向，不代表事实判断，也不会改写你的资料。`

## 7. 交互规则

必须支持：

- 用户可折叠。
- 用户可忽略。
- 用户不操作也能继续使用 Match detail。
- 用户仍可查看资料、返回、进入聊天。

不得出现：

- 弹窗打断。
- 自动跳转聊天。
- 自动生成消息。
- 自动发送消息。
- 强制完成。
- 强制展开。
- 高压转化按钮。

轻追问入口第一版：

- 只做 disabled / coming soon / 本地占位说明。
- 不接真实 AI。
- 不触发网络请求。
- 不写聊天草稿。
- 不跳转 AI 助手。

## 8. 视觉原则

整体风格：

- 延续 EliteSync 现代 UI 卡片风格。
- 轻量、克制、可扫描。
- 卡片应像解释辅助，而不是营销横幅。

视觉要求：

- 不做工程仪表盘。
- 不使用 debug / mock / internal 等主文案。
- 不做过重玄学视觉。
- 不做雷达图、星盘图、复杂分数盘。
- 不做高压转化按钮。
- 不出现真人咨询、上传截图、付费报告入口。
- 不使用大面积神秘主义装饰。

建议视觉层级：

- 标题小而清晰。
- 摘要一行到两行。
- 维度为 3 个轻量信息块。
- 建议 / 避免使用两个并列小块或上下分组。
- 免责声明使用较弱文本层级，但必须可见。

## 9. 数据与实现边界

本 UI 方案只描述未来如何展示。

第一版如后续实现，只能使用：

- 现有 Match detail 可见数据。
- 已授权展示资料。
- 静态本地展示文案。

不得：

- 新增 API。
- 新增数据库字段。
- 新增迁移。
- 写 profile。
- 改匹配算法。
- 写聊天消息。
- 自动发送。
- 接真人服务。
- 接付费报告。
- 引入上传截图。
- 修改版本号或发版链。

如果后续需要真实 AI 或远端解释，必须另开实现计划和 AI contract。
