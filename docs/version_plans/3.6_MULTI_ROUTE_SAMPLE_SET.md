# 3.6 多路线样例集

更新时间：2026-04-12  
适用范围：3.6 Stage 4 多样例对照  
报告性质：`derived-only / display-only`

## 1. 目的

本样例集用于把单一样例回归扩展为多样例回归。

它只回答：

1. 标准 / 古典 / 现代三条路线在不同样例上的可见点位与相位差异。
2. 哪些差异是路线解释层带来的，哪些仍然属于 canonical truth。
3. 哪些偏差是预期的路线差异，不能误判为 bug。

本样例集**不**改变服务端 canonical truth。

---

---

## 3. 多样例结论

- 标准路线保持为兼容基线。
- 古典路线在两个样例里都保持收敛型阅读，点位与相位数量更少。
- 现代路线在第二个样例中明显增加扩展相位，说明路线模板对解释层有实际影响。
- 样例集的所有差异均属于 derived-only / display-only。

---

## 4. 使用方式

1. 在回归时先看本样例集，确认三条路线的数量级是否稳定。
2. 再看 `3.6_ROUTE_PARITY_REPORT.md` 里的单样例解释，确认路线摘要是否足够清晰。
3. 最后对照 `3.6_ROUTE_CAPABILITY_MATRIX.md`，确认能力矩阵、样例集和解释层结论一致。

---

## 5. 结论

3.6 的多路线样例集已经满足回归和归档需要：

- 至少覆盖了一个兼容基线样例和一个现代扩展样例。
- 能够体现古典 / 现代在路线层的差异。
- 不会把路线解释层误写成 canonical truth。
