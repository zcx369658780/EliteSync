# EliteSync 3.0 执行计划（Beta 运营与匹配质量提升版）

更新时间：2026-04-08

当前基线版本：`0.02.09 / 209`

## 1. 版本目标

3.0 目标是在不破坏 2.9 工程底座的前提下，完成：

1. 新用户关键路径更清晰（知道下一步做什么）
2. 匹配解释更可理解、可行动
3. 首聊启动更顺畅
4. 指标与反馈闭环可学习
5. 弱网/冷启动/空态体验不退化

---

## 2. 阶段 0~5 总览

### 阶段 0：立项与边界冻结
- 目标：冻结范围、保护面、指标口径
- 交付：
  - `3.0_EXEC_PLAN.md`
  - `3.0_SCOPE_BOUNDARY.md`
  - `3.0_PROTECTED_SURFACES.md`
  - `3.0_METRICS_BASELINE.md`
- 验收门槛：Gemini 至少给出“有条件通过”，且无关键边界歧义

### 阶段 1：新用户关键路径与空态优化
- 目标：首页首屏 CTA、空态、冷启动、新手引导优化
- 交付：首页/匹配/消息关键提示和引导优化 + 最小埋点补齐 + 新手首周路径说明（`3.0_KEY_PATH_OPTIMIZATION.md`）
- 验收门槛：关键链路无 Sev1/Sev2 回归，新增引导可验证

### 阶段 2：匹配解释与首聊引导升级
- 目标：解释层三段式（结论/证据/行动）和首聊引导优化
- 交付：匹配解释模板升级、行动建议、首聊过渡文案
- 验收门槛：不改变底层匹配主分语义，仅做解释层/引导层升级

### 阶段 3：指标体系与反馈闭环落地
- 目标：形成可持续学习的 Beta 数据闭环
- 交付：指标口径、事件字典、反馈分类、周观察模板
- 验收门槛：指标可观测、可对比、可解释

### 阶段 4：体验一致性与质量收口
- 目标：主题、文案、状态一致性 + 回归收口
- 交付：3.0 回归清单、专项复测记录、一致性审计结果
- 验收门槛：弱网/冷启动/空态专项不过线不得进入阶段 5

### 阶段 5：总验收与发布判断
- 目标：判断 3.0 是否可作为 Beta 深化版结项
- 交付：`3.0_ACCEPTANCE_REPORT.md`、`3.0_FINAL_SUMMARY.md`、下一阶段建议
- 验收门槛：P0 项全部可验收，保护面未受损

## 5. 每批 Definition of Done（DoD）

每个批次必须同时满足：

1. 需求边界清晰（明确“做什么/不做什么”）
2. 变更文件清单完整，且无无关改动
3. 保护面检查项已执行并留痕
4. 新增体验项有对应观测事件或验证证据
5. Reviewer 给出风险等级（低/中/高）
6. Gemini 给出通过等级（通过/有条件通过/不通过）
7. 未验证项与剩余风险显式列出

---

## 6. 执行约束

1. 服务端 canonical 始终是唯一真源
2. 前端缓存只做体验兜底，不得成为第二真源
3. 未经过边界确认不得跨 P0/P1/P2 扩张
4. 任一批次触发保护面回归失败，必须先修复再继续下一批次
5. 未经用户明确同意，不推进对外发布动作

## 7. 阶段切换门槛（Phase Gate）

## 7.1 Phase 0 -> Phase 1 入场条件

必须全部满足：

1. `3.0_EXEC_PLAN.md`、`3.0_SCOPE_BOUNDARY.md`、`3.0_PROTECTED_SURFACES.md`、`3.0_METRICS_BASELINE.md` 完成并通过审查。
2. reviewer-codex 对四份文档给出无高风险阻断项结论。
3. spec-auditor-gemini 对四份文档给出至少“有条件通过”，且条件项有明确 owner 和补齐时点。
4. 3.0 事件字典版本 `3.0-metrics-v1` 已冻结，字段口径可执行。
5. 阶段 1 首批任务均已定义 Non-goal、验证集和回滚点。

## 7.2 阶段证据包要求（每批次）

每个批次提交必须附带：

1. 变更文件清单
2. 保护面检查结果
3. 回归/烟测证据链接或记录路径
4. 未验证项与剩余风险
5. 责任人（Claude/Codex/Gemini）与下一步动作
