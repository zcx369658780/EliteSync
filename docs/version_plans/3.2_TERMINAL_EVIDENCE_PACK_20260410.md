## 3. 出生地点搜索与落点

### 代码链路

- `app:dev:fill-synthetic-astro` 会从 `placePool` 里选真实地点，而不是写一个空壳城市名。
- `GeoController` 走服务端搜索链路：
  - `place/v2/suggestion`
  - `place/v2/search`
  - `geocoding/v3`
- 也就是说，产品层已经具备“城市 -> 候选 -> 落点”的真实链路。

## 6. Admin 统计口径澄清

### 口径说明

- `测试账号` = `account_type == test`
- `合成用户` = `is_synthetic == true`
- 两者可以重叠，不是互斥分类
- 顶部卡片统计的含义：
  - `总用户`：当前列表中的用户总数
  - `测试账号`：其中 `account_type == test` 的数量
  - `合成用户`：其中 `is_synthetic == true` 的数量

### 关键补丁

Admin 用户页已补一句解释文案，避免顾问误读：

> 测试账号按 `account_type = test` 统计；合成用户按 `is_synthetic = true` 统计。两者可以重叠，所以计数不会互斥。
