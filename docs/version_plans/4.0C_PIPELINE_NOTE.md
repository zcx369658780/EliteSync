# EliteSync 4.0C 队列与缓存闭环说明

日期：2026-04-18

## 已落地

- 媒体队列名已从 `config/media.php` 读取。
- 媒体缓存 TTL 已从 `config/media.php` 读取。
- `ProcessMediaAssetJob` 已形成最小 worker 链：读取媒体资产 -> 生成处理记录 -> 状态推进 -> 缓存回写。
- `MediaProcessingPipelineTest` 已验证 HTTP 触发 -> 队列处理 -> 数据库回写 -> 缓存落点。

## 当前约束

- 仍不接正式 OSS 直传。
- 仍不接分片、断点续传和签名 URL 正式协议。
- 仍不把缓存当成 canonical truth。

- `4.0D` 可以在此基础上接 Flutter 附件与上传体验骨架。
- `4.0E` 再补日志、监控、限流和交付文档。
