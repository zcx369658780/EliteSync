# EliteSync 5.1C Relationship Linkage Note

更新时间：2026-05-01

## 目标

5.1C 将匹配解释与聊天建议进一步联动，让用户能从完整解释页把共同点或行动建议转成可编辑聊天草稿。

本轮不改匹配算法，不改服务端 explanation composer，不改 profile canonical truth。

## 修改范围

- `apps/flutter_elitesync_module/lib/features/match/presentation/pages/match_detail_page.dart`

## 当前实现

### 1. 完整解释页行动卡

`MatchDetailPage` 新增 `解释到聊天` 行动卡：

- 显示为 `可编辑草稿`
- 明确提示“点击只写入聊天草稿，不会自动发送”
- 提供最多 3 个轻量建议

### 2. 建议来源

当前建议来源包括：

- `完整匹配解释`
- `模块解释 / 共同点`
- `行动建议`
- `关系节奏`
- `稍后再回队列`

来源只作为展示层说明，不写入服务端真值。

### 3. 草稿联动

点击建议后：

1. 读取现有 `matchResultProvider`，获取 `partnerId`。
2. 检查 `CacheKeys.chatDraftPrefix + peerId` 是否已有草稿。
3. 若已有不同草稿，先让用户选择 `取消 / 替换 / 追加`，避免静默覆盖。
4. 写入本地聊天草稿。
5. 跳转到 `${AppRouteNames.chatRoom}/$peerId`。
6. 不自动发送消息。

## 保留主链

- 仍以 `peerId` 进入 ChatRoom，不使用后端 `conversations.id`。
- 不改 `/api/v1/matches/*`。
- 不改 `/api/v1/messages`。
- 不新增 route contract。
- 不修改 Laravel 后端。

## 风险与观察项

- `MatchDetailEntity` 当前不携带 partnerId，本轮复用 `matchResultProvider` 获取 peerId。
- 若 `matchResultProvider` 无法返回 partnerId，会显示错误并停止跳转。
- 完整解释到聊天的草稿是本地缓存，不跨设备同步。
- 已有聊天草稿不会被静默覆盖；用户必须明确选择替换或追加。

## 验收口径

- Match Detail 页可见 `解释到聊天` 行动卡。
- 点击建议写入聊天草稿并进入对应 ChatRoom。
- 未拿到有效 partnerId 时不伪成功。
- 不改变匹配分数、解释算法或资料真值。
