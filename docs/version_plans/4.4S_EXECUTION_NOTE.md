## 本轮目标

修复图片 / 视频在聊天、状态、内容详情中的加载失败问题，重点收口媒体 URL 读链，不扩功能版本。

## 本轮结论

- 后端 `MediaAsset.public_url` 增加了访问层归一化逻辑，旧数据里残留的 `localhost` / `127.0.0.1` / 站内相对路径会按当前存储配置解析为真实可访问地址。
- Flutter 侧补了媒体 URL 规范化兜底，聊天气泡、视频预览、状态卡、内容详情页都会先把资源地址改写成当前 API 主机再渲染。
- 消息流、状态流、内容流都改为复用同一条媒体 URL 规范化链，不再各自处理。
