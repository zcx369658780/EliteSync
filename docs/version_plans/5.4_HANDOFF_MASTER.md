## 3. 5.4 Scope Freeze

Allowed in 5.4:

- Read-only governance overview surfaces.
- Health / version / release / account / media / notification / RTC status summaries.
- Cloud read-only DB access and audit runbook.
- Backup / restore / migration check runbook.
- Synthetic and smoke account governance.
- Observability evidence index.
- Regression checklist.
- This single handoff master.

Not allowed in 5.4:

- Reopening 5.0-5.3 product UI coverage as a main chain.
- Marketplace, coins, membership, gifts, live rooms, or multi-party RTC.
- Heavy admin platform or microservice split.
- Local production database writes.
- Release version bump unless explicitly requested.
- Backend contract, truth-chain, RTC contract, or release-chain rewrite.
- Modern UI regression.

## 4. Initial Implementation Direction

The first implementation slice should be a read-only governance overview entry, preferably under the existing Settings / Me operating area if that fits the current Flutter structure.

The slice should show real or clearly marked read-only status categories only. It must not fake successful health, version, media, notification, RTC, queue, or account states.
