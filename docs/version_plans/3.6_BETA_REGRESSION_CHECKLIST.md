# EliteSync 3.6 Beta 回归清单

更新时间：2026-04-12  
适用范围：3.6 Stage 4 / Stage 5 归档前回归  
版本状态：路线解释层与多样例集已落地

---

## 1. 回归目标

本清单用于验证 3.6 的路线解释层、多路线样例集和已知偏差归档是否稳定。

重点不是重新验证 canonical 算法，而是确认：

1. `route_mode` 只影响路线与解释上下文。
2. 路线差异报告是 derived-only / display-only。
3. 多路线样例集可以稳定复算。
4. 旧缓存、旧字段、旧页面入口仍然可读。

## 3. 手工回归

### 3.1 路线解释层

- 打开本命盘详情页，确认 compact 路线解释卡存在。
- 确认卡片只显示路线、工作台与固定样例，不显示任何真值改写提示。
- 切换路线后，路线摘要应同步变化，页面不应崩溃。

### 3.2 路线差异报告

- 打开设置页，确认路线差异报告能看到三条路线。
- 确认默认是折叠路线块，不会挤占首屏。
- 展开后能看见标准 / 古典 / 现代的摘要差异。

### 3.3 多路线样例集

- 确认 `baseline` 与 `dense-modern` 两份样例都能稳定复算。
- 确认基线样例与现代扩展样例的计数不同，但都属于 display-only 差异。
- 确认已知偏差文档里记录的差异，不会被误判为 bug。

### 3.4 旧入口

- 资料主页 / 资料编辑页 / 星盘页 / 设置中心 / 版本中心仍然可达。
- `chart_data`、`engine_info`、`metadata` 的旧字段仍然可读。
- `astro_chart_preferences_v1` 仍然只影响本地展示。

---

## 4. 回归通过标准

- 自动化测试全部通过。
- 手工检查未发现路线解释层把 display-only 写成 truth 的情况。
- 多路线样例集与路线能力矩阵数字一致。
- 旧页面入口和旧缓存读取没有回归。

---

## 5. 失败处理

若任一项失败：

1. 先确认是否是 sample fixture 或 route mapping 改动导致。
2. 再确认是否误把路线解释写进 canonical truth。
3. 最后再看是否只是文档 / 归档口径未同步。

---

## 6. 结论

3.6 的 Beta 回归应当把“路线解释层”当作最小可读层来验收，而不是把它当成新的算法真值层。
