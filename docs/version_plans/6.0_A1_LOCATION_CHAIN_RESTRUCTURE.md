## 4. Truth Layer / Display Layer / Explanation Layer

### Truth Layer

覆盖：

- `profile/basic`
- `users`
- `user_astro_profiles`
- `birth_time`
- `birth_place`
- `birth_lat`
- `birth_lng`
- astro canonical truth

规则：

- 不清空。
- 不覆盖。
- 不把 display location 写回 truth。
- 不把约会地点 / 搭子地点写回出生地或现居地 truth。
- 不把解释层字段写回 truth。
- 前端缓存只能兜底，不能抢服务端真值。

### Display Layer

覆盖：

- 用户侧 Profile / Me 展示。
- 用户侧 Astro / 星盘画像展示。
- Match / future Date Drop 展示。
- Buddy / future 搭子展示。

规则：

- 普通用户侧展示行政地名 / 城市 / 商圈。
- 不展示精确经纬度。
- 不展示 `position_signature`。
- 不展示 `location_source`。
- 不展示 engine / debug / hash / signature。
- 不展示火车站式误导地名。
- 不展示实时精确位置。

### Explanation Layer

覆盖：

- 玄学解释层。
- 关系解释。
- 搭子活动适配提示。
- Date Drop 推荐解释。

规则：

- derived-only / display-only。
- 不反写 truth。
- 不改匹配算法 contract。
- 不将玄学解释写成确定命运判断。
- 不将搭子反馈直接写成婚恋人格判断。

## 7. User-facing Display Rules

允许展示：

- 省 / 市 / 区县。
- 国家 / 城市。
- 商圈。
- 用户手动确认的地点名称。
- “用于计算，不对外展示精确坐标”一类说明。
- “仅显示城市 / 商圈级范围”一类隐私说明。

禁止展示：

- `lat` / `lng`。
- `birth_lat` / `birth_lng`。
- `longitude` / `latitude` raw number。
- `position_signature`。
- `location_source`。
- `location_shift_minutes`。
- `longitude_offset_minutes`。
- `equation_of_time_minutes`。
- engine version。
- `legacy_input`。
- hash / signature。
- debug。
- backend / API / contract / migration。
- 火车站名作为出生地默认展示。
- 实时精确位置。
- 附近 300 米之类高精度暗示。
- production / staging / v2 等工程词。

| 场景 | 禁止展示 | 推荐展示 |
|---|---|---|
| 出生地 | 南阳火车站 112.xx, 33.xx | 河南省南阳市 |
| 现居地 | xx.xxxxxx, yy.yyyyyy | 新加坡 / Tanjong Pagar |
| 约会地点 | 精确坐标 | 城市 / 商圈 / 用户确认地点 |
| 搭子地点 | 实时位置 / 住址 | 城市 / 商圈 / 公共场所附近 |
