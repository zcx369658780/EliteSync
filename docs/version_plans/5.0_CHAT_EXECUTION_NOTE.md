# EliteSync 5.0C Chat 执行记录

更新时间：2026-05-01

## 本轮目标

把 Chat 从“消息容器”整理成更明确的首聊 / 回聊 / 关系推进主路径，并形成可复验的 UI 证据。

## 已确认页面

### 1. 聊天列表默认态

已确认可见结构：

- 搜索会话、昵称、关键词
- `去匹配`
- `通知 73`
- `仅未读`
- `全部 / 未读 / 已读`
- 会话卡：`华霜魂`

### 2. 会话页

已确认可见结构：

- `刷新消息`
- `语音通话`
- `安全`
- `慢慢聊`
- `首聊 / 恢复建议`
- `从周末聊起`
- `问最近状态`
- `接住话题`
- `选择图片 / 视频`
- 图片消息
- 输入框

### 3. 返回路径

已确认：

- `BACK` 从会话页回到聊天列表；
- 聊天列表仍保留搜索、分类、未读、去匹配和通知入口；
- 返回栈可用。

## 关键发现

1. 聊天列表不是纯 inbox，而是关系推进入口层；
2. 会话页已经承担首聊、回聊、轻量 AI 续话、媒体联动和语音入口；
3. 当前输入区适合做轻按钮，不适合做重 AI 面板；
4. `语音通话` 在会话页中已经是自然入口，不需要重写 RTC 主链。

## 对后续实现的提醒

- 首聊 / 恢复建议要保持可回填、可跳过、可关闭；
- `去匹配` 和 `通知` 要保留联动；
- 媒体入口只做轻量承接；
- 不把聊天页扩成群聊、房间、重社区或商业导流页。

## 结论

5.0C Chat 已形成产品化补强起点，适合作为后续实现与验收的参考基线。
