## 3. Future Extremely Narrow Runtime Slice Proposal

以下三类 endpoint 仅是 future proposal。本文不新增任何 route / controller / DTO / Resource。

### 3.1 GET /api/v2/app/health proposal

要求：

- no auth
- no token
- no DB read
- no DB write
- no Redis write
- no storage access
- no external service call
- no production config mutation
- no release chain change
- 返回 static / framework-level health only
- 不暴露 secret / env secret / DB host / Redis host / token
- 不暴露用户数据
- 不影响 v1 health

必须明确：

- 本文不新增该 route。
- 本文只定义 future contract。
- 真正新增 route / controller 需要用户后续明确授权。

### 3.2 GET /api/v2/app/readiness proposal

要求：

- no auth
- no token
- no DB write
- no Redis write
- no storage write
- 不伪造 readiness pass
- 如果未来第一刀不允许真实 DB / Redis probe，则 readiness 必须返回 `not_checked` / `blocked` / `not_ready` 之类状态，而不是 `ok`
- queue worker 未常驻时不得返回 queue ok
- monitoring 未配置时不得返回 monitoring ok
- backup / restore / load test 未执行时不得返回 production_ready
- production health environment=local observation 必须继续携带

必须明确：

- readiness 不是 production readiness。
- readiness 不是 load test evidence。
- readiness 不是 monitoring setup evidence。
- readiness 不是 queue worker evidence。
- 本文不新增该 route。
- 真正新增 route / controller 需要用户后续明确授权。

### 3.3 GET /api/v2/contracts/location proposal

要求：

- no auth
- no token
- no user data
- no DB read
- no DB write
- no profile read
- no astro profile read
- no map SDK
- no real-time location
- static schema / semantics only
- 只返回四类位置语义：
  - birth_location
  - current_location
  - date_place
  - buddy_place
- 不返回 lat/lng
- 不返回 birth_lat / birth_lng
- 不返回 position_signature
- 不返回 location_source
- 不返回 engine/debug/hash/signature
- 不返回完整出生时间、手机号、真实地址、用户隐私数据
- 不写回 profile/basic
- 不写 user_astro_profiles
- 不影响 v1 profile/basic 或 astro/summary

必须明确：

- 本文不新增该 route。
- 本文只定义 future static contract。
- 真正新增 route / controller 需要用户后续明确授权。

### 6.2 v2 readiness response proposal

```json
{
  "success": true,
  "data": {
    "status": "not_ready",
    "checks": {
      "database": { "status": "not_checked" },
      "redis": { "status": "not_checked" },
      "queue": { "status": "worker_not_resident" },
      "monitoring": { "status": "not_configured" },
      "backup": { "status": "planning_only" }
    },
    "production_ready": false,
    "reason": "Readiness is intentionally conservative; DB/Redis/monitoring/backup/load-test evidence is not executed."
  },
  "meta": {
    "request_id": "future-request-id",
    "planning_note": "Do not treat this as production readiness evidence."
  }
}
```

Allowed readiness statuses include:

- `not_checked`
- `blocked`
- `not_allowed_in_first_slice`
- `not_ready`
- `worker_not_resident`
- `carry_over`
- `not_configured`
- `planning_only`
- `not_executed`

必须明确：

- readiness 不得谎称 ok。
- readiness 不得覆盖 production readiness。
- readiness 不得作为 load test / monitoring / backup evidence。
