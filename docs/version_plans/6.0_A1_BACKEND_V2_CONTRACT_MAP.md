# EliteSync 6.0-A1 Backend v2 Contract Map

日期：2026-05-13  
当前基线：`433a5115 docs: add 6.0-A1 framework runtime gate`  
Gate 状态：Framework / Runtime Support Gate `conditional pass accepted`  
阶段定位：v2 contract map 文档设计；不写 route / controller / DTO / migration。

## 1. v2 Contract 原则

- Contract-first：先冻结 contract map，再讨论 runtime skeleton。
- Parallel v2：v2 与 v1 并行，不替换、不删除 v1 route。
- Staged migration：先 staging / 受控环境验证，再考虑 production。
- Production 默认继续 v1：A1 不切 production 默认流量到 v2。
- Feature flag / base URL 可回滚：Flutter 与 backend base URL 必须支持回退到 v1。
- 不推倒重写：不以 v2 名义重写所有 controller、model、migration。
- 不删除 v1 route：所有 v1 保护接口继续保持原 path/method/response 兼容。
- Canonical truth 优先：profile / astro / location 真值不被 display layer、cache、frontend fallback 抢写。
- DB 后置：没有备份、回滚、staging 验证与用户确认，不新增表、不改 schema、不写 migration。

## 2. 后端目标栈

A1 v2 contract 使用以下目标口径：

- Laravel 12 作为 A1 优先目标。
- Laravel 13 作为后续评估项，不作为 A1 第一落地版本。
- Laravel 11 不作为 v2 商用目标版本。
- PHP 8.4 作为当前 runtime。
- MariaDB 10.11 LTS / MySQL-compatible 作为数据库口径。
- Redis 用于 cache / queue / session / rate limit。
- Nginx + PHP-FPM 作为 Web/runtime 承载。
- Queue worker / scheduler / systemd 或 Supervisor 进入后续 staging 方案。
- Monitoring / backup / load test / staging-production split 必须先设计再 runtime。

## 4. Location Contract

v2 必须拆分四类位置，禁止混写：

| 位置类型 | contract 名称 | 用途 | 真值来源 | 用户侧展示 | 经纬度策略 |
| --- | --- | --- | --- | --- | --- |
| 出生地 | `birth_location` | 星盘 / 八字 / 紫微真值 | `users.private_birth_*` 与 `user_astro_profiles.birth_*` | 行政地名，如省/市/区 | 内部可保存；普通用户侧不展示精确经纬度 |
| 现居地 | `current_location` | 生活圈 / 匹配范围 / 内测分布 | 未来 location context 或 profile display 字段 | 城市 / 行政区 / 商圈级 | 默认不展示精确经纬度 |
| 约会地点 | `date_place` | Date Drop 见面建议 | 未来用户确认地点 / 商圈 | 地点名 / 商圈 / 城市 | 可选内部坐标；不展示精确住址 |
| 搭子地点 | `buddy_place` | 学习 / 电影 / 吃饭 / 健身搭子 | 未来 buddy request / preference | 城市 / 商圈 / 用户确认地点 | 不默认请求实时定位，不展示精确住址 |

位置 contract 必须遵守：

- 出生地不得被现居地覆盖。
- 现居地不得反写出生地。
- Date Drop 约会地点不得写入 profile truth。
- 搭子地点不得写入 profile truth。
- 用户侧不得展示 `lat/lng`、`position_signature`、`location_source`、engine/debug 字段。
- Flutter 本地缓存只能兜底展示，不能抢服务端真值。

## 6. v1 -> v2 Migration Strategy

| 域 | 策略 | 阻断条件 |
| --- | --- | --- |
| app health | parallel v2 | blocked until Laravel 12 staging branch |
| app version check | keep v1 | release chain P0 protected |
| auth/user | keep v1, future wrap | blocked until auth smoke and session target |
| profile/basic write | keep v1 | P0 canonical truth; blocked until DB plan |
| profile/basic read | wrap v1 into v2 display | blocked until profile smoke |
| astro summary/chart | keep v1, v2 display wrapper later | P0 canonical truth; chart cache side effect |
| location context | parallel v2 proposal | blocked until DB plan and privacy review |
| Date Drop match | migrate later | A2+, blocked until contract and DB plan |
| buddy | migrate later | A3+, blocked until DB plan |
| messages | keep v1 | read-receipt side effect; blocked until smoke |
| media | keep v1 | blocked until queue/storage/OSS plan |
| notifications | keep v1 | route payload compatibility |
| RTC | keep v1 | LiveKit / heartbeat P0 protected |
| questionnaire | keep v1, future wrapper | match input compatibility |
| feedback | parallel v2 | can be first low-risk v2 write only after DB plan |
| operations/readiness | parallel v2 | blocked until staging config |

Legend：

- `keep v1`：继续使用 v1，不新增 v2 实现。
- `wrap v1`：v2 只读包装 v1 输出，但不改变 v1。
- `parallel v2`：新增 v2 contract，与 v1 并行。
- `migrate later`：后续版本迁移。
- `blocked until DB plan`：必须先出 DB/migration/backup/rollback/staging plan。
- `blocked until staging`：必须先有 staging 环境。
- `blocked until Laravel 12 upgrade`：必须先完成 Laravel 12 staging 方案。

7. 补齐 v1 P0 smoke 测试。
8. 明确任何 DB/migration 均不进入本阶段。
