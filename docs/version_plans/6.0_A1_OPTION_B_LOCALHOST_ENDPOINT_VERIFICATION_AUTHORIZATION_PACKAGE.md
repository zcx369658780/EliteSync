## 2. Future endpoint verification goal

If separately authorized by the user, a future issue may:

1. run local repo startup checks;
2. SSH to the authorized EliteSync server;
3. confirm symlink still exists and target is correct;
4. optionally run `sudo nginx -t` as a read-only syntax sanity check;
5. perform localhost-only requests from inside the server to the bound localhost staging port;
6. verify only a narrow public/read-only endpoint surface;
7. write a redacted endpoint verification execution report;
8. stop without production request and without changing config.

## 3. Future allowed endpoint scope

Future verification may only request localhost-only endpoints from the server itself, for example:

```text
http://127.0.0.1:8088/api/v2/app/health
http://127.0.0.1:8088/api/v2/app/readiness
http://127.0.0.1:8088/api/v2/contracts/location
```

If the actual route prefix or path differs, the future execution must stop and report mismatch instead of guessing or requesting broad paths.

The request method must be GET only.

## 4. Future allowed command shape

Future execution may allow commands shaped like:

```bash
curl -sS -i --max-time 10 http://127.0.0.1:8088/api/v2/app/health
curl -sS -i --max-time 10 http://127.0.0.1:8088/api/v2/app/readiness
curl -sS -i --max-time 10 http://127.0.0.1:8088/api/v2/contracts/location
```

The future execution report should summarize status code and non-secret JSON shape only. Do not paste long bodies if they contain sensitive or unexpected values.

## 5. Future forbidden scope

Even if localhost endpoint verification is authorized, still forbid:

- public staging request from user's PC;
- production request;
- `curl slowdate.top`;
- `curl 101.133.161.203`;
- SSH tunnel creation;
- authenticated endpoint requests;
- POST / PUT / PATCH / DELETE;
- write smoke;
- real user data request;
- `.env` / secret reads;
- artisan / PHPUnit / composer / migration;
- DB / Redis / queue / storage changes;
- Nginx config modification;
- Nginx reload / restart;
- Flutter base URL switch;
- release chain modification;
- claiming production verified;
- claiming A1 final acceptance complete.

## 6. Stop conditions before future endpoint requests

Future endpoint verification must stop if:

- local repo worktree is dirty;
- SSH target mismatch;
- symlink missing;
- symlink target mismatch;
- Nginx is not loaded / endpoint port appears unavailable before request planning;
- route path is uncertain;
- any command would read `.env`;
- any command would output secrets;
- any request would touch production or public host;
- any request would require auth or user data;
- any response contains unexpected secret-like content;
- any step would require config change or reload;
- any step would require DB / migration / write smoke.

- next gate.
