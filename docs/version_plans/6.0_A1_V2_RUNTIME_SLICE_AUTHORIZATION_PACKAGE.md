## 5. Future Response Contract

### /api/v2/app/health

只能返回 static / framework-level health：

- no auth
- no token
- no DB read
- no DB write
- no user data
- no storage access
- no external service call
- no secret
- no DB host / Redis host / token exposure

Suggested future fields:

- `success`
- `data.status`
- `data.api_version`
- `data.service`
- `data.timestamp`
- `data.runtime_stage`
- `meta.request_id`, if available without side effects

### /api/v2/app/readiness

不得谎称 `production_ready`。不得把 queue / DB / Redis / monitoring / backup / load test 写成 `ok`。

未检查项必须写为 one of:

- `not_checked`
- `not_ready`
- `blocked`
- `carry_over`
- `not_allowed_in_first_slice`

Minimum semantics:

- `data.production_ready = false`
- database / Redis checks default to `not_checked` or `not_allowed_in_first_slice` unless user later authorizes probe.
- queue defaults to `not_ready` / `worker_not_resident` / `carry_over` while worker is not resident.
- monitoring defaults to `not_configured` / `carry_over` while setup evidence is missing.
- backup / restore / load test default to `planning_only` / `not_executed`.
- production health `environment=local` observation must continue to be carried and must not be hidden by v2 readiness.

### /api/v2/contracts/location

只能返回四类位置静态语义：

- `birth_location`
- `current_location`
- `date_place`
- `buddy_place`

不得返回：

- `lat` / `lng`
- `birth_lat` / `birth_lng`
- `position_signature`
- `location_source`
- `engine` / `debug` / `hash` / `signature`
- `full_birth_time`
- `phone`
- `exact_address`
- `real_time_location`
- any user data

该 endpoint 只能说明 static contract semantics，不读取 profile，不读取 astro profile，不读取 DB，不写回 `profile/basic`，不写 `user_astro_profiles`，不影响 v1 `profile/basic` 或 `astro/summary`。
