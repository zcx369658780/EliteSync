# EliteSync 5.6A Match 关系解释层实现计划书

更新时间：2026-05-11

状态：implementation plan draft

当前口径：go to planning, do not implement yet

本文件只用于规划 5.6A Match 关系解释层的最小 UI 实现候选，不进入 Flutter / Laravel / 数据库 / API / 发版链实现。

## 1. 版本定位

5.6A 是 EliteSync 玄学能力二次产品化后的第一个最小 UI 实现候选，目标是在 Match detail 的现有体验中增加一个轻量、可忽略、display-only 的关系解释卡片。

本切片不替代现有匹配结果页，不改变匹配推荐算法，不改变聊天入口、资料入口、返回路径或任何主流程。它只在用户已经看到匹配对象时，提供一层低压、克制的关系理解辅助。

当前仍是 implementation plan，不进入 runtime implementation。后续是否进入代码实现，需要用户再次确认。

实现目标：

- 添加一个 display-only 关系解释卡片。
- 解释“为什么值得聊”，而不是给出强预测、强诊断或关系承诺。
- 保持 derived-only / display-only，不反写任何用户画像、星盘真值、匹配算法输入或聊天内容。

## 4. 功能切片

第一版功能切片为一个可折叠 display-only 卡片。

卡片建议：

- 标题：为什么值得聊
- 默认展示：
  - 一句关系摘要。
  - 3 个解释维度摘要。
  - display-only 免责声明。
- 展开后展示：
  - 共同点。
  - 表达节奏。
  - 慢约会适配度。
  - 建议 / 避免。
  - 轻追问 disabled / coming soon 占位。

首版解释维度：

1. 共同点
   - 解释双方资料中已授权展示信息所体现的轻量共同兴趣、生活节奏或沟通偏好。

2. 表达节奏
   - 解释双方在开场、回复、推进关系时可能更适合的低压节奏。

3. 慢约会适配度
   - 解释双方是否适合从轻量话题、低承诺互动、逐步熟悉开始。

建议 / 避免：

- 建议用于给用户低压开场和互动提醒。
- 避免用于提示不宜过快推进、过度判断、强行套用标签或做强关系承诺。
- “开场友好度”并入建议 / 避免区域，不作为独立维度上屏。

轻追问入口：

- 第一版只做 disabled / coming soon / 本地占位说明。
- 不接真实 AI。
- 不生成聊天草稿。
- 不自动发送。

免责声明：

- 必须明确这是辅助理解，不代表匹配算法结论、关系预测或用户画像真值。

## 5. 数据策略

第一版优先采用静态本地展示文案，或仅使用当前 Match detail 已经可见、用户已授权展示的数据。

明确禁止：

- 新增 API。
- 新增数据库字段。
- 缓存到服务端。
- 写入 `profile/basic`。
- 写入 `profile/astro/summary`。
- 写入 `profile/astro/chart`。
- 写入 `user_astro_profiles`。
- 改写匹配算法输入。
- 写入聊天消息。
- 自动生成并发送消息。
- 使用未授权聊天内容。
- 强制收集出生资料。

如果后续实现发现没有可用的安全展示数据，第一版应退回为静态本地展示文案或无可用解释态，而不是扩大数据范围。

## 6. AI 策略

第一版不接真实 AI。

轻追问入口只允许作为：

- disabled 状态。
- coming soon 状态。
- 本地占位说明。

明确禁止：

- 触发网络请求。
- 调用 AI 服务。
- 生成聊天草稿。
- 自动发送消息。
- 把追问结果持久化为用户画像。
- 把追问结果反写匹配算法或资料真值。

后续若要接入 AI，必须单独进入新一轮 PRD / contract / safety review，不得在 5.6A 第一版中顺手接入。

## 8. UI 验收计划

UI 验收应至少覆盖：

1. Match detail 原流程可用
   - 进入匹配详情页正常。
   - 查看资料入口仍可用。
   - Chat 入口仍可用。
   - 返回路径仍可用。

2. 关系解释卡片可见但不阻塞
   - 卡片位于 Match detail 的匹配解释区域或其下方。
   - 不遮挡主 CTA。
   - 不阻塞聊天、查看资料、返回等原流程。

3. 可折叠 / 可忽略
   - 默认展示短摘要。
   - 可展开查看维度详情。
   - 可折叠回短摘要。
   - 用户不需要完成任何操作即可继续使用原页面。

4. 文案合规
   - 不做强预测。
   - 不做心理诊断。
   - 不使用“命中注定”“必然适合”等绝对表述。
   - 不复制测测术语体系。
   - 不使用强玄学诊断口吻。

5. Display-only 提示可见
   - 明确说明该解释仅用于辅助理解。
   - 不代表关系预测、匹配算法结论或用户画像真值。

6. 禁止入口不存在
   - 不出现上传截图入口。
   - 不出现付费报告入口。
   - 不出现真人服务、咨询、达人、在线服务入口。

7. 轻追问不接真实 AI
   - 轻追问入口为 disabled / coming soon / 本地占位。
   - 点击后不触发网络请求。
   - 不生成聊天草稿。
   - 不自动发送。

## 10. 风险与停止条件

出现以下任一情况，后续实现必须停止并汇报，不得继续扩大范围：

- Match detail 实际路径不清。
- 存在多个 Match detail 实现且无法确认当前用户路径使用哪一个。
- 实现必须新增 API。
- 实现必须新增数据库字段或迁移。
- 实现必须改 Laravel。
- 实现必须触碰 `profile/basic`、`profile/astro/summary`、`profile/astro/chart` 或 `user_astro_profiles`。
- 实现会改变 Match 推荐算法输入。
- 实现会改变 Chat 消息发送链。
- 实现会写入聊天消息或自动生成聊天草稿。
- 实现需要真实 AI 请求。
- 实现需要上传、付费、真人服务或咨询入口。
- UI 需要大重构才能放下该卡片。
- 新卡片会阻塞 Match detail 原流程。

若发生以上情况，应回到 PRD / UI 方案重新收窄，而不是在实现中临时改产品范围。
