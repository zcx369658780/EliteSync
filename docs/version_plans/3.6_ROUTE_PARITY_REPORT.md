# 3.6 路线差异报告

更新时间：2026-04-12  
适用范围：3.6 Stage 3 路线解释层 / 固定样例对照  
报告性质：`derived-only / display-only`

## 1. 报告目的

本报告用于说明 3.6 的 `route_mode`、工作台预设与固定样例对照关系。

它只回答三件事：

1. 当前路线是什么。
2. 这条路线相对标准路线会展示什么差异。
3. 哪些内容仍然是 canonical truth，哪些只是路线解释层。

本报告**不**是新的算法真源，也**不**替代服务端 canonical 结果。

---

## 2. 固定样例

- 样例盘：`EliteSync`
- 生日资料：`1996-08-18 10:30:00 +08:00`
- 说明：本报告始终基于同一份固定样例，确保差异可重复复核

---

## 4. 与标准路线的差异

### 4.1 古典路线 vs 标准路线

- 点位差异：`-9`
- 相位差异：`-5`
- 差异摘要：
  - 收紧到核心点位
  - 仅保留主相位和最紧凑的阅读口径

### 4.2 现代路线 vs 标准路线

- 点位差异：`0`
- 相位差异：`+1`
- 差异摘要：
  - 保留标准路线点位规模
  - 继续放宽少量扩展相位

---

## 5. 真值边界

- `route_mode` 只影响展示与解释上下文。
- 路线模板只决定推荐工作台默认值。
- `chart_data` 仍然是本地 SVG 绘制的核心输入。
- `profile/basic` 的出生资料、经纬度、时间真值不因路线切换而改变。
- 路线解释层和 parity report 都不回写 canonical truth。

---

## 6. 结论

3.6 的路线解释层与 parity report 已经可以作为移动端阅读辅助层使用：

- 用户可以先看路线，再看工作台，再看盘面。
- 解释层只说明差异，不宣称“哪条路线更真”。
- 固定样例对照保持可重复复核，便于后续 3.7 / 引擎升级继续扩展。
