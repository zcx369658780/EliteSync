# EliteSync 5.0C Chat 聊天页关系推进说明

更新时间：2026-05-01

## 结论

5.0C 的 Chat 目标不是把聊天页做成复杂消息系统，而是把当前会话列表与会话页收拢成更明确的**首聊 / 回聊 / 关系推进主路径**。

当前基线在模拟器上可见：

- 会话列表；
- 搜索会话、昵称、关键词；
- `去匹配`；
- `通知 73`；
- `仅未读`；
- `全部 / 未读 / 已读` 分类；
- 唯一会话卡 `华霜魂`；
- 会话页的首聊建议、恢复建议、AI 轻按钮、语音通话与附件入口。

这说明 Chat 已经不是纯 inbox，而是具备了首聊辅助、关系摘要、回复建议、语音联动的雏形。

## 5.0C 的产品化方向

本轮只确认最小产品化方向，不扩写实现：

1. 会话列表要能服务首聊 / 回聊；
2. 会话页要能展示关系摘要；
3. 输入区要保留轻量 AI 续话能力；
4. 语音入口要保持自然联动；
5. 不重写 RTC 主链。

## 当前基线观察

### 会话列表

当前可见结构：

- 搜索框：`搜索会话、昵称、关键词`
- `去匹配`
- `通知 73`
- `仅未读`
- `全部 / 未读 / 已读`
- 会话卡：`华霜魂`

这说明会话列表不只是聊天 inbox，还承担了轻关系推进入口。

### 会话页

当前会话页可见：

- 标题：`华霜魂`
- `刷新消息`
- `语音通话`
- `安全`
- `慢慢聊`
- `首聊 / 恢复建议`
- `从周末聊起`
- `问最近状态`
- `接住话题`
- `选择图片 / 视频`
- 图片消息区域
- 输入框：`输入首句问候或你想继续聊的话题…`

这说明会话页已具备首聊辅助、回复引导和轻量媒体联动。

## 低风险分支观察

### 首聊 / 恢复建议

当前建议项已经能作为对话起点，不需要单独重页面。

建议后续保持：

- 推荐问候模板；
- 续话建议；
- 可直接回填输入框；
- 可跳过、可关闭。

### 语音入口

`语音通话` 已在会话页自然可见，后续应保持与关系推进语义一致，不回头重写 RTC 主链。

### 附件入口

`选择图片 / 视频` 与 `添加图片或视频` 已提供轻量媒体联动壳层，后续只做 additive 承接。

## 非目标

本轮不做：

- 重写 RTC 主链
- 引入重社区聊天
- 复杂群聊/房间化能力
- 支付、会员、账号安全、删除账号等深区

## 对后续实现的约束

- Chat 的增强只应做 additive 承接；
- 不破坏 UI protected surfaces；
- 不把会话列表做成纯消息列表；
- 不把输入区做成重 AI 面板；
- 不把语音入口从关系推进语义中剥离。

## 交接口径

本文件作为 5.0C Chat 关系推进最小增强的起点说明。
后续若补充实现，应继续配套：

- Chat UI evidence index
- 回归检查点
- 必要时的 handoff master 收口
