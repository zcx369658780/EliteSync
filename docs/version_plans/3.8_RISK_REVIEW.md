# EliteSync 3.8 风险评审

更新时间：2026-04-16  
适用范围：3.8 stage 0

## 1. 结论

3.8 属于中高风险版本。原因不是新增页面本身，而是它要把参数工作台、单项解读和校准样例正式纳入产品主线；一旦边界写错，很容易把 derived explanation 误写成 canonical truth，或者把移动端做成不可读的参数后台。

## 2. 高风险清单

### 2.1 canonical truth 污染风险

- 参数解读、星体解读、相位解读、主星 / 飞宫解释如果字段命名不清，容易被误认为新真值。
- 校准报告如果没有明确区分“引擎差异 / 参数口径差异 / 显示层差异 / 真 bug”，容易被后续误读。

### 2.2 旧入口回归风险

- 3.7 的高级预览页、路线工作台、设置中心、版本中心都必须保留。
- 若专业工作台首版首屏过重，可能压缩旧入口可见性。

### 2.3 状态与缓存风险

- 参数工作台一旦进入 provider / cache，没有 fallback 就容易污染旧读取逻辑。
- 新增解释块必须与现有 natal truth 隔离。

### 2.4 UI 过载风险

- 参数面板、摘要区、解读入口、样例校准、盘面主图如果全塞首屏，移动端会失控。

### 2.5 精度归因风险

- 参考样例如果只做视觉比较，不做字段归因，会把引擎差异与口径差异混在一起。

### 2.6 流程留痕风险

- 如果 Claude / Gemini 留痕缺失，3.8 不能直接进入正式归档。

## 3. 潜在回归点

- `POST /api/v1/profile/basic`
- `GET /api/v1/profile/astro/summary`
- `GET /api/v1/profile/astro/chart`
- `POST /api/v1/profile/astro`
- `POST /api/v1/profile/astro/render`
- 资料页 / 画像页 / 星盘页
- 参数工作台 / 解读页 / 校准页
- `chart_data`、`engine_info`、`metadata`
- `route_mode` / `pair_context` / `time_context` / `return_context` / `comparison_context`

## 4. 回滚建议

1. 保留 feature flag，默认关闭 3.8 专业工作台入口。
2. 新增解释字段必须可忽略，不影响旧客户端。
3. 新增校准页必须可回退到 3.7 参考图与旧样例矩阵。
4. 保留旧入口、旧读取、旧版本中心和旧高级演示页。

## 5. 兼容性提示

- 所有参数与解读结果都必须标记为 derived / display / advanced context。
- 绝不能把主星、宫主星、飞宫、相位表包装成 canonical truth。
- 参数扩展必须有默认 fallback。

## 6. 阶段门禁

- stage 0：先冻范围，再定风险。
- stage 1：先 Claude 审 schema / truth boundary / calibration structure。
- stage 2 / 3：再 Gemini 审布局 / 可读性 / 交互。
- stage 4：样例校准与偏差归档。
- stage 5：最终回归、验收与归档。
