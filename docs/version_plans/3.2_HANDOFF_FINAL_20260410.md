# EliteSync 3.2 最终交接稿

更新时间：2026-04-10
状态：核心完成，等待终验补证后归档

## 一句话结论
3.2 已完成核心底座、关键转化改造与最终风险修复，并已通过 Flutter acceptance smoke、Flutter analyzer、host APK 编译与后端定向回归；当前主线功能已完成，但仍需以终验补证包补齐 synthetic / 玄学样本 / Admin 口径证据后再正式归档。

## 已完成内容
### 1. 测试账号体系底座
- `normal / test / synthetic` 三层账号分层已落地
- synthetic batch 审计字段已加到 `users`
- `app:dev:sync-account-tier` 与 `app:dev:synthetic-users` 已支持批次、重建、清理、禁用
- synthetic 默认 `exclude_from_metrics = true`

### 2. Admin 客观治理视图
- Admin 用户列表已能按 `accountType / batch / accountStatus / visibilityScope / isSynthetic` 筛选
- 列表卡片展示 batch / version / scope / status / match / square / metrics / seed
- 后台治理视图已从“主观标签”升级为“客观事实面板”

### 3. 玄学 correctness / readability
- 八字页底部重复“精度”误导行已去掉
- 星盘 / 紫微 / 八字 / 盘面设置页统一把 `precision` 语义收口为“版本 / 可信度”
- 目标是减少验收歧义，不改 canonical 计算逻辑

### 4. 广场轻分层
- 状态广场与首页预览均展示作者层级 / 可见层级标签
- `公开层 / 广场层 / 私密层` 已明确
- 状态项已能体现 author layer / visibility tier

### 5. 匹配与首聊转化
- 匹配结果页已提供可点击开场草稿卡
- 聊天页顶部破冰建议可点击并预填输入框
- `partnerNickname` 已透传，用于更自然的会话跳转标题
- 目标是把“匹配结果”更顺地推进到“真正开聊”

### 6. 最终风险修复
- 匹配结果本地缓存已补齐 `matchId / partnerId`，避免远端失败后首聊入口丢失真值
- Admin 封禁动作已同步 `is_match_eligible / is_square_visible / exclude_from_metrics`
- 状态广场作者层级改为优先读服务端 `is_synthetic` 真源，避免前端误判 synthetic / test 标签
