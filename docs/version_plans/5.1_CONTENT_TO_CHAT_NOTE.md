# EliteSync 5.1D Content To Chat Note

更新时间：2026-05-01

## 目标

5.1D 将状态 / 动态卡片从单纯浏览与点赞，增强为低压关系转化入口。

本轮只做 Flutter 前端 additive 改动，不新增数据库字段，不新增评论 / 关注 / 社区系统，不伪造聊天会话。

## 修改范围

- `apps/flutter_elitesync_module/lib/features/status/presentation/widgets/status_post_card.dart`
- `apps/flutter_elitesync_module/lib/features/status/presentation/pages/status_square_page.dart`
- `apps/flutter_elitesync_module/lib/features/status/presentation/pages/status_author_page.dart`

## 当前实现

### 1. 状态卡低压 CTA

`StatusPostCard` 新增可选回调：

- `继续了解`
- `看看资料`
- `稍后再聊`
- `低压私聊`

这些按钮只有在上层传入回调时才显示，不影响 compact / 旧调用方。

### 2. 状态广场回流

`StatusSquarePage` 对状态卡接入低压动作：

- `继续了解`：打开作者公开页；
- `看看资料`：打开作者公开页；
- `稍后再聊`：仅给出本页本地提示，不写服务端队列；
- `低压私聊`：先提示“只有匹配关系成立后才会进入真实聊天”，再打开作者公开页。

### 3. 作者页低压回流

`StatusAuthorPage` 新增 `低压回流` 卡：

- `回到消息`：进入现有消息列表；
- `稍后再聊`：本地提示，不写库；
- 作者最近状态中的 compact 卡可触发轻量 `继续了解` 和 `稍后再聊`。

## 保留主链

- 不改 `StatusRemoteDataSource`；
- 不改 `StatusPostController`；
- 不新增 `status_posts` / `status_post_likes` schema；
- 不绕过消息主链的 matched + released + not blocked 规则；
- 不创建本地待发送消息；
- 不伪造私聊成功态。

## 风险与观察项

- 当前 `稍后再聊` 是本地 UI 提示，不跨设备、不写库。
- 当前 `低压私聊` 不直接进入 ChatRoom，因为状态流对象不一定具备匹配会话权限。
- 如果后续要从状态卡直接进入 ChatRoom，必须先验证 peerId、匹配关系、拉黑关系和消息 API 权限。

## 验收口径

- 状态卡可见低压动作。
- `低压私聊` 不自动发送、不伪造会话。
- 未匹配对象不会被强行打开 ChatRoom。
- 作者页提供回消息列表的自然回流。
- 举报 / 喜欢 / 删除旧动作仍可用。
