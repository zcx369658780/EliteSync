# EliteSync 3.9C 边界说明收口

更新时间：2026-04-18  
阶段：3.9C-1 边界说明收口  
目标：只整理版本边界说明，不改运行时契约

## 1. 当前结论

3.9C 只做说明层收口，明确 `canonical / derived / display-only / advanced-context` 的边界，不引入新的运行时契约或 schema 变更。

## 2. 需要明确的边界

### 2.1 星盘真值边界

- `profile/basic`
- `astro/summary`
- `astro/chart`
- `canonical truth`

以上仍是服务端真值与基础画像链路，不可被高级时法、pair_chart 或 timing 输出覆盖。

### 2.2 展示与关联边界

- `pair_chart`
- `timing`
- `advanced-context`
- `display-only`

以上只能作为展示层、关联层、解释层和版本边界说明存在，不得回写真值层。

### 2.3 本地偏好边界

- `astro_chart_preferences_v1`
- `astroChartWorkbenchPrefs`
- `astroChartRoutePrefs`

以上均属于本地展示偏好或工作台口径，不可改写 canonical truth。

## 3. 收口方式

- 文档中统一使用“展示与关联说明”作为对外口径
- 若需要保留工程术语，只保留在开发者说明或归档摘要中
- 对用户可见页面继续降噪，但不得删掉必要边界提示

## 4. 不做的事

- 不升级算法主线
- 不修改接口契约
- 不新增运行时 feature flag 行为
- 不改数据库 schema
- 不把边界说明写成新功能说明

## 5. 下一步建议

- 若后续需要继续做 3.9C-2，只允许单独开运行时契约任务
- 当前收尾阶段先保留这份边界说明，作为 3.x 与 4.x 的过渡口径
