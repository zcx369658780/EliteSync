# EliteSync 2.6.4 执行计划

更新时间：2026-04-06

## 1. 版本定位

2.6.4 是稳定性与发布门禁收口版本。目标不是继续扩功能，而是把 2.6.3 / 2.6.3a 已完成的玄学主链路、资料链路与页面状态收紧为可重复回归、可验证回滚的发布版本。

## 2. 本版目标

- 稳定资料保存与画像重算链路
- 稳定 `summary` / `chart` 分流
- 稳定玄学四页与设置页
- 固化保护面、回归清单、回滚方案、发布烟测

## 3. 必做范围

- `EditProfilePage` 保存链路稳定
- 百度地点搜索候选与经纬度写入稳定
- `POST /api/v1/profile/basic` 重算触发稳定
- `GET /api/v1/profile/astro/summary` 稳定
- `GET /api/v1/profile/astro/chart` 稳定
- `AstroOverviewPage` / `AstroBaziPage` / `AstroNatalChartPage` / `AstroZiweiPage` 可用
- `AstroChartSettingsPage` 展示偏好与渲染行为对齐
- 发布门禁、回滚、烟测文档补齐

## 4. 明确不做

- 不新增大型玄学模块
- 不改数据库 schema
- 不重构登录体系
- 不把前端缓存升级为真源
- 不大改聊天 / 匹配主流程

## 5. 保护面

- 数据库初始化 / migration / 登录态恢复
- 地点搜索 / 坐标 / 真太阳时
- profile/basic / summary / chart
- 玄学四页路由
- 设置页本地偏好

## 6. 实施顺序

1. 确认现有高风险 surfaces
2. 补齐 / 收敛门禁文档
3. 检查资料链路与页面状态
4. 修复阻塞级问题
5. 做 smoke / regression / acceptance
6. 收尾回滚与发布材料

## 7. 验收标准

- 保存后画像回读一致
- summary / chart 分流稳定
- 四大页面无阻塞级 bug
- 回归清单可重复执行
- 回滚方案可实际操作
