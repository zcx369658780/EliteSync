# EliteSync 未来版本开发路线图 5.6+ 玄学能力二次产品化

更新时间：2026-05-11

状态：roadmap draft

当前口径：本文件只定义 5.6+ 路线图，不进入 runtime implementation。

## 1. 路线图总览

5.0-5.5 作为已完成历史链路，不重写。

5.6 起进入：

```text
玄学能力二次产品化与校准线
```

参考关系：

- Soul：继续作为社交主链参考，服务 Discover、Match、Chat、Me、关系推进、低压开场和个人经营。
- 测测：作为玄学解释层参考，服务入口层、上下文层、解释对象层、解释 / 建议层的抽象。

不再继续测测 Stage 6。现有 Stage 1-5 已足够支撑 5.6+ 路线修订。

## 2. 测测研究抽象

测测 Stage 1-5 被抽象为五层：

1. 入口层。
2. 上下文层。
3. 解释对象层。
4. 解释 / 建议层。
5. 追问 / 转化层。

EliteSync 重点吸收前四层：

- 入口层：能力入口嵌入 Match、Me、Chat 等主链。
- 上下文层：单人解释依赖个人资料上下文，关系解释依赖双人匹配上下文。
- 解释对象层：把星盘、测试、合盘等转译为关系理解、表达偏好、匹配理由。
- 解释 / 建议层：用短句、维度、建议 / 避免组织可读结果。

第五层必须克制：

- 可保留轻追问。
- 不默认接真人服务。
- 不上传截图作为主路径。
- 不做付费报告。
- 不把入口可见写成完整链路已验证。

## 3. 全路线保护面

所有玄学解释、合盘解释、AI 追问都必须保持：

```text
derived-only / display-only / explanation layer
```

全路线不得反写：

```text
profile/basic
profile/astro/summary
profile/astro/chart
user_astro_profiles
```

全路线不得改变：

- canonical truth。
- profile contract。
- astro contract。
- match contract。
- release contract。
- API contract。
- 数据库 schema。
- migration。

全路线不得引入：

- 真人玄学咨询市场。
- 付费报告商店。
- 上传截图主路径。
- 强出生资料收集。
- 娱乐化测试主路径。
- 测试结果反写画像。
- 大规模社区讨论。
- 测测术语体系复制。

- 不改版本号。
- 不改 release chain。

## 5. 5.7：Match 关系解释层最小产品化版

### 5.1 版本定位

5.7 是 5.6+ 路线批准后的第一个 runtime 候选版本。

### 5.2 核心目标

在 Match detail 中增加克制的关系解释层，帮助用户理解：

- 为什么推荐这个人。
- 可以从哪里开场。
- 哪些表达方式适合。
- 哪些表达方式应避免。

### 5.3 功能边界

建议结构：

- 标题：`为什么值得聊`。
- 一句关系摘要。
- 2-4 个解释维度。
- 建议 / 避免。
- 轻追问 disabled / coming soon 或受控占位。
- display-only 免责声明。

建议首版维度：

- 共同点。
- 表达节奏。
- 慢约会适配度。

### 5.4 不做

- 不接真实 AI。
- 不新增 API。
- 不新增 DB 字段。
- 不改匹配算法。
- 不写 profile。
- 不写聊天消息。
- 不自动发送。
- 不上传截图。
- 不接真人服务。
- 不做付费报告。

## 6. 5.8：Me 个人解释层与表达建议版

### 6.1 版本定位

5.8 将解释层从 Match 扩展到 Me / Profile，但因为靠近资料真值，必须更保守。

### 6.2 核心目标

帮助用户理解：

- 我如何被看见。
- 我的表达风格是否清晰。
- 资料是否支持慢约会。
- 自我介绍和展示是否需要优化。

### 6.3 功能边界

建议结构：

- 我的表达风格。
- 资料展示建议。
- 慢约会友好表达。
- 可以补充什么。
- display-only 免责声明。

### 6.4 不做

- 不重做星盘图谱。
- 不强收出生资料。
- 不深度人格诊断。
- 不自动修改资料。
- 不写 profile。
- 不改 `user_astro_profiles`。
- 不接付费报告。
- 不接真人服务。

## 7. 5.9：Chat 轻追问与低压开场版

### 7.1 版本定位

5.9 将 Match / Me 的解释层轻量衔接到 Chat，但不做自动代聊。

### 7.2 核心目标

帮助用户：

- 在首聊时找到低压开场。
- 冷场时换一种说法。
- 从关系解释转化为可编辑草稿。
- 不让 AI 代替用户表达。

### 7.3 功能边界

建议结构：

- 从共同点开场。
- 换个更自然的说法。
- 低压问候建议。
- 不要太急的提醒。
- 可编辑草稿候选。

### 7.4 不做

- 不自动发送。
- 不自动代聊。
- 不读取未授权私密聊天内容。
- 不上传截图。
- 不对对方心理做确定判断。
- 不把 AI 输出写回用户画像。
- 不接真人咨询。

## 8. 5.10：解释层治理与用户控制版

### 8.1 版本定位

5.10 是治理与统一控制版本，用于收口 5.7-5.9 的解释层能力。

### 8.2 核心目标

- 给用户统一控制权。
- 固化免责声明。
- 统一解释层开关。
- 统一回归清单。
- 防止解释层膨胀成伪真值、伪诊断、伪 AI 决策。

### 8.3 功能边界

建议结构：

- 关系解释提示开关。
- AI 辅助建议开关。
- 解释层免责声明。
- 关闭后仍可使用主流程。
- 不写入资料说明。

### 8.4 不做

- 不做重后台。
- 不做付费。
- 不做真人服务。
- 不做算法训练。
- 不做数据库大重构。
- 不做上传分析。
- 不做社区讨论系统。

## 9. 对既有 5.x 路线的修订关系

本路线图继承 2026-05-01 的 5.x 方向重排版：

- 4.9 仍是稳定门禁基线。
- 5.0-5.5 仍作为历史链路保留。
- Discover / Chat / Me 仍是社交主链。
- Soul 仍是社交主链参考。
- AI 必须嵌入主链，不作为孤立平台。
- 所有新增能力必须 additive，不破坏旧主链。

本路线图新增：

- 测测作为玄学解释层参考。
- 5.6+ 进入玄学能力二次产品化与校准线。
- Match / Me / Chat 分阶段承接解释层。
- 5.10 做统一治理与用户控制。

## 10. 测测结论吸收 / 不吸收总表

| 类别 | 吸收 | 不吸收 |
| --- | --- | --- |
| Stage 1 入口矩阵 | 入口层、价值解释、上下文入口 | 商业入口密集混排 |
| Stage 2 AI问 / 在线 | AI解释层和真人服务层分开 | 真人市场、优惠券、连麦、付费咨询 |
| Stage 3 测试 | 轻互动到解释的节奏参考 | 娱乐化测试主路径、测试结果反写画像 |
| Stage 4 星盘 | 单人档案上下文、图谱先行、分层解释 | 未验证 DeepSeek / AI问 / 在线桥接 |
| Stage 5 合盘 | 双人上下文、关系短句、维度、建议 / 避免 | 上传截图主路径、真实双人结果页外推、真人服务桥接 |
