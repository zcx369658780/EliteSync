# EliteSync 4.5B-D 执行记录

更新时间：2026-04-21

## 本轮范围

本轮在 `4.5A` 边界冻结之后，继续补了 4.5 的中段能力：

- `4.5B` 后端通知写入、读取与状态闭环
- `4.5C` Flutter 通知中心、入口与页面回流
- `4.5D` 通知噪声控制、入口一致性与回归保护

## 已完成内容

### 后端通知闭环

- `notifications` 作为站内通知真值表继续稳定使用
- `NotificationService` 支持：
  - 创建通知
  - 读取列表
  - 未读计数
  - 单条已读
  - 全部已读
  - 短窗口重复通知去重
- `NotificationController` 支持：
  - `/api/v1/notifications`
  - `/api/v1/notifications/unread-count`
  - `/api/v1/notifications/{id}/read`
  - `/api/v1/notifications/read-all`

### 事件映射

已接入最小可用通知来源：

- 新消息
- 动态点赞
- 匹配成功 / 喜欢

### Flutter 通知中心

- 新增通知中心页
- 首页 / 消息页 / 匹配页均接入通知入口
- 通知卡支持点击回流到：
  - 聊天页
  - 动态作者页
  - 匹配页
  - 问卷历史页
  - 内容详情页
  - 设置页

### 最小可观测性

- `notification_center_opened`
- `notification_item_opened`
- `notification_all_read`

### 回归保护

- 后端通知 feature tests 通过
- Flutter 通知中心页测试通过
- telemetry 测试通过

## 保留边界

- 不做厂商推送平台化
- 不做 RTC / 通话
- 不做在线状态系统
- 不重构消息 / 动态 / 匹配主链
- 不把通知中心做成运营后台

## 下一步建议

- 如果继续推进 `4.5`，优先补 `4.5E` 的截图证据与正式归档材料
- 如需更细颗粒度通知策略，再在后续版本扩写，不回头重写当前主链
