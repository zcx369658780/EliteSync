## 关键调用链

### Chat 首聊 / 回聊 / 冷场恢复

调用链：

`消息入口 -> ConversationListPage -> conversationListProvider -> ChatRemoteDataSource.getConversations -> ChatRoomPage -> chatRoomMessagesProvider -> GET /api/v1/messages?peer_id=... -> sendMessageUseCase -> POST /api/v1/messages -> MessageController -> ConversationDomainService -> NotificationService`

5.1 允许增强：

- 把现有破冰建议扩成 `继续聊 / 稍后再回 / 冷场后重新接起` 的轻队列；
- 标注建议来源为匹配解释、共同点、最近互动、状态内容；
- 仅填充输入框或草稿，不自动发送真实消息。

### 匹配解释到聊天建议

调用链：

`MatchPortalPage / MatchResultPage / MatchDetailPage -> matchResultProvider / matchDetailProvider -> MatchRemoteDataSource -> 匹配解释 / 共同点 -> chat draft -> ChatRoomPage`

5.1 允许增强：

- 在匹配结果 / 完整解释中提供 `继续聊天 / 带一句去聊 / 稍后再聊`；
- 复用现有 `partnerId`、`MatchDetailDto` 和聊天草稿链；
- 不改匹配算法、分数、explanation composer 或 profile canonical truth。

### 状态 / 动态到私聊

调用链：

`StatusSquarePage -> statusPostsProvider -> StatusRemoteDataSource.fetchStatusPosts -> StatusPostCard -> StatusAuthorPage -> status_like notification -> NotificationCenterPage`

5.1 允许增强：

- 在状态卡与作者页增加低压 CTA；
- 文案优先采用 `继续了解 / 看看资料 / 低压私聊 / 稍后再聊`；
- 未确认已匹配和未拉黑前，不得伪造私聊成功。

### 通知中心回流

调用链：

`NotificationCenterPage -> notificationListProvider -> NotificationRemoteDataSource.fetchNotifications -> GET /api/v1/notifications -> route_name / route_args -> GoRouter`

5.1 允许增强：

- 分类、文案、动作语义产品化；
- 保持未知 `kind` 与未知 `route_name` 可降级展示；
- 新字段只能 additive，不得改变现有 route args 语义。

### 语音联动

调用链：

`ChatRoomPage 语音按钮 -> RtcPermissionService -> RtcRemoteDataSource.createCall -> POST /api/v1/rtc/calls -> RtcSessionService -> rtc_call_invite notification -> Rtc pages / LiveKit / heartbeat`

5.1 允许增强：

- 增加语音邀请前关系节奏提示；
- 未接通 / 结束后提供回聊建议；
- 不改 RTC 状态机、LiveKit join-info、heartbeat 或多人 / 视频能力。

## 关键风险闸门

后续实现前必须重点验证：

1. `conversationId` 在 Flutter 聊天链路中实际被当成 `peerUserId` 使用，不是后端 `conversations.id`。
2. Message 通知 payload 中的 `conversation_id` / `peer_user_id` 方向存在风险；接收方打开通知前必须验证不会跳到自己或无效会话。
3. 通知中心依赖 `route_name` 字符串协议，新增动作前必须确认后端 payload 与前端分发一致。
4. `StatusRemoteDataSource.fetchStatusPosts` 有失败后回退 mock 的风险，5.1 验收不能只看 UI 成功态。
5. `lastKnownProfile`、profile snapshot 和本地缓存只能兜底，不能参与覆盖资料、出生地、坐标、八字、紫微、星盘等真值。
6. 状态到私聊必须先尊重消息主链的 matched + released + not blocked 规则。
7. RTC 文档曾记录 10 秒失联，但实现中存在 30 秒超时口径；5.1 验收不得按错误口径误判回归。
8. 版本链当前已有 dirty diff；5.1 若不发包，必须明确仍维持 `0.04.09 / 40900`。
