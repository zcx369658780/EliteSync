# 《恋爱交友 APP 玄学匹配算法 2.0 技术方案》
> 面向 Codex 的可落地实现稿  
> 适用技术栈：Android + Laravel / Java/Kotlin / Python 微服务均可  
> 目标：把当前“规则能跑”的 1.0，升级为“结果更稳、解释更强、可持续迭代”的 2.0

---

## 0. 先说结论

2.0 的核心原则不是“把玄学做得更玄”，而是：

1. **底层计算必须标准化**  
   历法、行星位置、宫位、时区、节气，不再手写近似公式。
2. **解释层和计算层分离**  
   开源库负责“算位置/算盘面”，你们自己负责“关系评分/文案解释”。
3. **展示层和排序层分离**  
   用户前台可以看八字/星盘/MBTI，后台真实排序要用可验证的行为指标校准。
4. **先做可上线版本，再做深度版本**  
   第一阶段先修正明显错误，第二阶段再升级命理/占星深度，第三阶段再做业务校准。

# 1. 2.0 总体架构

## 1.1 分层设计

### A. 基础计算层（Deterministic Engine）
负责“算出正确的基础数据”，不负责恋爱结论。

包含：
- 八字排盘
- 节气/农历/干支/生肖
- 西方占星行星位置
- 宫位
- 相位
- 比较盘 / 组合盘基础数据
- MBTI-like / Big Five 人格原始分数

### B. 特征工程层（Feature Layer）
把基础盘面转换成可打分的特征。

包含：
- 八字：日主强弱、十神、用忌、夫妻宫、合冲刑害
- 西占：行星落星座、落宫、相位、orb、个人行星互动
- 关系盘：synastry / composite 的结构化特征
- 人格：Big Five / MBTI-like 维度差、互补、风险项
- 用户行为：活跃度、回复率、偏好一致度、地域/年龄/目标等

### C. 评分层（Scoring Layer）
负责输出：
- compatibility_score
- confidence
- risks
- evidence_tags
- explanation_modules

### D. 业务校准层（Calibration / Re-rank）
负责把“玄学分”转成更适合真实配对的排序分。

---

## 1.2 推荐总公式

不要再直接：
`personality * 0.5 + mbti * 0.15 + astro * 0.35`

改为四段式：

```text
base_rule_score
= hard_filter_pass
+ behavior_score
+ personality_score
+ astro_score

display_score
= 用户展示分（可解释、可读）

rank_score
= 对业务目标做校准后的真实排序分

final_exposure_score
= rank_score × fairness × freshness × anti-repeat
````

### 建议解释

* **display_score**：给用户看的“你们多适合”
* **rank_score**：给系统排序用的“更可能聊起来/聊得久”
* 两者可以高度相关，但不要强制相等

## 2.1.5 验收标准

* 随机抽样 200 个生日，年柱/月柱/日柱/时柱与标准排盘结果一致
* 春节前后、立春前后、时辰边界样本人工复核通过率 > 99%
* 不再使用 `(year - 4) % 12` 生成生肖
* 匹配引擎可输出结构化证据，不再使用“八字字符串相似度”作为主逻辑

# 5. 推进阶段与验收标准

---

## Phase 0：纠错与底层标准化（1~2 周）

### 目标

修复当前明显错误，统一基础计算口径。

### 任务

* 删除属相公历年硬编码
* 八字/生肖统一由历法库派生
* 星盘底层切换到 Swiss Ephemeris
* 统一时区、经纬度、出生时间缺失策略
* 重新定义“合盘”术语

### 交付

* `BaziEngine`
* `WesternEphemerisEngine`
* `ProfileNormalizationService`

### 验收标准

* 边界案例（春节/立春/时辰边界）全部通过
* 太阳/月亮/ASC/宫位与标准盘面一致
* 前后端使用相同的出生参数规范

---

## Phase 1：2.0 基础评分版（2~4 周）

### 目标

让 2.0 能上线替换 1.0 的核心评分。

### 任务

* 八字：结构分 + 关系分
* 西占：本命适配 + 比较盘基础版
* MBTI：停止 3 题强推，改为人格维度过渡版
* 总分拆成 display_score / rank_score

### 交付

* `BaziFeatureExtractor`
* `BaziCompatibilityScorer`
* `NatalFeatureExtractor`
* `SynastryScorer`
* `PersonalityScoreServiceV2`
* `CompatibilityAggregatorV2`

### 验收标准

* 每个模块可输出 evidence_tags
* 不再依赖八字字符串相似度
* 不再把 MBTI 作为核心主分驱动
* 任意一对用户的评分可解释

---

## Phase 2：关系占星 + 深度八字版（3~5 周）

### 目标

提升差异化，做出“专业感”。

### 任务

* 八字：加入十神、用忌、夫妻宫、运势同步
* 西占：加入组合盘、行星落宫、土星绑定、金火吸引、沟通指标
* 输出关系优势 / 风险 / 时间提示

### 交付

* `TenGodsAnalyzer`
* `UsefulElementEngine`
* `SpousePalaceScorer`
* `CompositeChartBuilder`
* `HouseOverlayScorer`
* `RelationshipNarrativeGenerator`

### 验收标准

* 同一对用户可生成结构化关系报告
* 能输出“为什么吸引 / 为什么容易冲突 / 为什么关系能维持”
* 文案解释与分数证据一致

---

## Phase 3：业务校准版（2~4 周）

### 目标

让“高分用户”真的更容易产生业务转化。
