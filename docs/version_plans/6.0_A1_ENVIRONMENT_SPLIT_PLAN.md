| migration boundary | 任何 migration 仍 forbidden；environment split plan 不创建、不运行 migration。 |

必须遵守：

- staging dry-run 不得直接使用 production DB。
- production DB 不得连接、不写入、不 restore，除非用户明确授权。
- staging DB clone / restore drill 需要用户确认。
- 没有 backup evidence，不允许 migration / DB write / production config change。
- 任何 DB host / username / password 不得写入文档。
- 任何 migration 仍 forbidden。

## 7. Redis / Cache / Session / Queue Split Rules

承接 env/cache/session plan：

- `APP_CACHE_STORE` 作为 cache 主口径。
- `APP_SESSION_DRIVER` 作为 session 主口径。
- `APP_SESSION_CONNECTION` 作为 session connection 主口径。
- `QUEUE_CONNECTION` 作为 queue 主口径。
- `CACHE_STORE` / `SESSION_DRIVER` 只作为迁移期兼容观察。
- cache staging / production 目标为 Redis。
- session A1 目标建议 Redis session，但 production v1 可短期保留 database session 过渡。
- queue 目标为 Redis queue。
- queue worker 未常驻仍是 P1 carry-over。

Split rules：

| 配置域 | staging 要求 | production 要求 | skeleton 前判定 |
|---|---|---|---|
| Redis DB / keyspace | staging-specific DB or prefix。 | production keyspace 不复用。 | must confirm |
| cache prefix | staging-specific。 | production-specific。 | must confirm |
| session prefix | staging-specific。 | production-specific。 | must confirm |
| queue prefix | staging-specific。 | production-specific。 | carry-over for readonly only |
| `APP_CACHE_STORE` | target `redis` after confirmation。 | target Redis for commercial baseline。 | plan complete, not executed |
| `APP_SESSION_DRIVER` | Redis recommended, database allowed only with reason。 | production v1 may keep database temporarily。 | must confirm before auth/session migration |
| `APP_SESSION_CONNECTION` | staging Redis connection if Redis session。 | production-specific connection。 | must confirm |
| `QUEUE_CONNECTION` | target Redis。 | target Redis, worker plan required。 | carry-over until async/write |

必须写清：

- staging Redis prefix 不得与 production 混用。
- cache prefix、session prefix、queue prefix 必须分环境。
- Redis DB / keyspace 不得复用 production。
- readiness 不得伪造 queue ok。
- queue worker 未常驻时，纯只读 skeleton 可以 carry-over；任何 async / write 链路前必须解决。
