## 2. Future server inventory 目标

未来若用户另行授权，server inventory 只读盘点目标是：

- 确认当前服务器是否具备创建 IP-based controlled staging 的基础条件。
- 只读确认 Nginx / PHP-FPM / Laravel / DB / Redis / storage / 端口 / systemd / supervisor / cron / firewall / security group 相关边界。
- 确认是否存在可安全新增 staging 端口或 SSH tunnel 的空间。
- 确认哪些内容必须备份后才能修改。
- 不修改任何服务器文件。
- 不读取 secrets。
- 不读取真实 `.env` / `.env.*` 内容。
- 不执行部署。
- 不执行 migration。
- 不请求 production / staging URL。

## 3. Future allowed readonly inventory scope

未来若用户明确授权，只允许执行只读命令类别。

允许盘点：

- OS / hostname / uptime / disk / memory。
- Nginx 是否安装、版本、enabled site 文件名清单。
- Nginx 当前监听端口清单。
- PHP / PHP-FPM 版本。
- Laravel 项目目录是否存在。
- 当前 backend deploy 目录结构。
- 当前 storage / logs 目录结构，不读取敏感日志正文。
- DB 服务类型和版本，只读查看服务状态，不登录 DB。
- Redis 服务状态，只读查看服务进程 / 端口，不读取 keys。
- open ports / listening ports。
- systemd service names。
- supervisor / queue worker service names。
- cron / scheduler 是否存在，只读列出任务名称时必须脱敏。
- security group 仅由用户在云控制台确认，不由 Codex 自动修改。
- release / deploy scripts 的服务器侧路径是否存在，不执行脚本。

必须强调：

- 只列文件名、服务名、端口号、版本号、状态。
- 不 cat 真实 `.env`。
- 不输出 secret-like values。
- 不输出 DB password / Redis password / OSS key / LiveKit secret / API key / token / cookie / session。
- 不读取用户数据。
- 不执行写操作。

## 5. Future command allowlist draft

只能写命令草案，不得执行。

未来可能允许的只读命令类型与脱敏规则：

| Category | Draft command examples | Redaction / output rule |
|---|---|---|
| Identity | `whoami`, `hostname`, `uname -a`, `uptime` | Host/user output may be redacted if requested |
| Resource | `df -h`, `free -h` | No file content output |
| Nginx version | `nginx -v` | Version only |
| Nginx config metadata | `nginx -T` only after confirming no secrets | Prefer file names only; do not output full config if secrets may appear |
| Nginx files | `ls -la /etc/nginx/sites-enabled /etc/nginx/sites-available` | File names only |
| Ports | `ss -tulpn` | Port / process summary only |
| Services | `systemctl status nginx php-fpm redis mariadb --no-pager` | Status only; redact paths / env if sensitive |
| PHP | `php -v` | Version only |
| Redis | `redis-server --version` | Version only; no `redis-cli keys` |
| DB | `mysql --version` or `mariadb --version` | Version only; no DB login |
| Deploy dirs | `ls <backend_deploy_dir>` | Directory names only |
| Directory shape | `find <backend_deploy_dir> -maxdepth 2 -type d` | Limited-depth structure only |
| Supervisor | `supervisorctl status` only if installed and safe | Service names/status only |
| Cron | `crontab -l` only if user explicitly allows | Must redact commands, paths, tokens, URLs if sensitive |

必须同时禁止的命令：

- `cat .env`
- `grep secret/password/key/token .env`
- mysql login / redis-cli keys
- `composer install` / `composer update`
- `php artisan migrate`
- `php artisan config:clear` / `php artisan config:cache`
- nginx reload / restart
- systemctl restart
- ufw / firewall modify
- chmod / chown / rm / mv / cp writes
- deploy scripts
- release scripts

## 6. Redaction policy

未来 server inventory 输出必须 redacted：

- 所有 IP 可保留公网 IP 或脱敏为 first-two-octets，按用户要求。
- 所有路径可保留非敏感路径。
- 所有 secret-like values 必须替换为 `[REDACTED]`。
- 不输出 `.env` 内容。
- 不输出 DB host / username / password。
- 不输出 Redis password。
- 不输出 OSS / LiveKit / Baidu / API keys。
- 不输出 cookie / session / token。
- 不输出用户数据。
- 不输出完整 Nginx config，除非确认不含 secrets。

## 7. Stop conditions

未来只读 inventory 遇到以下情况必须停止：

- SSH 登录信息不明确。
- 连接到非预期服务器。
- 发现当前用户权限过高且命令可能越权。
- 任何命令可能读取 secrets。
- 任何命令可能修改系统。
- 输出包含 secret-like values。
- 需要读取真实 `.env` / `.env.*`。
- 需要登录 DB / Redis。
- 需要 Nginx reload / restart。
- 需要改安全组 / firewall。
- production 服务异常。
- worktree 不 clean。
- 用户授权范围不明确。
