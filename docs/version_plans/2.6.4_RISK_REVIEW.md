# EliteSync 2.6.4 风险评审

更新时间：2026-04-06

## 1. 核心风险

### 1.1 资料链路漂移

- 出生地选择、经纬度、保存 payload、回读时机一旦不一致，会导致保存成功但页面不一致。

### 1.2 summary / chart 互相拖累

- 星盘 SVG 链路不稳定时，不能拖垮八字 / 紫微 / 总览。

### 1.3 登录态与缓存

- 前端缓存若抢真值，容易出现旧数据覆盖新数据。

### 1.4 发布门禁缺失

- 若没有统一 smoke / rollback / acceptance 文档，后续回归会依赖记忆。

## 2. 高风险 surfaces

- 数据库初始化 / migration
- `POST /api/v1/profile/basic`
- `GET /api/v1/profile/astro/summary`
- `GET /api/v1/profile/astro/chart`
- `GET /api/v1/geo/places`
- `EditProfilePage`
- `AstroOverviewPage`
- `AstroBaziPage`
- `AstroNatalChartPage`
- `AstroZiweiPage`
- `AstroProfilePage`

## 3. 回滚点

- 最近一次稳定 release
- 资料保存链路
- 画像 summary 链路
- 星盘 chart 链路

## 4. 风险控制建议

- 每次改动先写影响面
- 每次改动后先跑最小 smoke
- 不把视觉收口和状态逻辑改动混成一批
- 文档先行，代码后改
