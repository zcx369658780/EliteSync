## 5. Claude Tester Development Record

Claude was introduced and validated as an Android app testing subagent during 5.5.

Role now proven useful:

- Appium / ADB-assisted app walkthrough.
- Page source and screenshot capture.
- UI reachability checks.
- Reproduction of small-sample feedback.
- Independent regression confirmation after Codex fixes.
- RTC path observation with Codex coordinating API / ADB / device state.

5.5 tasks where Claude materially contributed:

- Confirmed Status Plaza publish path structure before user write verification.
- Reproduced Profile floating action overlap and confirmed the fix.
- Checked Astro Overview Ziwei preview structure after fix.
- Reproduced Status Author Profile cast failure and confirmed the fix.
- Ran broad exploratory smoke across core app surfaces.
- Performed logged-in guarded route smoke after user authorized the smoke account.
- Supported RTC dual-device investigation and evidence collection; Codex retained responsibility for write operations, API calls, deploy, and cleanup.

Operational lessons to carry forward:

- Claude app tests can be slow; allocate long timeouts and avoid duplicate concurrent Appium sessions.
- Require structured outputs: path, visible text, evidence, and `pass / risky pass / fail / uncertain`.
- Claude can test low-risk UI paths by default; login, write actions, publish, delete, install, deploy, migration, restore, and production DB operations still require explicit user authorization and Codex coordination.
- A short Claude `PASS` is not enough for evidence. Ask for operation path, visible text, exceptions checked, and proof artifacts.

Persistent rules are recorded in:

- `AGENTS.md`
- `docs/project_memory.md`
