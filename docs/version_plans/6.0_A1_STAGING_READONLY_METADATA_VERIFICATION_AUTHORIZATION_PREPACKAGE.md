## 2. Future staging readonly metadata verification 目标

未来若获用户另行明确授权，staging readonly metadata verification 只验证 staging 的只读 metadata，不验证业务写链。

目标限于：

- staging 环境是否存在明确边界；
- staging 是否能与 production 区分；
- staging 是否可只读访问健康 / readiness / metadata；
- production 默认是否继续 v1；
- Flutter 默认是否仍不切 v2；
- R1 是否仍只保持三个 readonly endpoints；
- 是否未触发 DB write / migration / release chain；
- 是否没有 secrets 输出；
- 是否没有真实用户写操作。

## 3. Future allowed verification scope

未来若用户另行授权，最多只允许规划以下只读动作：

- 读取 repo tracked config source 的文件名与 key 名，不读取 secret values；
- 如 staging URL 已由用户显式提供，可只请求 staging 的 readonly metadata endpoint；
- 只允许 GET readonly metadata；
- 只允许验证 environment marker / app health / readiness / location contract metadata；
- 只允许记录 HTTP status、response shape、environment marker、version marker；
- 只允许保存脱敏证据；
- 只允许输出 redacted report。

注意：本 prepackage 阶段不得执行上述动作，只能写未来授权范围。

- 执行 API smoke / write smoke；

- 读取真实 `.env` / `.env.*` 内容；

- 新增 route / controller / DTO / Resource；
- 切 Flutter default base URL；
- 修改 release chain；
- 执行 Candidate C implementation；
