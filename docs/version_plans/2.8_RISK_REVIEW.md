# EliteSync 2.8 风险评审

更新时间：2026-04-06

## 高风险面
- 认证 / 登录链路
- 举报 / 封禁 / 申诉状态机
- 后台查询与处理接口
- 埋点写入
- 资料真源与缓存
- 2.6.4 发布门禁
- 2.7 匹配主链路

## 主要风险
1. 后台能力若与前台真值耦合过深，会拖慢主流程。
2. 状态机字段若设计不清，会导致举报 / 封禁 / 申诉互相覆盖。
3. 埋点若侵入主流程，可能影响性能与稳定性。
4. 认证 provider 抽象若过度扩展，容易演变成大重构。

## 回滚点
- 后台新增页面与接口
- 举报 / 审核 / 看板相关 provider
- 埋点接入点
- 任何新状态机字段

## 风险控制原则
- 优先模块化新增，不改写已有主链路。
- 后台先 MVP，不追求一次做满。
- 前端缓存只能兜底，不能变真源。
- 任何改动都必须先过回归与门禁。
