# EliteSync 3.4 风险评审

更新时间：2026-04-11  
版本状态：草案 v1  
对应执行计划：`3.4_EXEC_PLAN.md`

## 1. 风险总判断

3.4 的风险等级为：**中低风险，可控，但必须保持边界清晰**。

本版允许做：
- 页面口径统一
- 资料编辑体验收口
- 星盘显示精修
- 设置中心正式化
- Beta 运营准备与回归材料收口

本版禁止做：
- canonical 真值层大改
- 星盘算法重写
- 宫位制 / 黄道制 / 相位容许度 / 虚点的专业级开放
- 把显示层改造成算法调参面板

## 2. 高风险清单

### 2.1 页面口径与版本中心

风险：
- 版本中心、关于与更新、更新历史、health / version/check 之间文案不一致
- 页面标题与接口可观测性混淆成业务真值

回归点：
- `AboutUpdatePage`
- `SettingsPage`
- `app/health`
- `app/version/check`

回滚建议：
- 仅回滚文案与标题，不要回滚版本检查接口

### 2.2 资料编辑与出生地链路

风险：
- 保存后反馈不清楚，用户误以为没保存
- 重算失败 / 慢网时页面状态误导
- 出生地搜索与回填链路受影响

回归点：
- `EditProfilePage`
- `ProfilePage`
- `ProfileController`
- `GeoController`
- `POST /api/v1/profile/basic`
- `GET /api/v1/geo/places`

回滚建议：
- 保持接口响应兼容
- 若前端文案或加载态有问题，优先回滚 UI，不动后端真值流程

### 2.3 星盘阅读性精修

风险：
- 只改显示层但误伤当前本地 SVG 结构
- 预设档位或开关误改成算法层开关
- 标签、盘心、相位线的视觉可读性变化引入误判

回归点：
- `AstroNatalChartPage`
- `AstroChartSettingsPage`
- `natal_chart_svg_builder.dart`
- `astro_chart_settings_provider.dart`

回滚建议：
- 盘面显示出问题时，优先回到 3.3 的显示样式，不触碰后端 `chart_data`

### 2.4 设置中心正式化

风险：
- 入口重排导致旧入口不可达
- 设置项归类后影响用户操作路径
- 本地偏好项与服务端真值边界再次混淆

回归点：
- `SettingsPage`
- 隐私设置页
- 性能偏好
- 盘面设置页

回滚建议：
- 保留旧入口映射
- 本地偏好项只回滚前端层

### 2.5 Beta 运营准备

风险：
- health / version/check 结果被误当作业务真值
- 弱网首进、回滚检查、版本校验材料不完整
- 归档材料缺截图导致顾问复核成本上升

回归点：
- `/api/v1/app/health`
- `/api/v1/app/version/check`
- 弱网首进
- 回滚检查

回滚建议：
- 只回滚到已验证的 Beta 观测口径
- 不把健康检查扩展成第二真源

## 3. 兼容性提示

1. `3.2 / 3.2a / 3.3` 已通过能力必须保住
2. `profile/basic` 保存响应兼容旧字段读取
3. `chart_data` 与本地 SVG 结构兼容
4. 设置项本地偏好不回写 canonical 真值
5. 版本中心和 about 页标题最终要统一

## 4. 回滚优先级

1. 先回滚文案 / 标题 / 说明卡
2. 再回滚前端设置分组与显示开关
3. 最后才考虑回滚资料编辑与版本中心页面结构
4. 不允许直接回滚 canonical 真值链路，除非出现数据损坏或接口契约破坏

## 5. 结论

3.4 不是高风险算法版本。只要严格守住“显示层优化”和“真值层”边界，风险整体可控。
