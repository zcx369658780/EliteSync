# 4.6A 测试计划

更新时间：2026-04-21  
版本：`4.6 RTC / 通话基础设施版（语音优先）`

## 最小验证集

### 后端

- RTC 会话创建 / 查询 / 接受 / 拒绝 / 挂断
- 状态迁移与事件流水
- 通话通知写入与回流
- `AppVersionApiTest`
- `MessageApiTest`
- `NotificationApiTest`
- `MatchApiTest`

### Flutter

- 会话页通话入口
- 来电页 / 通话页 / 结束页
- 权限失败态
- 网络失败态
- 通知回流入口
- 旧主链回归：消息、通知、匹配、首页

## 手工验收步骤

1. 登录两台模拟器账号
2. 从会话页发起 1v1 通话
3. 被叫侧观察来电页 / 来电提醒
4. 接受通话
5. 验证通话状态变化
6. 挂断 / 拒绝 / 超时
7. 验证通知与结果回流
8. 验证旧主链未受影响

## 失败后补测建议

- 如果双端状态不一致，优先查信令 / 状态机
- 如果权限失败，优先查 AndroidManifest / 运行时权限桥
- 如果通知不回流，优先查通知写入与路由绑定
- 如果旧链受伤，立即回滚 RTC 入口和新增表
