# EliteSync 5.1F Voice Rhythm Note

更新时间：2026-05-02

## 目标

5.1F 将语音邀请从孤立按钮增强为关系节奏中的自然升级动作。

本轮只做 Flutter 前端 additive 改动，不重写 RTC 状态机，不改 LiveKit 主链，不扩多人语音或视频。

## 修改范围

- `apps/flutter_elitesync_module/lib/features/chat/presentation/pages/chat_room_page.dart`
- `apps/flutter_elitesync_module/lib/features/rtc/presentation/pages/rtc_call_result_page.dart`
- `apps/flutter_elitesync_module/test/features/chat/presentation/pages/chat_room_page_test.dart`
- `apps/flutter_elitesync_module/test/features/rtc/presentation/pages/rtc_call_result_page_test.dart`

## 当前实现

### 1. 聊天页语音前提示

聊天页在 `关系摘要` 内新增 `语音节奏` 区块：

- 提醒用户先用文字确认舒适度；
- 说明连续回聊、共同点被接住后再语音更自然；
- 提供 `查看语音前提示` 入口。

同时，关系摘要、语音提示、破冰建议、附件入口已进入聊天消息区的可滚动 header，避免在小屏或测试视口下挤压消息列表和底部输入框。

点击 `查看语音前提示` 或顶部语音入口发起前，会看到轻量确认弹窗：

- `继续文字`
- `现在语音`

只有用户确认 `现在语音` 后，才继续走既有权限检查与 `createCall` 流程。

### 2. 通话结果页关系回流

RTC 结果页新增 `通话后的继续建议` 或 `未接通后的回聊建议`：

- 通话结束后建议回到文字接住具体细节；
- 未接通、拒绝、忙线、失败时给出低压回聊文案；
- 不自动发送消息，只提供可读建议。

### 3. 主链保护

本轮未改：

- `RtcRemoteDataSource`
- `RtcLiveKitService`
- `rtc_providers.dart`
- RTC 后端 API
- LiveKit token / join / leave 主链
- 路由定义
- 权限服务
- 多人语音或视频能力

## 风险与观察项

- `conversationId` 仍作为 peer id 使用，延续前序观察项，5.1F 不改该 contract。
- 语音前确认弹窗会多一次点击，这是为了降低突兀呼叫风险；后续可通过测试反馈决定是否对高熟悉度关系减少提示。
- 结果页建议不写聊天草稿，避免自动生成未确认内容。

## 验收口径

- 聊天页可看到语音节奏提示；
- 发起语音前存在轻量确认；
- 未接通 / 通话后结果页能看到低压回聊建议；
- 聊天页不会因新增关系提示造成纵向溢出，输入框保持固定在底部；
- RTC 数据层、LiveKit、权限链路未被改动。
