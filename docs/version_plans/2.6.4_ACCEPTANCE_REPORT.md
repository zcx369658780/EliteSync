## 3. 临时烟测账号机制说明

### 3.1 创建机制
`scripts/smoke_backend_alpha.ps1` 在固定回归账号登录失败时，会自动创建一个临时烟测账号，自举 auth chain。

具体规则：
- 号段使用非真实前缀 `90`
- 默认用户名为 `SmokeUser`
- 默认生日为 `1998-01-01`
- 默认密码为 `Smoke12345`
- `realname_verified = true`
- fallback 注册支持多次重试，避免随机手机号碰撞导致 `422`

### 3.2 清理机制
烟测账号在完成 auth chain 后，会尝试调用：
- `DELETE /api/v1/auth/account`

后端已对该接口做了保护：
- 仅允许 smoke / synthetic 账号自删
- 通过后会删除当前 token 与用户记录

### 3.3 不污染后续匹配测试的原因
- 烟测账号只在 smoke/auth chain 中临时生成，不进入正式业务创建流程
- smoke 流程不会为该账号写入问卷答案，因此不会进入匹配候选池
- 匹配命令默认排除 `is_synthetic=true` 用户
- 日志会显式记录 fallback 账号，便于审计与手工清理
