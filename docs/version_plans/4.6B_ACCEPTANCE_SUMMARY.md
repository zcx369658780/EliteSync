## 已完成项

1. 后端已形成 RTC 会话域与事件域。
2. `/api/v1/rtc/calls` 已支持创建、查看、接听、拒绝、进入通话、挂断。
3. 通话状态机已具备最小闭环。
4. Flutter 已具备通话入口页与通话页骨架。
5. 通知中心已能承接通话回流。
6. Telemetry 已补齐通话入口与状态变化事件。

## 验收依据

- `RtcApiTest` 覆盖主要状态流
- `FrontendTelemetryApiTest` 覆盖通话事件 allowlist
- `RtcCallPage` widget test 已通过
- Flutter analyze 无新增 issue

## 非阻断观察项

- 目前仍是语音优先骨架，未接入真实 RTC 媒体引擎
- 来电页、通话结果页、权限与设备异常恢复仍可继续加厚
- 视频通话只保留预留语义，不做完整媒体闭环

## 不在本轮范围

- 多人通话
- 直播
- 在线状态平台
- RTC 平台化
- 重写消息 / 通知 / 媒体主链

## 建议

- 冻结当前 4.6 最小闭环
- 后续若继续扩展，优先围绕来电页、权限提示、异常恢复、回流通知做增量
