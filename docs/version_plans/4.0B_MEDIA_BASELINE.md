# EliteSync 4.0B 媒体对象存储最小闭环

日期：2026-04-18

## 已落地的闭环

- `config/media.php` 已提供媒体磁盘、单文件大小、允许 mime 前缀和公共路径前缀配置。
- `config/filesystems.php` 已补 `oss` 适配层，支持通过环境变量切换到 OSS / S3 兼容对象存储。
- `MediaUploadPolicy` 已能读取媒体配置并做 mime / size 最小校验。
- `MediaStorageService` 已封装对象存储写入、路径生成和 public URL 解析。
- `MediaAssetService::registerUploaded()` 已形成真实上传闭环：
  - 创建 `uploading` 记录
  - 写入对象存储主路径
  - 更新为 `uploaded`
  - 创建处理 job
  - 触发处理 job 进入 `processing / ready`
  - 失败时回写 `failed / blocked` 与错误码
- `MediaStateMachine` 已固定媒体状态与允许迁移关系。

## 现在的状态机口径

- `pending`
- `uploading`
- `uploaded`
- `processing`
- `ready`
- `failed`
- `blocked`
- `deleted`

## 仍然不做的事

- 不做分片上传。
- 不做断点续传。
- 不做签名 URL 工业化协议。
- 不做客户端直传完整 SDK 产品化。
- 不把对象存储逻辑泄露到 Flutter UI。
- 不把媒体状态改成业务真值。
