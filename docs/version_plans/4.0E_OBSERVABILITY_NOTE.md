# EliteSync 4.0E 可观测性与限流说明

日期：2026-04-18

## 已落地

- 新域 API 已按用途拆分 limiter：
  - `conversations`
  - `media`
  - `relationships`
  - `notifications`
- `MediaController`、`ConversationController`、`RelationshipController`、`NotificationController` 已补最小结构化日志。
- 媒体队列可通过 `app:media:process-queue` 做最小 worker 扫描。

## 口径

- 限流只作用于新域，不扩散到 3.x 真值链。
- 日志只记录必要的业务元信息，不记录敏感值。
- 4.0E 的目标是可定位、可回放、可交接，不是搭建完整监控平台。

## 边界

- 不引入重型监控系统。
- 不把日志当作数据真值来源。
- 不重写现有版本检查、画像或匹配链路。
