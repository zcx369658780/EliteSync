# 4.6P 验收摘要

更新时间：2026-04-25

## 当前结论

`4.6P` 建议按 `pass with observations` 收口。

本轮已经完成的核心事实是：

- 1v1 RTC 状态机继续成立
- LiveKit 真语音链路已进入可用阶段
- 手机端发言时，电脑端模拟器可以实际听到声音
- 模拟器麦克风权限已补齐
- 现阶段不需要再改 APK 架构或回退 RTC 方案

## 当前音频算法与链路说明

### 1. 通话生命周期仍由后端状态机掌控

后端 `rtc_sessions` / `rtc_session_events` 负责：

- 发起
- 接听
- 忙线
- 超时
- 挂断
- 终态回收

状态机继续作为真源，不承担媒体传输。

### 2. 真语音由 LiveKit 承担

当通话进入 `connecting / in_call` 后，Flutter 通过：

- `GET /api/v1/rtc/calls/{callId}/livekit`

获取：

- `url`
- `token`
- `room`
- `identity`
- `participant_name`

然后使用 `RtcLiveKitService` 建立房间连接，并启动：

- `Room.connect(...)`
- `room.startAudio()`
- `room.setSpeakerOn(true)`
- `localParticipant.setMicrophoneEnabled(true)`

### 3. 诊断链路

`RtcLiveKitService` 目前保留以下诊断：

- `RTC_LIVEKIT_ROOM_CONNECT_OK / FAIL`
- `RTC_LIVEKIT_TRACK_SUBSCRIBED`
- `RTC_LIVEKIT_REMOTE_AUDIO_REFRESH`
- `RTC_LIVEKIT_REMOTE_AUDIO_STATS`
- `RTC_LIVEKIT_LOCAL_AUDIO_FRAME`
- `RTC_LIVEKIT_AUDIO_PLAYBACK_*`
- `RTC_LIVEKIT_PEER_CONNECTION`

`RtcAudioSpectrumBar` 用于区分：

- 等待远端音轨
- 已接入但无声
- 远端音轨输入中

### 4. 本轮关键验证

- 手机端本地麦克风 PCM 已经能看到非零值
- 模拟器端已经能接到对端音轨并在实测中听到声音
- 说明问题不再是“完全没有音轨”
- 说明媒体链与播放链已经闭合到可用

## 已成立的事实

- 来电页可唤起
- 双端可连接
- 双端可自动接通
- 双端可挂断并回到结果页
- 后端 join-info 可用
- 云端 LiveKit 服务可用
- 实际音频输出已在现场联测中验证成功

## 仍保留的观察项

- 模拟器端麦克风路径建议再做一次反向复测，确保“模拟器说话 -> 手机听到”也稳定
- 音频频谱仍是诊断层，不应替代真实听感

## 验收建议

建议将 4.6P 结论固定为：

> `pass with observations`

并将观察项保留为：

- 模拟器反向发言链的最终复测
- 诊断 UI 继续保留，但不再继续扩写 RTC 主链
