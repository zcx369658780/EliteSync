## 4. 不要动的边界

Codex 接手时请继续遵守：

1. **不要改 canonical 真源规则**
   - 服务端仍是唯一真源
   - 前端缓存只做体验兜底

2. **不要大改受保护链路**
   - 注册 / 登录
   - 问卷提交流程
   - 资料保存与画像重算
   - 匹配主链路
   - 聊天权限与治理拦截
   - 版本检查 / 健康检查
   - smoke / regression / rollback 体系

3. **不要在 3.0 阶段引入大范围架构重写**
   - 不重写消息底层
   - 不重写匹配算法
   - 不重做数据库 schema
   - 不扩成完整实验平台
