# 4.6F 依赖链与受影响面

更新时间：2026-04-23

## 现有调用链

- 通话入口 -> `RtcPermissionPage`
- 来电拉起 -> `rtc_invite_coordinator.dart`
- 通话主链 -> `RtcCallPage`
- 来电页 -> `RtcIncomingCallPage`
- 终态页 -> `RtcCallResultPage`
- 后端状态机 -> `RtcController` / `RtcSessionService`

## 隐藏耦合点

- 现有 RTC 状态机仍是调用真源
- 语音媒体层必须与会话状态同步，但不能反向覆盖状态机
- 通话页返回逻辑必须继续保持可回消息页
