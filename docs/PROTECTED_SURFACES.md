# 高风险模块保护清单

> 默认保护这些面。任何任务碰到下面任一项，都必须先 plan-first，再实施，再验收。

## 1. 数据库与迁移

- 本地数据库
- 远端数据库
- 初始化逻辑
- 数据迁移脚本
- 回填脚本
- 恢复脚本
- 去重 / 清理脚本

保护规则：

- 先备份，后修改
- 先写回滚点，再写实现
- 先确认数据兼容，再改 schema

## 2. 地图 / 定位 / 坐标

- 定位权限申请
- 百度地图 / 其他地图 SDK
- 地点搜索
- 逆地理编码 / 地理编码
- 经纬度保存与刷新
- 坐标转换

保护规则：

- 不得把地图能力悄悄降级成静态城市表
- 不得只改 UI 而不查坐标链路
- 不得在未确认 SDK 可用时直接改入口

## 3. 路由 / 页面生命周期

- 页面跳转
- 入口合并 / 拆分
- 返回栈
- RouteAware / lifecycle
- deep link / query 参数

保护规则：

- 旧入口必须写清楚映射
- 页面重构不得丢失原行为

## 4. 状态管理 / 缓存

- provider / view model / session
- 本地 snapshot
- 持久化缓存
- 登录态恢复

保护规则：

- 单一真源优先
- 保存后回读必须明确
- 禁止多层缓存互相覆盖而无记录

## 5. 配置 / 环境变量 / 第三方 SDK

- `.env`
- `local.properties`
- 证书 / AK / token
- SDK 初始化入口
- 第三方服务开关

保护规则：

- 运行时配置与仓库文档必须一致
- 禁止把关键配置写死在 UI 或测试兜底里

## 6. 自动备份 / 恢复 / 版本升级

- 备份脚本
- 恢复脚本
- 发布脚本
- changelog
- 版本检查 / 下载地址

保护规则：

- 每次版本更新必须验证备份链路
- 回滚必须能实际执行

## 7. 兼容层 / 废弃项

- 旧入口
- 旧字段
- 旧接口
- 旧文案

保护规则：

- 只能在计划中逐步收束
- 不得直接删除未确认的兼容映射
