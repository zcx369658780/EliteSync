# EliteSync 当前开发计划（2026-03-27）

## 一、当前阶段结论
- 当前版本基线：`0.02.01`
- 里程碑状态：已完成 UI 重构并切换到 Flutter 主界面体系，Android 宿主完成合并；发布链路与回归链路可用。
- 当前主目标：在新 UI 基线上做功能补全与稳定性收敛，进入 Beta 前的可测版本准备。

## 二、已完成（重点）
1. 完成 Flutter 新 UI 架构落地并接入 Android 宿主。
2. 完成主流程链路打通：登录/注册、匹配、消息、我的、设置。
3. 完成关键设置能力：退出登录、检查更新、修改密码。
4. 完成发布与回归链路修复：CI/Regression 均支持 Flutter AAR 依赖构建。
5. 完成版本发布：`0.02.01` 已上线阿里云，并通过版本接口与下载自检。

## 三、UI 遗留任务（新架构下）
1. 搜索历史交互细节：
- 当前可用，但“仅输入框聚焦时展示”体验仍需一轮统一验收。
2. 子页面动画一致性：
- 主界面过渡已稳定，部分子路由动效节奏仍可统一（入场/返回时长、曲线）。
3. 文案反馈统一：
- 个别页面的操作反馈（toast/气泡）样式和展示时机仍需统一规范。
4. 列表骨架屏与空态：
- 已有基础实现，仍需按页面逐个校验视觉一致性与加载策略。

## 四、功能侧下一步（Beta 前）
1. 内容服务化：
- 首页/发现的卡片内容由后端持续补齐，减少本地兜底占比。
2. 匹配解释增强：
- 在现有匹配分项基础上继续完善可读解释与证据标签映射（中文化、去变量名）。
3. 账号与安全：
- 补齐密码修改后的风控校验与异常态提示；后续接入更完整账号安全流程。
4. 配置治理：
- 将环境与发布配置继续标准化，减少本地缓存对构建结果的影响。

1. 开发分支持续迭代，主分支每天最多一次 Merge + Regression（按现有限额策略执行）。
