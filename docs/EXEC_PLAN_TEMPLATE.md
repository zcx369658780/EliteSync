## 4. 计划前置 subagent

必须先并行完成以下只读评审：

1. `dependency-mapper`
2. `risk-reviewer`
3. `test-planner`
4. `architecture-guardian`

### 4.1 dependency-mapper 重点问题

- 按钮/入口到页面的调用链是什么？
- 状态从哪里来，写到哪里去？
- 哪些文件是核心耦合点？
- 旧功能入口是否需要映射？

### 4.2 risk-reviewer 重点问题

- 哪些高风险模块会被触碰？
- 有没有数据库 / 定位 / 路由 / 权限 / 配置回归风险？
- 有没有回滚点？

### 4.3 test-planner 重点问题

- smoke test 该覆盖什么？
- 回归测试该覆盖什么？
- 手工验收步骤是什么？
- 失败后的补测是什么？

### 4.4 architecture-guardian 重点问题

- 需求是否越过分层边界？
- UI 改动是否顺手改了业务逻辑？
- 是否需要拆单？
