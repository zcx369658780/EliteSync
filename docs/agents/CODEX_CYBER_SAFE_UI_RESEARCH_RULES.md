# Codex Cyber-Safe UI Research Rules

更新时间：2026-05-11

本文件是 EliteSync 仓库内竞品 UI 研究的稳定安全规则路径。适用于 Codex、Claude、Gemini 或其他工具协助观察本人自有测试设备上的公开可见 App UI。

## 1. 任务定性

竞品 App 拆解任务默认属于低风险产品 UI 研究，不属于：

- 安全测试
- 逆向工程
- 漏洞研究
- 权限绕过
- 抓包
- 接口分析
- 数据提取
- 批量采集

如任务内容开始接近上述范围，必须暂停并重新向用户确认边界。

## 2. 允许的证据

仅允许采集和保存以下证据：

1. 当前屏幕截图。
2. Android UI hierarchy，仅用于记录当前屏幕可见控件结构和文案。
3. 手工整理的产品结构报告。

Android UI hierarchy 只能作为可见 UI 的辅助记录，不能被解释为读取本地私有文件、内部数据、网络响应、接口字段或后端实现。

## 3. 禁止行为

默认禁止：

- 读取本地私有文件或 App 内部数据。
- 抓取网络流量。
- 调用或分析接口。
- 绕过登录、权限、付费墙或风控。
- 逆向 App、解包、Hook、注入、调试内部逻辑。
- 批量抓取内容。
- 填写、编辑、保存或提交真实个人资料。
- 上传图片、截图、文件或资料。
- 支付、下单、开会员。
- 进入真实咨询、成交或服务链。
- 发布、评论、点赞、收藏、关注。
- 授权额外权限。
- 登录其他账号。

## 4. 必须停止的页面

遇到以下状态必须停止并记录 blocker：

- 登录页或要求切换账号。
- 付费、下单、开会员、咨询成交。
- 填写或保存资料。
- 选择完成并可能提交 / 保存 / 生成。
- 上传图片、截图、文件或资料。
- 授权敏感权限。
- 发布、评论、点赞、收藏、关注。
- 任何看起来会改变账号、内容、订单、咨询、档案或关系状态的操作。

停止记录至少包含：

- blocker 页面。
- 触发动作。
- 当前可见文案。
- 如果继续需要什么授权。
- 本轮是否点击了高风险 CTA。

## 5. 报告写法

报告必须区分：

- 页面入口可见。
- CTA 可见。
- 浅层页面已进入。
- 目标页已验证。
- 真实结果页已验证。
- 只是示例 / 预览 / 历史对照。
- 只是产品意图推断。

不得把“入口存在”写成“完整链路已验证”。

不得从竞品 UI 推断：

- 服务端 contract。
- 算法。
- 数据库 schema。
- 内部接口。
- 风控逻辑。
- 商业转化真实效果。

## 6. EliteSync 产品化边界

将竞品 UI 观察转化为 EliteSync 产品设计时，必须遵守：

- 不重写 canonical truth。
- 不改写 `profile/basic`。
- 不改写 `profile/astro/summary`。
- 不改写 `profile/astro/chart`。
- 不改写 `user_astro_profiles`。
- 玄学、关系解释、AI 追问、DeepSeek 解读、建议文案只能作为 derived-only / display-only / explanation layer。
- 不能把解释层、AI 追问或前端展示反写为资料真值。
- 不能让前端缓存抢占服务端真源。

## 7. Claude / Gemini / Shell 规则

- 调用 Claude / Gemini 前先确认 shell 身份，不得使用 `CodexSandboxOffline` 用户启动。
- 当前推荐用户身份是 `zcx_265k\zcxve`，SID 后缀 `-1001`。
- 禁止在同一设备上并发启动多个 Claude / Appium / UI 自动化观察任务。
- 外层 shell 超时不等于 Claude 已失败；若进程仍在运行，应等待或确认退出后再决定是否重试。
- 每次 Claude UI 观察 prompt 必须包含本文件等价的低风险 UI 研究声明。
