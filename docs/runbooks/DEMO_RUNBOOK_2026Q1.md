# EliteSync 演示手册（2026 Q1）

更新时间：2026-03-19  
适用版本：`phase-a-2026-03-12`

## 1. 演示目标
- 向项目组展示当前已完成的核心闭环：
  - 注册/登录
  - 20题问卷
  - 匹配结果与解释
  - 聊天收发
- 展示“本地开发 + 阿里云后端联调”能力已可用。

## 2. 演示环境
- 后端地址：`http://101.133.161.203`
- 健康检查：`http://101.133.161.203/up`
- 安卓端当前配置：
  - API：`BuildConfig.API_BASE_URL`（默认指向上面 IP）
  - WS：`BuildConfig.WS_BASE_URL`（默认 `ws://101.133.161.203:8081/`）

## 3. 会前准备（10分钟）
1. 检查云端服务状态（任一终端）  
`systemctl status nginx php8.4-fpm mariadb redis-server elitesync-ws`
2. 检查健康页可访问  
打开 `http://101.133.161.203/up`，确认 `Application up`。
3. Android Studio 预编译 debug/release 包，避免会议现场等待构建。
4. 准备两台设备（推荐：真机 + 模拟器）。

## 4. 标准演示流程（建议 8-12 分钟）
1. 账号A注册并登录  
- 输入手机号与密码（密码需字母+数字，至少8位）。
2. 完成问卷并进入匹配  
- 展示题目可读化后的选项。
3. 账号B重复上述流程  
- 确保形成可匹配对象。
4. 进入匹配页  
- 展示匹配对象、匹配解释/分数字段。
5. 双端进入聊天  
- 发送/接收消息，展示实时刷新与已读状态。
6. 展示异常兜底  
- 非法密码注册失败（422）作为安全基线示例。

## 5. 演示话术要点（建议）
- 当前不是“UI 完成版”，是“功能可运行 + 可迭代”版本。
- 已完成的关键里程碑：
  - 云端联调打通
  - 2100题题库可读化
  - 聊天消息数据库加密存储
  - 接口限流与 HTTPS 强制开关就绪
- 待会议决策后再进入：
  - 产品方向（题库/匹配策略）
  - 合规接口（微信/学信网）
  - 商业化路径

## 6. 现场故障快速处理
1. 提示“网络不可用”  
- 检查安卓 `network_security_config` 与 API 地址。
2. 注册返回 500  
- 检查云端 `.env` 的 DB 配置是否被覆盖。
3. 匹配为空  
- 用第二账号完成问卷后重试；必要时触发开发态匹配接口。
4. 聊天不刷新  
- 检查 `elitesync-ws` 是否 active；确认 8081 监听。
