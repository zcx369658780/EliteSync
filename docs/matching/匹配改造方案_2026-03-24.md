# 匹配系统改造方案（2026-03-24）

## 目标
1. 让人格测试与 MBTI 真正进入主分（解决“答题无感”）。
2. 让匹配理由可解释（每模块有证据字段）。
3. 让前端展示可读（分行、分色、分层）。

---

## 一、现状结论（问题定位）
- 当前总分结构失衡：`astro_total` 直接进入 `base`，人格分与 MBTI 参与度偏低。
- MBTI 当前仅在 `penaltyFactors` 中做轻量乘子，存在感不足。
- 理由文本偏模板化，缺少可核验的证据字段。
- UI 上理由虽已有结构，但未做到“模块卡片 + 颜色层级 + 证据粒度”。

---

## 二、权重方案（顾问建议）

- MBTI：15%

玄学内部：

## 三、数据结构改造

## 3.1 后端评分层（新增显式子分）
在 `dating_matches` 扩展字段：
- `score_personality_total`
- `score_mbti_total`
- `score_astro_total`
- `score_overall`

说明：保留现有 `score_base/score_final/score_fair` 兼容旧客户端。

## 3.2 match_reasons 结构升级
从当前简单 `{summary,match,mismatch,confidence}` 升级为：
```json
{
  "summary": "...",
  "confidence": 0.82,
  "modules": [
    {
      "key": "personality|mbti|bazi|zodiac|constellation|natal_chart",
      "label": "人格匹配",
      "score": 81,
      "weight": 0.50,
      "confidence": 0.90,
      "verdict": "strong|medium|weak",
      "highlights": [{"text":"...","evidence":{...}}],
      "risks": [{"text":"...","evidence":{...}}],
      "degraded": false,
      "degrade_reason": ""
    }
  ]
}
```

## 3.3 证据字段规范（关键）
- personality：`vector_distance`, `attachment_gap`, `conflict_repair_gap`, `interest_overlap`, `bidirectional_acceptance_score`
- mbti：`type_a`, `type_b`, `matched_letters`, `temperament_group_a/b`
- bazi：`wu_xing_a/b`, `wu_xing_distance`, `wu_xing_complement_index`
- zodiac：`animal_a/b`, `relation_type`（六合/三合/冲/刑/害）
- constellation：`sign_a/b`, `element_a/b`, `relation_type`
- natal_chart：`sun/moon/asc` 兼容分、`data_completeness`

## 四、算法改造（P1）

## 4.1 总分公式
```
core_total = personality_w * personality_total
           + mbti_w * mbti_total
           + astro_w * astro_total

final_total = core_total * penalty_product * fairness_multiplier
```

## 4.2 人格分（personality_total）
建议拆分：
- `profileSimilarity` 35%
- `biQuestion` 35%
- `categoryCompatibility` 20%
- `interestSimilarity` 10%

## 4.3 MBTI 分（mbti_total）
先用轻量稳定版：
- 字母同位命中：最多60
- 同气质组（NT/NF/SJ/SP）：+20
- 完全同型：+10
- 置信度修正：+/-10

## 4.4 防失衡保护
- 人格分过低时，限制最终上限；人格分高时避免被单模块过度拉低。

## 五、前端展示改造

## 5.1 匹配详情页结构
- 顶部：Verdict Badge + 总分 + 置信度 + Summary
- 中部：模块卡片（人格/MBTI/八字/属相/星座/星盘）
- 下部：每条理由单独一行（ReasonRow），并显示 EvidenceChip
- 风险项单独分区

## 5.2 推荐页简版
- 仅显示：Badge + 总分 + 一句话总结 + Top2理由

## 5.3 颜色规则（按模块分数）
- >=80：高亮冷白/淡青
- 65-79：正文主色
- 50-64：次级偏黄
- <50：警示橙红
