## 顾问任务 A：八字匹配（50%）
### 输入
- 出生日期、出生时刻、出生经纬度（可为空）
- 已保存八字结构（天干地支、五行强弱）
### 输出
- `score_bazi` (0-100)
- `reason_bazi_match[]` / `reason_bazi_mismatch[]`
- `confidence_bazi` (0-1)
### 要求
- 可复现规则，禁止纯主观文案
- 缺失输入时提供降级策略并说明

## 顾问任务 B：属相匹配（30%）
### 输入
- `zodiac_animal`（双方）
### 输出
- `score_zodiac` (0-100)
- `relation_type`（六合/三合/冲/刑/害/普通）
- `reason_zodiac`
### 要求
- 关系冲突时需有优先级
- 输出可追踪（配置化关系表）

## 顾问任务 C：星座匹配（10%）
### 输入
- 太阳星座
### 输出
- `score_constellation` (0-100)
- `reason_constellation`
### 要求
- 基于元素/宫位相容规则
- 缺失时有默认策略

## 顾问任务 D：星盘匹配（10%）
### 输入
- 太阳/月亮/上升，及主要相位（可降级）
### 输出
- `score_chart` (0-100)
- `reason_chart`
- `confidence_chart`
### 要求
- 输入缺失时可按太阳或太阳+月亮降级

## 融合模块（后端实现）
- `final_score = 0.5*bazi + 0.3*zodiac + 0.1*constellation + 0.1*chart`
- 输出：
  - `match_verdict`（高/中/低）
  - `why_match[]`
  - `why_not_match[]`
  - `algo_version`

## 验收指标
- 解释覆盖率 > 95%
- 所有评分项可落库可回放
- 1000账号回放耗时可控（CLI 匹配命令）
