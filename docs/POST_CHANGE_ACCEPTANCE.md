# 修改后验收评估规范

> 这是 `acceptance-auditor + regression-sentinel` 的可复用执行规范。修改完成后，必须先完成这份复核，再决定是否交付。

## 1. 必须并行完成的验收角色

1. `acceptance-auditor`
2. `regression-sentinel`
3. `test-planner` 复核覆盖
4. `architecture-guardian` 复核边界漂移

## 2. 验收目标

- 核实需求是否真正实现
- 核实是否引入回归
- 核实高风险模块是否被保护
- 核实是否存在未验证项

## 3. 统一输出格式

### 3.1 已通过项

- 功能实现是否达标
- 关键流程是否可用
- 回归测试是否通过

### 3.2 风险项

- 可能残留的问题
- 可能误伤的旧功能
- 可能不稳定的配置

### 3.3 未验证项

- 还没测到的路径
- 还没覆盖的边界条件
- 还没复核的兼容层

### 3.4 建议补测项

- 手工验收
- 边界测试
- 旧功能回归
- 数据一致性核对

### 3.5 交付结论

- `pass`
- `risky pass`
- `fail`

## 4. regression-sentinel 必查项

- 数据库初始化/保留
- 地图定位链路
- 权限链路
- 页面跳转
- 关键按钮事件
- 错误处理

## 5. 交付门槛

只要任一验收角色输出 `fail`，必须进入修复轮次，不能直接结束。

## 6. 验收报告模板

```
已通过项：

风险项：

未验证项：

建议补测项：

是否允许交付：
```
