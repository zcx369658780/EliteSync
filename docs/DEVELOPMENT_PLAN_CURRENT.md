# EliteSync 整体开发计划书入口

当前可编辑的主计划请看：
- `docs/version_plans/elite_sync_整体开发计划书_4_x方向重整版_2026_04_17.md`
- 当前 4.3 动态流基础版已正式归档；当前 4.4 视频消息版已正式归档并通过 `pass with observations`，后续如果继续，只应从 4.4 的 additive 基线承接新版本，不要回头重开 4.3 主链。
- 当前 4.5 通知与社交转化增强版已进入边界冻结与最小骨架阶段，后续实现应围绕站内通知、回流与轻量转化，不得扩成推送平台或实时系统。
- 当前 4.6 RTC / 通话基础设施版（语音优先）已完成 `4.6D` 权限提示与异常恢复壳层；本地环境默认只做前端开发、UI 联调与文档整理，后端开发、数据库迁移、备份、恢复和任何会写数据库的操作统一在阿里云端执行。
- 当前 4.6F LiveKit 语音接入子任务已进入收尾裁决阶段：保留现有 RTC 状态机，仅新增语音媒体层、join info 接口和 Flutter 连接壳层；目前通话能连但真实语音仍未闭合，后续不再盲修主链，先交 GPT 顾问裁决。
- 当前 4.6P 已作为 4.6 / 4.6A 之后的插入式修正版进入收口：只盯“听到声音”这一 blocker，范围收窄到音频播放链、远端音轨绑定、AudioPlayback 与音频路由收口；现在已完成真语音可听闭环，建议按 `pass with observations` 收口，不再扩 UI 或重写 RTC 主链。
- 阿里云端已拉起 LiveKit 自托管容器并写入 `LIVEKIT_*` 环境变量，`/api/v1/rtc/calls/{callId}/livekit` 已可返回可用 join-info；后续只做两端真语音联测与收口，不回头重写 RTC 状态机。
- 本地环境默认只做前端开发、UI 联调与文档整理；后端开发、数据库迁移、备份、恢复和任何会写数据库的操作统一在阿里云端执行，避免本地更新误污染生产后端数据库。

本文件只保留为当前计划入口，不作为计划正文。
