# Soul Chat Relationship Progress Map

日期：2026-05-02

## 1. 关系推进链总览

Soul Chat 的关系推进链可以抽象为：

`发现 / 奇遇曝光 -> 关闭或未处理 -> 稍后再聊队列 -> 单聊页关系摘要 -> AI 低压破冰 -> 输入工具盘互动 -> 关注 -> 通话邀请`

## 2. 阶段拆解

| 阶段 | Soul 表达 | 用户心理任务 | UI 证据 | EliteSync 可参考 |
| --- | --- | --- | --- | --- |
| 曝光 | 广场、同城、奇遇 | 看见潜在对象 | Discover 同城、同频信号、私聊按钮 | Discover 到 Chat 的低压转化 |
| 暂停 | 关闭用户 | 暂不处理但不彻底丢失 | `奇遇铃-稍后再聊` | 冷场/稍后队列 |
| 回收 | 最近 3 天关闭用户 | 给用户一个重新打开理由 | `仅保留最近3天你直接关闭的用户哦` | 设置有限时效，避免堆积 |
| 解释 | 匹配度、ISFJ、引力签、共同点 | 明白为什么能聊 | 单聊页关系卡 | Match 解释进入 Chat |
| 破冰 | AI 灵感回复 | 降低开口成本 | `灵感回复可提升回复率哦` | AI 首聊建议靠近输入框 |
| 轻互动 | 交换答案、猜拳邀请 | 低风险互动升级 | 输入更多工具盘 | 慢约会小游戏/问答候选 |
| 升级 | 关注后可邀请通话 | 建立最低信任阈值 | `关注后可邀请通话` | 语音入口受关系节奏控制 |

## 3. 关键设计规律

### 3.1 回流不靠强提醒，而靠“可处理队列”

`奇遇铃-稍后再聊` 不是弹窗提醒，而是放在聊天列表第一屏的队列入口。它不会强迫用户立即回应，但让用户下次进入 Chat 时能看到未完成关系。

### 3.2 解释必须贴近行动

匹配度、人格标签、引力签和共同点没有只停留在匹配页，而是进入单聊页顶部。这样用户在准备开口时仍能看到理由。

### 3.3 AI 建议靠近输入动作

AI 破冰不作为独立功能孤岛，而是紧贴输入框。用户看到建议后可以直接决定是否使用。

### 3.4 语音升级需要关系门槛

`关注后可邀请通话` 明确告诉用户：通话是关系升级动作，不是陌生聊天默认动作。

## 4. EliteSync 5.x 映射

已适配 5.1 的方向：
- 冷场恢复队列。
- 匹配解释到聊天建议联动。
- 语音节奏确认。
- 通知/回流产品化。

可给后续版本参考：
- 把“稍后再聊”从本地 UI 表达扩展为真实队列。
- 对每个待回流对象保留一个低压理由：共同点、附近、近期动态、匹配解释。
- 在聊天输入区提供“问一个轻问题 / 夸一句 / 约一个低压活动”的候选按钮。
- 不急于引入礼物、商品、商业化互动。
