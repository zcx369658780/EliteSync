# 当前计划与下一步

更新时间：2026-04-19

## 当前最建议继续推进的方向

- 如果继续按当前产品线推进，下一版建议先等待顾问下发 `4.5` 计划书。
- 若顾问继续沿用当前 4.x 扩展路径，则建议从 `4.5` 开始，不再回头重开 `4.3` / `4.4` / `4.4S` 已归档主链。

## 为什么这样排

- `4.3` / `4.4` / `4.4S` 已经把动态流、视频消息和媒体链稳定性修完，当前不建议再回头扩写这些已归档主链。
- 如果继续向前，最自然的是开启下一条明确的新版本计划，而不是继续修补已归档版本。
- 这样可以避免把已冻结的媒体 / 动态 / 消息主链重新搅在一起。

## 已满足的前置条件

- 4.0 媒体 / 存储 / 附件基础设施已完成
- 4.1 人格模块已正式化
- 4.2 图片消息已经接入聊天主链
- 4.3 动态流基础版已归档
- 4.4 视频消息版已归档
- 4.4S 媒体链稳定性修正已完成
- 版本中心、版本检查、发版脚本已收口
- 保护面接口和真值链已稳定

## 不能跳过的前置条件

- 不能跳过 `profile/basic`、`profile/astro/summary`、`profile/astro/chart` 真值链
- 不能跳过 `media_assets` / `message_attachments` / `chat_messages` 的稳定性
- 不能跳过版本检查和发版脚本一致性
- 不能跳过截图证据、回归清单和 closeout 文档
- 不能跳过云端后端与本地前端的环境边界

## 如果要做 4.3

- 先写边界说明，再写执行计划
- 优先定义“动态流基础版”能做什么、不能做什么
- 明确是否只做 feed / post / lightweight interaction，不要直接把内容平台化

## 如果要做 4.4

- 在 4.3 的分发和互动边界稳定后再做
- 重点是视频的上传、预览、回放与治理边界
- 不建议在 4.4 同时再开新的人格或玄学大改

## 当前建议

- 继续做新版本前，先冻结 `4.2` 主链。
- 任何新版本都必须从当前稳定基线承接，不要回头重做已归档版本。
