# 3X 到 4X 的版本演进

更新时间：2026-04-19

## 3.x 收口阶段做了什么

- `3.4 - 3.7`：资料真值、星盘阅读性、路线解释、多路线样例、高级能力 scaffold 和最终归档逐步完成。
- `3.8`：版本中心三层口径、参数联动、截图证据与第二次验收材料包完成。
- `3.9`：高级时法框架首版、细粒度解释层、验收摘要、截图证据与多 Agent 审查链完成，正式归档通过。

## 为什么 4.x 转向“基础能力补全”

- 3.x 已经把玄学 / 画像 / 匹配的核心真值链稳定下来。
- 后续如果继续做社交能力，优先级不再是“重做真值”，而是“补工程底座”。
- 因此 4.x 的主线变成：
  - 媒体基础设施
  - 人格模块正式化
  - 图片消息正式接入

## 4.0 的定位与完成情况

- 定位：基础能力基建版
- 完成：
  - 数据骨架
  - 对象存储主路径
  - 媒体状态机
  - 队列 / 缓存 / worker 最小闭环
  - 附件上传骨架
  - 最小可观测性

## 4.1 的定位与完成情况

- 定位：非官方四维人格问卷与人格能力补全版
- 完成：
  - 版本化问卷
  - 结果生成
  - 历史与复测
  - 首页 / 匹配轻量联动
  - walkthrough 证据链

## 4.2 的定位与完成情况

- 定位：图片消息正式接入版
- 完成：
  - 选图
  - 上传
  - 附件绑定
  - 图片消息发送
  - 图片气泡展示
  - 预览
  - 失败重试
  - 会话摘要回读

## 4.3 的定位与完成情况

- 定位：动态流基础版
- 完成：
  - 动态发布
  - 动态读取
  - 点赞 / 取消点赞
  - 删除自己的动态
  - 举报 / 拉黑
  - 作者页与首页轻量联动

## 4.4 的定位与完成情况

- 定位：视频消息版
- 完成：
  - 单视频消息发送
  - 视频预览
  - 视频回读
  - 会话摘要语义承接

## 4.4S 的定位与完成情况

- 定位：媒体链稳定性修正版
- 完成：
  - `public_url` 规范化
  - 图片 / 视频内容端点兜底
  - 历史媒体资源恢复可读

## 关键演进节点

- `0.03.02a`：星盘本地绘制和宿主 bootstrap 修正
- `0.03.09`：3.9 正式归档
- `0.04.02`：4.2 图片消息与 4.1 问卷正式收口
- `0.04.04`：4.3 动态流、4.4 视频消息、4.4S 媒体链稳定性修正收口

## 结论

- 3.x 解决“真值和阅读”
- 4.0 解决“媒体底座”
- 4.1 解决“人格正式化”
- 4.2 解决“图片消息正式接入”
- 4.3 解决“动态流基础版”
- 4.4 解决“视频消息正式接入”
- 4.4S 解决“媒体链稳定性修正”
- 后续版本应继续沿着已稳定的 4.x 基线做扩展，不要回头改已归档主链
