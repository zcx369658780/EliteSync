# EliteSync 项目总览

更新时间：2026-05-11

## 项目名称

- `EliteSync`（慢约会）

## 产品定位

- 以“慢约会 / 深度匹配 / 资料真值 / 轻量转化”为核心的社交产品。
- 当前已经从 4.x 的基础能力补全阶段，完成 5.0-5.5 主链历史链路，进入 **5.6+ 玄学能力二次产品化与校准线**。

## 当前开发阶段

- `5.5` 已完成并发布到阿里云，当前最新对外发布版本为 `0.05.05 / 50500`；5.0-5.5 作为已完成历史链路保留，不重写。
- 当前主线切换到 `5.6+`：Soul 继续作为社交主链参考，测测作为玄学解释层参考，围绕慢约会、深度匹配、资料真值和轻量转化做解释层产品化。

## 核心差异化能力

- 资料真值优先：`profile/basic` 保存后由服务端重算 canonical truth。
- 星盘 / 八字 / 紫微等画像字段以服务端真值为准，前端缓存只兜底。
- 非官方四维人格问卷已正式产品化，但不冒充官方 MBTI。
- 图片 / 视频消息、动态、通知中心、1v1 真语音主链都已形成稳定主干。
- 版本中心、版本检查、发版脚本和下载包保持严格绑定。

## 当前技术路线概览

- 宿主：Android
- 前端：Flutter module
- 后端：Laravel API
- 发布：阿里云下载包 + 版本检查接口 + release script
- 数据原则：服务端真值优先，前端缓存只兜底

## 当前版本基线

- 当前最新对外发布版本：`0.05.05 / 50500`
- 上一条发布基线 / 历史发布链：`0.05.04 / 50400`
- 算法版本：沿用既有模块标记；matching marker 是否同步到 `5.4` 仍待确认，不作为当前正式口径。
- 当前 4.9 状态：已完成验收并收口为 `pass with observations`

## 5.x 当前方向

- 5.6：planning / boundary / calibration，不做 runtime implementation。
- 5.7：Match 关系解释层承接位。
- 5.8：Me 个人解释层承接位。
- 5.9：Chat 低压开场和轻追问承接位。
- 5.10：Settings / 用户控制与解释层治理承接位。
- 所有玄学解释、合盘解释、AI 追问保持 `derived-only / display-only / explanation layer`，不反写真值链。

## 一句话结论

- 这个项目已经从“能跑的社交原型”进入“5.0-5.5 主链已完成、5.6+ 转入玄学解释层产品化与校准”的正式规划阶段。
