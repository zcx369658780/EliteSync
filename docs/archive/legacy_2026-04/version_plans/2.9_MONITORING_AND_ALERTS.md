# EliteSync 2.9 Monitoring and Alerts

更新时间：2026-04-08

## 1. 最小可观测性

Beta 阶段先保证“看得见”，再谈“看得深”。

### 现有入口

- `GET /api/v1/app/health`
- `GET /api/v1/app/version`
- `app:dev:pair-outcome-metrics`
- 请求日志（Flutter 与后端均保留）

## 2. 建议关注的关键指标

- 应用启动成功率
- 首页首屏成功率
- 登录成功率
- 资料保存成功率
- 匹配揭晓成功率
- 消息页可达率
- 举报 / 封禁 / 审核动作成功率

## 3. 最小告警建议

- health 失败
- 登录 422 / 401 激增
- profile/basic 超时激增
- match/current 超时激增
- messages 超时激增
- 后台处理动作失败激增

## 4. 记录要求

- 任何线上故障必须记录：时间、影响面、回滚点、恢复时间
- 任何人工操作必须记录：操作者、对象、结果、备注
