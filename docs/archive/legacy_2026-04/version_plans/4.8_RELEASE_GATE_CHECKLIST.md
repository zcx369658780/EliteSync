# 4.8 Release Gate Checklist

更新时间：2026-04-28

## 目标

确保 4.8 进入 Alpha smoke 之前，版本链、下载包、版本中心与后端检查一致。

## 检查项

- [ ] `apps/android/app/build.gradle.kts` 的宿主版本号明确
- [ ] `apps/android/app/src/main/assets/changelog_v0.txt` 与当前版本链一致
- [ ] `services/backend-laravel/config/app_update.php` 与当前版本一致
- [ ] `GET /api/v1/app/health` 正常
- [ ] `GET /api/v1/app/version/check` 返回一致结果
- [ ] 版本中心显示与宿主版本一致
- [ ] 下载包地址可访问
- [ ] 安装后的 APK 版本与发布号一致
- [ ] 不使用旧 APK 截图冒充新版本
- [ ] 若本轮不发正式包，原因已写入 release gate note

## 回归要求

- [ ] 版本中心仍是现代 UI
- [ ] 版本中心没有退回旧布局
- [ ] 后端 version check 不与前端版本展示冲突

## 失败处理

如任一项失败：

1. 先写 blocker report
2. 再请 Claude 缩层
3. 只做最小修复
4. 修复后重新验证
