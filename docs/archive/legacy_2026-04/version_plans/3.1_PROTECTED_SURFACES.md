# EliteSync 3.1 保护面清单

更新时间：2026-04-09

## 必须保护的现有主链路

### 1. 注册 / 登录
- `POST /api/v1/auth/register`
- `POST /api/v1/auth/login`
- 登录态恢复

### 2. 资料与画像
- `POST /api/v1/profile/basic`
- `GET /api/v1/profile/astro/summary`
- `GET /api/v1/profile/astro/chart`
- `GET /api/v1/geo/places`

### 3. 匹配与消息
- `GET /api/v1/matches/current`
- `GET /api/v1/match/current`
- `GET /api/v1/match/history`
- `GET /api/v1/messages`
- 首聊 / 安全菜单 / 会话列表

### 4. 运营与治理
- `GET /api/v1/admin/*`
- 认证审核
- 举报处理
- 用户列表
- 封禁 / 解封 / 删除状态

### 5. 稳定性保护
- 首页首进
- about 健康检查
- version/check
- release gate
- regression baseline

## 3.1 新增应重点保护的 surfaces

### 后端
- 测试账号生成 / 重建命令
- 账号类型字段回填
- 状态发布写入链路
- 广场展示读取链路
- 管理员触发匹配接口

### 前端
- 发布状态入口
- 广场入口
- 管理员入口
- 账号类型与治理状态展示

## 结论

3.1 的改动如果触碰上述 surfaces，必须先有回归计划和回滚点，不能只凭 UI 验收直接推进。
