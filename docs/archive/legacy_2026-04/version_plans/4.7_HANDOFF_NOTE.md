# 4.7 交接说明

更新时间：2026-04-26

## 本轮交付

本轮 4.7 已完成的不是新业务扩张，而是稳定化与门禁收口：

- UI protected surfaces 已固定
- rollback / recovery policy 已固定
- UI baseline evidence 已补齐主要页面
- UI regression checklist 已补齐

## 当前交接重点

交接给后续 session 或顾问时，默认口径如下：

1. 不再默认允许 repo 级恢复
2. UI 现代化表面必须当作 protected surfaces 处理
3. 任何跨层 blocker 必须先写 blocker report
4. 恢复只能做 path-level / file-level / smallest-scope
5. smoke / regression 必须复查 UI baseline

## 当前不再做

- 不再用整仓恢复来“顺手修”构建 / 后端 / RTC / 媒体问题
- 不再把 UI 还原成旧版作为默认策略
- 不再把业务稳定化与 UI 回退混在同一次恢复里
