# EliteSync 4.0A 数据骨架说明

日期：2026-04-18

## 领域边界

- `conversation`：会话容器与参与者关系。
- `message`：既有文本消息主链继续沿用 `chat_messages`。
- `media`：媒体元数据、状态机与后续对象存储接入承载。
- `relationship`：关系事件预留骨架。
- `notification`：通知中心预留骨架，不承载正式业务流。

## 新增表

- `conversations`
- `conversation_members`
- `media_assets`
- `media_processing_jobs`
- `message_attachments`
- `user_relationship_events`

## 状态约定

- `media_assets.status`：`pending` / `uploading` / `uploaded` / `processing` / `ready` / `failed` / `blocked` / `deleted`
- `media_processing_jobs.status`：`queued` / `running` / `succeeded` / `failed`

## 路由骨架

- `GET /api/v1/conversations`
- `POST /api/v1/conversations`
- `GET /api/v1/conversations/{id}`
- `GET /api/v1/media`
- `POST /api/v1/media`
- `GET /api/v1/media/{id}`

## 约束

- 以上骨架只用于 4.0A 领域边界和未来接入预留。
- 任何对象存储、队列、缓存、上传协议的实作，应放到后续 4.0B / 4.0C / 4.0D。
