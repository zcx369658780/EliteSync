# EliteSync 3.7 风险评审

更新时间：2026-04-12  
适用范围：3.7 stage 0

## 1. 结论

3.7 属于高风险版本。原因不是新增页面本身，而是它要把关系维度和时间维度引入产品；这些能力最容易把 derived-only 误写成 canonical truth，或者把页面做成无法维护的高级后台。

## 2. 高风险清单

### 2.1 canonical truth 污染风险

- 合盘 / 对比盘 / 行运 / 返照如果没有严格标注字段角色，很容易被后续读取当成新真值。
- `engine_info`、`metadata`、解释卡字段如果命名不清，容易被误当成 profile truth。

### 2.2 旧入口回归风险

- 3.6 的路线工作台、版本中心、资料页、星盘页都必须保留。
- 若高级入口布局过重，可能会挤压旧入口首屏或破坏导航预期。

### 2.3 状态与缓存风险

- 高级上下文一旦进入 provider / cache，如果没有回退策略，容易污染旧缓存读取。
- 双人上下文和时间上下文都属于新增上下文，必须与现有 natal context 隔离。

### 2.4 UI 过载风险

- 合盘 / 对比盘 / 行运 / 返照天然信息密度高。
- 如果把解释卡、摘要卡、主图、样例集全塞到首屏，移动端会失控。

### 2.5 验收链缺失风险

- 本版强制要求 Claude 做架构边界审查，Gemini 做 UI / 可读性审查。
- 如果审查链缺失，就不能把 3.7 宣布为正式归档版本。

## 3. 潜在回归点

- `POST /api/v1/profile/basic`
- `GET /api/v1/profile/astro/summary`
- `GET /api/v1/profile/astro/chart`
- `POST /api/v1/profile/astro`
- `POST /api/v1/profile/astro/render`
- 资料主页 / 资料编辑页
- 星盘主页面 / 盘面设置页
- 设置中心 / 版本中心
- `chart_data`、`engine_info`、`metadata` 的旧字段读取
- `route_mode` 的 display-only 边界

## 4. 回滚建议

1. 保留 feature flag，默认关闭高级入口。
2. 任何新增高级接口都必须有 fallback 到 3.6 行为。
3. 任何新增展示字段都必须能在旧客户端上被忽略。
4. 保留样例矩阵与差异报告，避免回滚后无法解释偏差。

## 5. 兼容性提示

- 高级结果只能做 derived / display / advanced context。
- 不得把对方资料、时间上下文或关系摘要回写到 canonical natal truth。
- 所有新增接口应优先 additive，不删旧字段，不破坏旧读取路径。

## 6. 阶段门禁

- 阶段 0：先冻范围，再谈实现。
- 阶段 1：先 Claude 审 schema / truth boundary，再写 UI。
- 阶段 2 / 3：必须让 Gemini 审可读性与信息层级。
- 阶段 4：必须有样例矩阵与已知偏差文档。
- 阶段 5：必须补齐回归、截图、验收与留痕。
