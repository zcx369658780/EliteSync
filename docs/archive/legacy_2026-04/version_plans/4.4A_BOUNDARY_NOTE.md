# 4.4A 边界冻结说明

更新时间：2026-04-19

## 当前状态

`4.4` 的边界已经冻结，并且对应的视频消息主链已完成归档。  
本说明只用于保留版本范围、接口边界、保护面与不做项的稳定记录，不等同于功能实现说明。

## 版本定位

`4.4` 的定位是 `视频消息版`，目标是在聊天主链中补齐单视频消息能力。

允许的最小闭环：
- 聊天页选择单个视频
- 视频上传走既有 `/api/v1/media`
- 视频进入统一媒体状态机
- 视频作为 `attachment_ids` 绑定到 `chat_messages`
- 聊天线程展示视频气泡、封面或预览图
- ready 状态可点击播放或预览
- failed 状态可重试、替换或清除

## 本版承接的稳定基线

- `4.0`：媒体 / 存储 / 队列 / 缓存 / worker 最小闭环
- `4.1`：非官方四维人格问卷与轻量联动
- `4.2`：图片消息主链、附件绑定、预览与失败重试
- `4.3`：动态流基础版与治理挂点

## 允许复用的数据与状态面

优先复用，不新造平行主链：

- `media_assets`
- `media_processing_jobs`
- `message_attachments`
- `chat_messages`
- `conversations`
- `conversation_members`

### 视频最小元信息

视频能力只允许在既有 `media_assets` 统一媒体抽象上扩展最小 metadata，不允许新建平行视频资产主表作为主路径。

建议最小字段 / metadata：
- `media_type = video`
- `duration_ms`
- `width`
- `height`
- `thumbnail_key` 或等价封面引用
- `playback_variant` 或等价播放信息

## 允许复用的接口

- `POST /api/v1/media`
- `GET /api/v1/media/{assetId}`
- `POST /api/v1/messages`
- `GET /api/v1/messages`

如确有必要，只允许最小补充，不允许新建脱离主链的大量试验接口。

## 保护面

以下边界必须保持稳定，不得被视频能力误伤：

- `POST /api/v1/profile/basic`
- `GET /api/v1/profile/astro/summary`
- `GET /api/v1/profile/astro/chart`
- `GET /api/v1/app/health`
- `GET /api/v1/app/version/check`
- `app_release_versions`
- `chat_messages`
- `media_assets`
- `media_processing_jobs`
- `message_attachments`
- `status_posts`
- `status_post_likes`
- `4.2` 图片消息主链
- `4.3` 动态流主链
- `4.0` 媒体底座
- `3.x` 已归档真值链边界

## 明确不做

本版禁止：

- 视频动态
- 多视频消息
- 视频广场 / 推荐流
- 视频剪辑 / 滤镜 / 美颜 / 配乐
- RTC / 通话 / 在线状态系统
- 工业级分片上传完整版
- 大型媒体平台化改造
- 重写 `4.0` / `4.2` / `4.3` 的主链

## 验收前提

4.4A 的验收只看边界是否冻结，至少要做到：

1. 明确单视频消息的版本使命
2. 明确数据面 / 接口面 / 保护面
3. 明确状态链与失败态口径
4. 明确不做项
5. 明确后续实现不得越界到动态流或通话版

## 后续承接

`4.4A` 通过后，按顺序进入：

1. 后端视频上传 / 处理 / 消息绑定
2. Flutter 视频选择 / 气泡 / 播放页
3. 治理、缓存、回归保护
4. walkthrough 级证据包与 closeout
