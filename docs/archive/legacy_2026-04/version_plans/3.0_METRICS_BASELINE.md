# EliteSync 3.0 Metrics Baseline

更新时间：2026-04-08

当前基线版本：`0.02.09 / 209`

## 1. 指标目标

3.0 指标体系目标：

1. 能观测关键路径是否更顺
2. 能判断匹配解释是否更可行动
3. 能判断首聊与持续互动是否改善
4. 能支持周对周（WoW）对比，形成学习闭环

---

## 2. 核心漏斗指标（P0）

> 指标口径版本：`3.0-metrics-v1`（阶段 0 冻结，后续变更必须升版本并记录兼容说明）

1. 注册完成率
2. 新用户 24h 内问卷继续率
3. 新用户问卷完成率
4. 新用户 24h 内资料完善率
5. 匹配结果查看率
6. 双向确认率
7. 首聊启动率
8. 首聊后 24h 持续互动率
9. 举报率 / 拉黑率
10. 关键页面异常率
11. 弱网恢复成功率

## 4. 口径与采集方式

## 4.0 指标定义表（v1）

| 指标名 | 分子事件 | 分母事件/样本 | 观察窗口 | 去重键 | 备注 |
|---|---|---|---|---|---|
| 注册完成率 | `register_complete` | `register_start` | 当周 | `user_id` | 新用户口径 |

## 4.1 口径规则

1. 所有转化率必须定义分母窗口（例如“新用户 24h”）。
2. 同一指标在 3.0 全周期不可随意改口径；必须版本化。
3. 新增事件不得替换既有核心事件，仅可补充。
4. 事件字段必须可解释，避免只记“按钮点击”而无法归因。

## 4.2 基线采集（WoW）

1. 以周为单位输出固定观察窗口（W1/W2/W3...），统一使用北京时间（UTC+8）日切。
2. 每周输出：
   - 核心指标数值
   - 与上周对比（绝对值/相对值）
   - 本周主要改动点
   - 归因假设与待验证项
3. 保持同一过滤条件（渠道、版本段、用户分层）保证可比性。

## 5. 已知缺口与分阶段补齐计划

## 5.1 已知缺口

1. 部分前台行为事件仍未标准化
2. 匹配反馈闭环存在“本地反馈”与“运营学习”断点
3. 弱网恢复成功率在不同页面的定义尚未统一
4. 关键事件 owner 与复核节奏未完全固化

## 5.2 补齐计划

### Phase A（阶段 0~1）
- 冻结事件字典与字段口径
- 先打通首页、匹配、首聊三条核心路径事件

### Phase B（阶段 2~3）
- 补齐解释层动作与反馈分类事件
- 统一弱网恢复事件定义

### Phase C（阶段 4~5）
- 固化周报模板与归因标签体系
- 输出 3.0 期末指标总结与下一阶段建议

---

## 6. 验收标准

指标体系验收必须同时满足：

1. 关键路径事件可稳定采集
2. 周对周可比（口径不漂移）
3. 指标异常可定位到页面/动作层
4. Gemini 验收结论至少“有条件通过”
5. 未验证项与数据盲区显式记录
