# 4.6A 风险评审

更新时间：2026-04-21  
版本：`4.6 RTC / 通话基础设施版（语音优先）`

## 高风险点

### 1. 版本越界

风险：把 `4.6` 做成多人通话、直播、在线状态、推送平台大版本。  
控制：只做 1v1 语音优先最小闭环，不扩散。

### 2. 状态机漂移

风险：前后端通话状态不同步，UI 显示和真实信令不一致。  
控制：先收口 `rtc_sessions / rtc_session_events` 的状态迁移，再做视觉。

### 3. 误伤消息 / 通知主链

风险：为了通话改坏 `chat_messages`、`notifications` 或摘要语义。  
控制：通话层只做 additive 联动，不重写主链。

### 4. 权限与失败态不清晰

风险：麦克风 / 摄像头权限失败、网络失败、对方不可达直接暴露底层异常。  
控制：失败态必须产品化表达，不能裸异常。

### 5. 本地 / 云端边界失守

风险：本地数据库、临时 SQLite、测试快照污染远端后端。  
控制：本地只做前端；后端写库统一在阿里云端。

## 回滚建议

- 所有 RTC 路由与表结构优先可单独关闭
- 通话入口应能在不影响消息页的前提下隐藏
- 通话通知应可独立停用，不影响现有通知中心

## 兼容性提示

- 4.0 / 4.2 / 4.4 / 4.5 归档主链必须保持可达
- `app/health` 与 `app/version/check` 不得因 RTC 改坏
- 旧版本客户端若不识别 RTC 入口，应优雅退化
