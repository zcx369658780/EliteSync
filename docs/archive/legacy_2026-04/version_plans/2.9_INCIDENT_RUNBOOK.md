# EliteSync 2.9 Incident Runbook

更新时间：2026-04-08

## 1. 首页白屏

1. 检查 `/api/v1/app/health`
2. 确认是否装错了 module 产物
3. 检查 `home_remote_data_source.dart` 的短超时兜底是否仍在
4. 必要时回滚到最近稳定提交

## 2. 登录异常

1. 检查 auth 服务是否正常
2. 检查 401 / 422 比例
3. 检查测试账号是否失效
4. 先恢复账号或配置，再看回滚

## 3. 资料 / 匹配 / 消息挂起

1. 检查是否命中短超时兜底
2. 检查对应接口是否慢返回
3. 检查 about / health 卡是否正常
4. 必要时切换到低流量状态

## 4. 后台治理异常

1. 检查举报 / 审核 / 用户列表接口
2. 检查权限是否正确
3. 检查 mock 开关是否误入生产
4. 必要时回滚后台变更

## 5. 复盘要求

- 事件描述
- 影响范围
- 恢复时间
- 回滚点
- 后续预防动作
