# 2.6 今晚玄学可视化执行方案

日期：2026-04-01

> 目标：在不扩大 2.6 主线范围的前提下，优先完成“玄学指标可视化”收口。
>
> 本方案只处理前端展示、接口契约确认、以及与可视化强相关的后端输出补齐。
> 不碰数据库备份主线、不重写算法主流程、不引入新的计算入口。

---

## 1. 评估结论

### 1.1 2.6 当前状态

2.6 的主计划还没有彻底完成，当前已落地的是：

- 数据库双备份已完成过一轮实际运行
- 资料保存与画像重算链路已接通
- 出生地、出生时间、真太阳时、八字、紫微、星象已有基础联动
- 星盘画像页已能展示出生地、经纬度、真太阳时和部分计算元信息

但 2.6 的原始主线中仍有未彻底收口的部分：

- 备份/恢复/告警的运维闭环还需要继续打磨
- 画像重算幂等与观测还没完全形成正式门禁
- 前端玄学可视化仍偏“字段展示”，还没完整升级到“卡片化 / 图表化 / 详情化”

### 1.2 今晚目标

今晚只做一件事：

> **把玄学指标可视化做到可验收。**

也就是把现在的“能看见字段”升级为：

- 有结构
- 有层级
- 有图表
- 有稳定详情页
- 有更新状态

---

## 2. 多 Agent 评估结果摘要

### 2.1 dependency-mapper

当前可视化主链路是：

- `ProfilePage`
- `EditProfilePage`
- `AstroProfilePage`
- `profile/basic`
- `profile/astro`
- `user_astro_profiles`

可视化应当围绕服务端返回的 canonical 结果做展示，不再在前端做计算分歧。

### 2.2 risk-reviewer

最大风险点：

- 资料保存后，服务端结果和前端缓存可能不一致
- 出生地搜索一旦 fallback，不应让 UI 误判为“数据没回来”
- 图表页如果直接读取本地缓存，会把旧结果伪装成新结果

### 2.3 test-planner

今晚最小验证集：

1. 打开“我的”页，能进入八字 / 星盘 / 紫微 / 总览页
2. 修改出生时间 / 出生地后，页面能看到“更新中”或等价提示
3. 星盘页能看到：
   - 出生时间
   - 出生地点
   - 经纬度
   - 真太阳时
   - 位置修正
   - 位置签名
   - 八字精度 / 置信
   - 西占精度 / 置信
4. 八字页至少有：
   - 总览卡
   - 五行图或等价图表
   - 四柱表
5. 紫微页至少有：
   - 命宫 / 身宫
   - 简化宫盘或等价结构

### 2.4 architecture-guardian

边界要求：

- UI 只负责展示
- 状态层只负责拉取与缓存，不得成为真值来源
- 算法结果必须来自后端
- 图表和轮盘可用 Flutter 自绘或轻量图表库，但不要把计算逻辑搬进前端

---

## 3. 今晚可执行拆解

### P1：玄学总览页

目标：

- 在“我的”页增加一个统一入口
- 显示当前画像状态、最近更新时间、模块概览

验收：

- 用户一眼能看懂当前有没有画像
- 能区分八字 / 星盘 / 紫微 / 生肖 / 星座

### P2：八字详情页

目标：

- 做成“总览卡 + 五行图 + 四柱表 + 标签摘要”

验收：

- 至少能看到五行分布
- 能看到四柱文本
- 能看到真太阳时与来源提示

### P3：星盘详情页

目标：

- 简化轮盘 + 三层摘要
- 保留当前出生地 / 经纬度 / 真太阳时 / 位置修正 / 元信息

验收：

- 能直观看到这次计算是否使用了出生地
- 能看到精度、置信、位置签名

### P4：紫微详情页

目标：

- 命宫 / 身宫高亮
- 简化宫盘
- 宫位摘要

验收：

- 能看到命宫、身宫、引擎、精度、置信

### P5：资料更新状态

目标：

- 保存出生时间 / 出生地点后显示“更新中”
- 完成后自动刷新摘要

验收：

- 不再出现“改了资料但页面像没变”的错觉

### P6：最小回归清单

目标：

- 不破坏原有资料保存、图片入口、返回刷新

验收：

- 编辑页可保存
- 星盘页可刷新
- 不空白

---

## 4. 本轮明确不改什么

- 不改数据库备份主线
- 不改后端算法核心逻辑
- 不重写紫微 / 八字 / 西占引擎
- 不新增新的计算入口
- 不引入第二套真值来源
- 不恢复 MBTI / 性格测试入口

---

## 5. 推荐执行顺序

1. 先做玄学总览页
2. 再做八字详情页
3. 再做星盘详情页
4. 再做紫微详情页
5. 最后补资料更新状态
6. 跑最小回归清单

---

## 6. 今晚交付标准

今晚完成后，至少满足：

- “我的”页可作为玄学入口中心
- 八字 / 星盘 / 紫微 三个详情页可稳定打开
- 星盘页能证明出生地、经纬度、真太阳时确实进入计算链路
- 修改资料后前端有明确更新状态
