# EliteSync 2.9 风险评审

更新时间：2026-04-07

## 1. 最高风险面

1. 资料保存与画像重算链路
2. 地点搜索与百度 Web 服务
3. 匹配 / 聊天主链路
4. 举报 / 拉黑 / 审核治理链路
5. 发布脚本 / 回滚脚本 / smoke / regression
6. 环境变量与版本检查配置

---

## 2. 潜在回归点

### 2.1 资料链路
- 保存失败导致头像/出生地/生日回读异常
- summary / chart 接口返回空白或字段缺失
- 前端缓存覆盖服务端真值

### 2.2 匹配链路
- MatchPortal / Result / Feedback 页面状态机漂移
- 解释文案/按钮状态导致主流程中断
- 反馈本地闭环误伤 canonical 数据

### 2.3 治理链路
- 举报 / 拉黑入口不可达
- 用户状态反馈与后台状态不一致
- 误封 / 误解封 / 申诉状态显示不清

### 2.4 发布链路
- smoke 账号漂移
- regression 账号失效
- 版本检查与 APK 下载地址不一致
- 阿里云回滚点不可用

## 4. 风险控制建议

1. 所有新工作必须先通过 regression baseline 的最小回归项
2. 所有 beta 上线改动必须保留 smoke / rollback 演练记录
3. 所有后台功能必须明确 dev / beta / production 的入口区别
4. 所有前台治理入口必须可被真机验证
5. 所有新字段必须先明确是否属于 canonical 真值
