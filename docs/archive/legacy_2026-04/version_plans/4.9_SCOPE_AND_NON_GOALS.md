# 4.9 范围与非目标

更新时间：2026-04-28

## 本轮范围

4.9 是 **测试前治理、限流、监控、发布链强化版**。本轮范围只覆盖以下工程化收口：

1. 通知中心工程 slug 降噪；
2. 关键接口限流矩阵；
3. 媒体 / 队列 / worker 可观测性；
4. 通知 / 事件 / 回流可观测性；
5. RTC / LiveKit / heartbeat / join-info 可观测性与 runbook；
6. release gate 强化与版本链复验；
7. 数据库备份、恢复与云端操作演练说明；
8. 健康检查、日志索引、Alpha readiness dashboard 文档；
9. UI protected surfaces 回归巡检；
10. 文档收口、验收摘要与交接。

## 本轮非目标

4.9 明确不做：

1. 新匹配算法；
2. 新星盘 / 八字 / 紫微算法；
3. 人格问卷大改；
4. 多人 RTC；
5. 视频通话增强；
6. 直播、语聊房、在线状态平台；
7. 内容社区化改造；
8. 运营后台平台化；
9. 付费 / 会员 / 商业化；
10. 消息主链重构；
11. 媒体主链重构；
12. 通知主链重构；
13. RTC 主链重构；
14. 动态流主链重构；
15. repo 级回滚；
16. 本地数据库覆盖远端；
17. 为了监控或测试方便制造 mock 成功态；
18. debug / internal / mock / scaffold 文案进入正式用户页面；
19. UI 现代化基线回退；
20. 把 release gate 当成可跳过项。

## 核心约束

- UI protected surfaces 继续作为硬约束；
- canonical truth 仍以服务端为准；
- cache 只能兜底；
- 跨层问题必须先 blocker report，再请 Claude 缩层；
- 修构建只动构建层，修后端只动后端层，修 RTC 只动 RTC 层，修 UI 只动 UI 层。

## 说明

4.9 的目标不是再证明“能不能跑”，而是把 4.8 已验证的主链变成测试阶段可持续使用的工程化基线。
