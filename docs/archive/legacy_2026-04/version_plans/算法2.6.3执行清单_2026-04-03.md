# EliteSync 2.6.3 执行清单（玄学盘面专业化收口版）

更新日期：2026-04-03

本文档由 `算法2.6.3开发规划与gemini监督方案.md` 拆解而来，目标是把 2.6.3 的工作压成可执行、可验收、可回归的任务清单。

本版本只做三件事：
1. 紫微盘中宫升级：中宫显示八字身份锚点；
2. 星盘页专业化：黑底、完整盘面、文本下沉；
3. 盘面设置页：用户可控制线条和显示项。

---

## 0. 版本目标与边界

### 0.1 版本目标
- 把星盘页做成真正的主视觉页，而不是缩略图页；
- 把紫微页做成真正的标准方盘，而不是卡片拼贴页；
- 把盘面显示偏好独立出来，避免界面过密、不可控；
- 保持服务端 canonical 真值不变，前端只做展示和偏好控制。

### 0.2 明确不做
- 不改八字 / 星盘 / 紫微 canonical 算法；
- 不改数据库 schema，除非确实需要保存展示偏好；
- 不引入 2.7 的多场景匹配和调权；
- 不在前端新增第二真源；
- 不把工程参数直接暴露在主视觉。

---

## 1. 任务拆分总览

建议按 6 个阶段执行：

1. `P0` 基线冻结与接口确认
2. `P1` 紫微中宫八字锚点
3. `P2` 星盘黑底主视觉与 SVG 收口
4. `P3` 盘面设置页
5. `P4` 全局去框化与信息降噪
6. `P5` 回归验收与发布准备

---

## 2. P0 基线冻结与接口确认

### 目标
确认 2.6.3 的改动只作用于展示层和偏好层，不影响画像真值与保存重算链路。

### 任务
1. 确认当前稳定接口：
   - `POST /api/v1/profile/basic`
   - `GET /api/v1/profile/astro/summary`
   - `GET /api/v1/profile/astro/chart`
   - `GET /api/v1/geo/places`
2. 确认当前 Flutter 页面职责：
   - `ProfilePage`
   - `EditProfilePage`
   - `AstroOverviewPage`
   - `AstroBaziPage`
   - `AstroNatalChartPage`
   - `AstroZiweiPage`
3. 冻结真值原则：
   - summary/chart 继续分流；
   - 前端缓存只做 fallback；
   - 星盘设置只影响展示，不影响 canonical。

### 验收标准
- 能明确说出这轮只改哪些页面、哪些组件；
- 能明确说出哪些字段不能被前端改成真值；
- 能明确回滚点。

## 3. P1 紫微中宫八字锚点

### 目标
让紫微地盘中宫成为“身份中枢”，显示八字四柱与命理身份锚点。

### 涉及文件
- `apps/flutter_elitesync_module/lib/features/profile/presentation/pages/astro_ziwei_page.dart`
- `apps/flutter_elitesync_module/lib/features/profile/presentation/widgets/standard_ziwei_grid.dart`

### 任务
1. 中宫区域显示八字四柱：
   - 年柱
   - 月柱
   - 日柱
   - 时柱
2. 中宫显示命理身份锚点：
   - 命宫
   - 身宫
   - 八字日主
3. 保持中宫简洁：
   - 不显示真太阳时
   - 不显示位置修正
   - 不显示 engine / precision / confidence
4. 中宫视觉要高于弱标签：
   - 八字信息应是主内容
   - 命宫 / 身宫作为弱辅助说明

### 设计原则
- 中宫是“紫微 + 八字”的统一锚点；
- 中宫不是调试面板；
- 中宫只承载结果型摘要。

### 验收标准
- 中宫能直接看到四柱八字；
- 命宫 / 身宫与四柱之间有清晰层级；
- 中宫不再出现工程态字段。

---

## 4. P2 星盘黑底主视觉与 SVG 收口

### 目标
把星盘页变成专业软件式主视觉：黑底、完整圆盘、文本下沉、盘面占据页面视觉中心。

### 涉及文件
- `apps/flutter_elitesync_module/lib/features/profile/presentation/pages/astro_natal_chart_page.dart`
- `apps/flutter_elitesync_module/lib/features/profile/presentation/widgets/natal_chart_svg_card.dart`
- `services/api/app/services/astro.py`

### 任务
1. 星盘主视觉改为黑底：
   - 盘面区域使用深色背景；
   - 圆盘居中显示；
   - 进入页面第一眼看到的是盘，而不是说明文字。
2. SVG 渲染收口：
   - 优先使用纯圆盘 / wheel-only 输出；
   - `InteractiveViewer` 保留；
   - 默认缩放和适配要保证首屏完整可见。
3. 文本下沉：
   - 太阳 / 月亮 / 上升摘要放在盘下；
   - 出生地、经纬度、真太阳时、宫位制、黄道体系等技术参数继续下沉；
   - 行星摘要、宫位摘要、相位摘要改成文本化、低干扰展示。
4. 不再使用缩略图心智：
   - 星盘页不是“列表 + 小图”；
   - 星盘页必须是“盘面主视觉 + 辅助文本”。

### 设计原则
- 盘面优先，文本辅助；
- 首屏必须是完整星轮；
- 盘面的存在感高于说明区。

### 验收标准
- 打开页面第一眼是完整大盘；
- 盘面不会被相位表拖成小图；
- 黑底与盘面线条对比清楚；
- 文本区不抢主视觉。

## 5. P3 盘面设置页

### 目标
把“线条太多、太密集、用户不可控”的问题收口成可配置显示偏好。

### 涉及文件
- 新增 `ChartDisplaySettingsPage`
- 星盘页右上角入口
- 本地偏好存储组件

### 任务
1. 新增独立设置页：
   - 从星盘页右上角进入；
   - 只服务盘面显示控制。
2. 设置项 MVP：
   - 主题：深色盘面
   - 行星：太阳 / 月亮 / 水星 / 金星 / 火星 / 木星 / 土星
   - 虚点：上升 / 天顶 / 北交
   - 相位：0° / 60° / 90° / 120° / 180°
   - 容许度：按预设值调节
3. 设置存储：
   - 首版优先本地存储；
   - 不进入 canonical；
   - 修改后即时回显。
4. 容错：
   - 关闭某个显示项后不得白屏；
   - 恢复默认后必须立即生效。

### 验收标准
- 右上角设置入口可达；
- 设置改变能立即影响盘面；
- 关闭项后不影响页面稳定；
- 设置页关闭后偏好保留。

---

## 6. P4 全局去框化与信息降噪

### 目标
减少蓝色胶囊、减少工程态标签、提升排版呼吸感。

### 涉及文件
- `apps/flutter_elitesync_module/lib/features/profile/presentation/widgets/astro_profile_sections.dart`
- `apps/flutter_elitesync_module/lib/features/profile/presentation/pages/astro_ziwei_page.dart`
- `apps/flutter_elitesync_module/lib/features/profile/presentation/pages/astro_natal_chart_page.dart`
- `apps/flutter_elitesync_module/lib/features/profile/presentation/pages/astro_bazi_page.dart`

### 任务
1. 所有页面减少 AstroPill 的滥用：
   - 仅保留命宫 / 身宫 / 核心五行等少数高价值标签；
   - 其他参数改成文本、KeyValue 行、轻量说明。
2. 星空 / 紫微 / 八字页面统一去框化：
   - 减少厚重卡片嵌套；
   - 减少重复边框；
   - 减少“后台管理系统感”。
3. 工程词下沉到诊断页：
   - `precision`
   - `confidence`
   - `engine`
   - `location_shift`
   - `true_solar_time`
   - `location_source`
   这些不再出现在主视觉。

### 验收标准
- 页面蓝色胶囊密度明显下降；
- 主视觉更像专业分析软件；
- 技术参数只出现在诊断区或折叠区。

## 7. P5 回归验收与发布准备

### 目标
确保 2.6.3 改动不破坏旧功能，并可安全交接。

### 最小 smoke
1. 登录后进入 `AstroOverviewPage`
2. 进入 `AstroZiweiPage`
3. 进入 `AstroNatalChartPage`
4. 进入 `AstroBaziPage`
5. 打开星盘设置页
6. 关闭 / 开启部分显示项后返回星盘页

### 回归点
1. 资料保存仍然可触发重算；
2. summary/chart 分流仍然成立；
3. 星盘页不会拖死其它玄学页；
4. 紫微中宫八字显示不丢；
5. 设置页不影响真值；
6. 旧缓存不会变成新真值。

### 手工验收步骤
1. 打开八字页，看四柱是否完整；
2. 打开紫微页，看中宫是否有八字与命宫/身宫；
3. 打开星盘页，看是否黑底完整大盘；
4. 进入设置页切换显示项，看盘面是否即时变化；
5. 退出登录再登录，确认页面不白屏。

### 验收标准
- 页面结构符合 2.6.3 定位；
- 不引入新的加载死锁；
- 不破坏资料保存和画像回显；
- 不新增第二真源。
