更新时间：2026-04-12

## 1. 现状说明

本轮理论上应并行调用：

- dependency-mapper
- risk-reviewer
- test-planner
- architecture-guardian

但当前环境已达到 agent 线程上限，四个并行评审无法再额外起新线程。

因此，本轮先基于代码检索与现有稳定基线完成 stage 1 的架构拆解，并将可归档的架构边界结论记录如下。

## 2. 可确认的架构结论

### 2.1 现有可复用基线

- `route_mode` 已作为 additive display context 存在。
- `engine_info` / `metadata` 已是可扩展元信息层。
- `chart_data` 仍然是 natal truth 的核心输出结构。
- Flutter 侧已经通过 `route_mode`、workbench prefs 和 display prefs 把“真值 / 路线 / 显示”做了分层。

### 2.2 3.7 的新增边界

- 合盘 / 对比盘 / 行运 / 返照必须进入 advanced context，而不是写进 canonical truth。
- 高级摘要卡只能是 derived-only / display-only。
- 旧接口和旧缓存必须继续可读。

### 2.3 不可越线的地方

- 不能把对方资料当成 canonical truth。
- 不能把 target datetime 写成 natal truth。
- 不能删除 `chart_data`、`engine_info`、`metadata` 的兼容读取路径。
