# 3.6 已知偏差

更新时间：2026-04-12  
适用范围：3.6 Stage 4 多样例对照 / 差异归档  
报告性质：`derived-only / display-only`

## 1. 目的

本文件记录 3.6 里已经明确知道、但不应当被误判成 bug 的偏差。

这些偏差主要分三类：

1. 路线解释层与 canonical truth 的边界差异。
2. 路线能力矩阵与样例集的解释差异。
3. 当前仍未进入 3.6 的能力项。

---

## 2. 已知偏差清单

### 2.1 路线解释只服务于 display / derived

- `route_mode` 只表示路线与解释上下文。
- 路线切换不会改写 `profile/basic` 真值。
- 路线差异报告只负责展示差异，不负责定义真值。

### 2.2 多样例样本仍然是固定基线

- 当前样例集只包含基线样例和现代扩展样例两份固定样例。
- 这些样例用于回归和归档，不代表生产库中的完整样本覆盖。

### 2.3 现代路线在扩展样例里会显著增加相位

- 在 `dense-modern` 样例中，现代路线会比标准路线多出扩展五分相位 / 其他非标准相位。
- 这是路线模板的预期行为，不是计算错误。

### 2.4 古典路线维持收敛阅读

- 古典路线会收紧点位与相位。
- 这会让可见数显著低于标准 / 现代路线。
- 这种差异是路由层口径，不应误判为数据缺失。

### 2.5 当前还未进入 3.6 的能力项

- 合盘
- 行运 / 三限 / 次限 / 月返 / 日返
- 法达 / 小限 / 大限 / 流年
- canonical 引擎全面替换

---

## 3. 归档原则

- 若差异来自路线选择与工作台预设，优先视为 derived-only。
- 若差异来自 canonical truth 保存与读取，应优先视为回归问题。
- 若差异仅存在于说明层或样例集，不应升级为算法 bug。

---

## 4. 结论

3.6 当前阶段的已知偏差都已纳入路线解释层的预期范围。
这些偏差不会阻止 3.6 继续推进，也不会影响 canonical truth 的稳定性。
