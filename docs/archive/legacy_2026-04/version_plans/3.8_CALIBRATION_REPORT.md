# EliteSync 3.8 校准报告

更新时间：2026-04-16  
版本状态：stage 4 校准完成版  
适用范围：参数工作台、校准样例、偏差归档、回归锚点

## 1. 报告性质

- 报告性质：derived-only / display-only / advanced-context
- 说明：本报告用于参考样例校准、差异归档与回归锚定
- 禁止事项：不得把本报告写成新的 canonical truth 源

## 2. 校准目标

- 用固定样例验证 3.8 的参数工作台是否会按预期改变可见点位与可见相位
- 区分路线差异、参数差异、显示层差异与真实 bug
- 为 stage 5 的最终回归和截图归档提供稳定基线

## 3. 固定样例集

### 3.1 baseline

- 样例标识：`baseline`
- 样例名称：`基线样例`
- 用途：3.8 的主校准基线
- 路线结果：
  - 标准：20 点位 / 9 相位
  - 古典：11 点位 / 4 相位
  - 现代：20 点位 / 10 相位

### 3.2 dense-modern

- 样例标识：`dense-modern`
- 样例名称：`现代扩展样例`
- 用途：验证现代路线在更密集样例下的可见差异
- 路线结果：
  - 标准：20 点位 / 8 相位
  - 古典：11 点位 / 5 相位
  - 现代：20 点位 / 12 相位

## 4. 差异摘要

- baseline 的现代路线比标准路线多 1 条相位，属于预期 display delta
- dense-modern 的现代路线比标准路线多 4 条相位，属于样例密度提升后的预期差异
- 古典路线在两组样例中都收紧到更少点位与更少相位，符合“核心点位 + 主相位”的路线口径
- 所有差异均保持在 derived-only / display-only 范围内

## 5. 已知偏差归档

- 路线模板切换会影响可见点位 / 相位统计，但不会改变 canonical truth
- 参数工作台中的黄道制 / 宫位制当前仍是工作台口径，尚不直接改写引擎真值
- 校准样例的差异是预期的路线与密度差异，不应误判为 bug

## 6. 校准结论

- baseline 与 dense-modern 两组样例均可稳定复算
- 路线模板切换、参数面板切换与样例差异保持一致
- 3.8 的参数工作台深度与校准闭环已达到 stage 4 归档要求
- 现在可以进入 stage 5：最终回归与归档
