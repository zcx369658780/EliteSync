# 4.7 给 GPT 顾问的交接稿

更新时间：2026-04-26

## 交接目的

4.7 不是继续加功能，而是把“现代 UI 必须保护”这件事变成正式门禁。

## 交接要点

### 1. 事故已被制度化处理

此前一次跨层恢复把 UI 回退到了旧形态。现在已经通过文档把这件事定为流程事故，并新增：

- protected surfaces
- rollback / recovery policy
- UI baseline evidence
- regression checklist

### 2. 当前版本判定

4.7 建议按 `pass with observations` 收口。

观察项只保留：

- 独立 live-call runtime state 可再补充
- 后续恢复必须继续遵守 protected surfaces

### 3. 后续不允许的动作

- repo 级回滚
- 未 checkpoint 的恢复
- 跨层问题时顺手改 UI
- 用旧基线覆盖现代 UI

### 4. 后续允许的动作

- 仅路径级、文件级最小恢复
- 先 blocker report，再请求定点分析
- 先保护 UI，再修构建 / 后端 / RTC / 媒体问题

## 建议顾问检查点

请顾问重点确认：

1. 当前 UI baseline evidence 是否足够覆盖现代布局
2. protected surfaces 是否列得足够完整
3. rollback / recovery policy 是否足以防止再次回退
4. 4.7 是否可以冻结为 `pass with observations`
