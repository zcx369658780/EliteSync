# 4.6F 风险评审

更新时间：2026-04-23

## 高风险点

1. **把 RTC 状态机误当音频引擎**
   - 风险：继续只做页面跳转，看起来像语音，实际上还是没有声音
   - 处理：明确引入 LiveKit 媒体层

2. **破坏现有通话页回退**
   - 风险：通话页接入媒体后又把返回消息页链路搞坏
   - 处理：保留显式回消息页的路由收口

3. **把 LiveKit 变成平台化重构**
   - 风险：一不小心扩成多人、直播、在线状态平台
   - 处理：只做 1v1 语音最小闭环

4. **后端 token 误放开权限**
   - 风险：非参与者也能拿到 room token
   - 处理：严格校验 rtc session 参与者身份

5. **本地 / 云端环境混用**
   - 风险：token 配置或 LiveKit 地址在本地误写到生产
   - 处理：配置独立、默认无配置时安全降级

## 回滚点

- 保留现有 RTC 状态机可独立运行
- LiveKit 接入失败时应可退回到“状态机 + 通知”骨架
- 不动 4.0 / 4.2 / 4.4 / 4.5 主链
