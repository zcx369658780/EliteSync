# EliteSync 4.0D Flutter 附件上传状态骨架

日期：2026-04-18

## 已落地

- `ChatRoomPage` 中加入了附件上传状态骨架卡，明确展示待上传、上传中、处理中、失败、重试、已完成等状态。
- `MessageAttachmentShelf` 已升级为统一状态卡组件，而不是散乱的说明文字。
- `MessageInputBar` 保留附件按钮，作为未来真实文件选择的接入点。
- 现有聊天发送主流程、会话列表和安全菜单保持不变。

## 当前骨架状态

- 卡片上可以直接看到当前状态与可切换的状态芯片。
- 失败态保留重试入口，便于后续接真实上传重试逻辑。
- 不引入额外导航或复杂状态流，仍然挂在当前聊天页内。
- 保持和当前 EliteSync 聊天页风格一致，不替换整套 UI 结构。

## 边界

- 不在 4.0D 里实现完整图片/视频消息产品。
- 不改 3.x 高级时法与画像页面。
- 不把附件状态当成 canonical truth。
- 不做复杂动效和多页面上传流程。
