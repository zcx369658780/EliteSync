# 4.9 基线起点说明

更新时间：2026-04-28

## 真实 checkpoint

- 当前 4.8 已按 `pass with observations` 冻结
- 当前 4.9 定位为 **测试前治理、限流、监控、发布链强化版**
- 当前仓库保留 4.7 / 4.8 的 UI protected surfaces、rollback / recovery policy 与 smoke / release gate 经验
- 当前处理方式：保留现有工作树，在 4.8 冻结基线上继续做 4.9 的工程化收口

## 版本定位

4.9 不做新业务功能，不重开 4.8，不扩 RTC，不重构 UI。

本版本目标是把 4.8 已经证明能跑通的主链，转化为测试阶段可监控、可限流、可排障、可恢复、可发布、可交接的工程化基线。

## 当前承接基线

承接以下已完成或已冻结基线：

- `3.9`：高级时法框架、细粒度解释层、真值链保护已收口
- `4.0`：媒体 / 存储 / 队列 / 缓存 / worker 基础设施已收口
- `4.1`：非官方四维人格问卷已收口
- `4.2`：图片消息主链已收口
- `4.3`：动态流基础版已收口
- `4.4`：视频消息版已收口
- `4.4S`：媒体链稳定性修正版已收口
- `4.5 / 4.5E`：通知与社交转化增强版已收口
- `4.6P`：1v1 真语音可听闭环已收口
- `4.7`：UI protected surfaces、rollback / recovery policy 与测试前质量门禁已收口
- `4.8`：Alpha Smoke 与测试前真实路径复验版已按 `pass with observations` 收口

4.9 的第一原则是：

> 不把任何已归档版本重新当成待开发版本，不把 4.8 observation 变成重开 4.8 的理由，而是在 4.9 中集中处理测试前工程化收敛项。

## 本轮起点规则

1. 不做 repo 级回滚。
2. 不把旧 UI 或旧 APK 当成当前可恢复目标。
3. 不把 4.8 的 observation 当成 4.8 未完成。
4. 不在本地执行后端写库、数据库迁移、备份、恢复。
5. 所有跨层 blocker 先写 blocker report，再做最小修复。
6. UI protected surfaces 继续作为硬保护面。

## 4.9 的工作方向

- 通知中心工程 slug 降噪
- 关键接口限流策略
- 媒体 / 通知 / RTC / 版本链最小可观测性
- release gate 与版本链复验
- 云端后端操作 runbook
- 数据库备份与恢复演练
- 跨层 blocker 流程固化
- UI protected surfaces 回归巡检
- 文档收口与交接

## 当前结论

4.9 的重点不是新功能，而是把已经打通的主链变成：

- 可测试
- 可复验
- 可恢复
- 可保护 UI 基线
- 可继承到 5.0
