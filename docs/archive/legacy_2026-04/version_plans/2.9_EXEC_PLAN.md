# EliteSync 2.9 执行计划（Beta 上线准备版）

更新时间：2026-04-07

## 1. 版本目标

2.9 的目标不是扩新功能，而是把 EliteSync 推到可小规模接入真实用户的 Beta 准备状态。

### 2.9 P0
- 测试体系补齐
- 性能与稳定性补齐
- 安全与合规补齐
- 灰度与运维补齐
- 发布门禁固化

### 2.9 P1
- 匹配解释回归保护
- 运营可用性增强
- 可观测性增强

### 2.9 P2
- 不纳入主线的算法/重构/复杂实验平台

---

## 2. 计划前置边界

### 明确不做
- 不扩张 2.7 / 2.8 的前台功能范围
- 不重写资料 canonical 链路
- 不大改数据库 schema
- 不把前端缓存当真源
- 不在主线上做大规模算法重写

### 必须保护
- `POST /api/v1/profile/basic`
- `GET /api/v1/profile/astro/summary`
- `GET /api/v1/profile/astro/chart`
- `GET /api/v1/geo/places`
- `MatchPortalPage` / `MatchResultPage` / `MatchFeedbackPage`
- `ChatRoomPage`
- `AdminDashboardPage` / `AdminModerationPage` / `AdminVerificationPage` / `AdminUsersPage`

---

## 3. 阶段 1 任务拆解

### 3.1 冻结边界
- 明确 2.9 非目标
- 固化 Protected Surfaces
- 固化 Beta 最小回归清单

### 3.2 真实链路联调
- 主业务链路：注册 / 登录 / 问卷 / 匹配 / 确认 / 聊天
- 资料链路：编辑资料 / 地点搜索 / 保存 / 画像重算
- 治理链路：举报 / 拉黑 / 认证审核 / 用户状态

### 3.3 风险补强
- 接口契约测试
- smoke / regression 扩充
- 回滚演练
- 观测与埋点最小集

---

## 4. 验收标准

### 必须通过
- 资料链路能保存并回读
- summary / chart 正常展示
- 关键页面不空白
- 举报 / 拉黑 / 审核 / 用户治理链路可到达
- 发布 / 烟测 / 回滚脚本可执行

### 可延后
- 高级性能优化
- 复杂 A/B 平台
- 高精度星盘 / 八字扩展
- 多媒体消息
