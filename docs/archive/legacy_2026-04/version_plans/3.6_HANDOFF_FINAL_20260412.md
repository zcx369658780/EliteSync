# EliteSync 3.6 最终交接稿

更新时间：2026-04-12  
版本状态：可交接 / 可归档

---

## 1. 本版定位

3.6 是 EliteSync 的古典 / 现代双路线版。

它的核心目标是：

1. 在不破坏 3.5 稳定基线的前提下，加入 `route_mode`。
2. 把路线能力拆成可切换、可解释、可比较、可回归的结构。
3. 保持 canonical truth 单源，不把路线解释层升级成新真值层。

## 3. 已落地的关键能力

### 3.1 Schema / 后端

- `route_mode` 进入 FastAPI 与 Laravel astro 链路。
- `engine_info`、`metadata` 保持增量扩展。
- `metadata.route_context` 与 `field_roles.display_only` 已同步。

### 3.2 Flutter 路线工作台

- 路线模板单独成卡。
- 路线切换会同步推荐工作台参数。
- 星盘详情页显示路线摘要与工作台摘要。

### 3.3 路线解释层

- 星盘详情页保留 compact 解释卡。
- 设置页提供完整路线差异报告。
- 多路线样例集与已知偏差文档已完成。

## 4. 稳定边界

以下内容在 3.6 中必须保持不变：

- `POST /api/v1/profile/basic`
- `GET /api/v1/profile/astro/summary`
- `GET /api/v1/profile/astro/chart`
- `GET /api/v1/profile/astro`
- `POST /api/v1/profile/astro/render`
- 资料主页 / 资料编辑页
- 星盘主页面
- 盘面设置页 / 参数工作台
- 设置中心 / 版本中心
- `chart_data` 向后兼容结构
- `astro_chart_preferences_v1` 只做本地显示
- `route_mode` 只做 display / route context
