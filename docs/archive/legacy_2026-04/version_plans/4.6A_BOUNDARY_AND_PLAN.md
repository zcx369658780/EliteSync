# 4.6A 边界与执行计划

更新时间：2026-04-21  
版本：`4.6 RTC / 通话基础设施版（语音优先）`

## 版本定位

`4.6` 不是完整 RTC 平台，也不是多人房间、直播或在线状态系统。  
本版只做 **1v1 语音优先通话最小闭环**，并为后续视频增强和外部唤醒预留边界。

## 本版必须做

1. 1v1 通话发起与响铃闭环
2. 通话状态机最小闭环
3. 权限、设备、网络、失败态最小收口
4. 通知与回流闭环
5. 与会话页、通知中心、匹配页的轻量联动
6. 仅顺手收口与通话直接相关的 4.5 轻量通知文案 observation

## 本版不做

- 多人通话
- 语聊房 / 直播间
- 在线状态全量体系
- 完整外部推送平台
- 视频美颜 / 滤镜 / 贴纸 / 屏幕共享
- 微服务重构
- 重写 4.0 / 4.2 / 4.4 / 4.5 主链

## 状态机建议

- `idle`
- `calling`
- `ringing`
- `connecting`
- `in_call`
- `rejected`
- `missed`
- `busy`
- `ended`
- `failed`

## 数据面建议

优先收敛为最小 RTC 域：
- `rtc_sessions`
- `rtc_session_events`

如 `notifications` 足以承接通话回流，则优先复用通知真值，不单独扩张通知表。

## 接口面建议

- `POST /api/v1/rtc/calls`
- `POST /api/v1/rtc/calls/{callId}/accept`
- `POST /api/v1/rtc/calls/{callId}/reject`
- `POST /api/v1/rtc/calls/{callId}/end`
- `GET /api/v1/rtc/calls/{callId}`
- `GET /api/v1/rtc/config`（如确有必要）

## 环境边界

- 本地环境：前端开发、UI 联调、截图采集、文档整理
- 阿里云端：后端开发、数据库迁移、备份、恢复、写库操作
