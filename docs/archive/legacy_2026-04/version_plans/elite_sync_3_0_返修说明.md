# EliteSync 3.0 返修说明（正式归档前修订）

当前 3.0 验收报告整体方向正确，阶段性成果可接受，但作为新版本起点，当前交付包中仍存在若干“不一致 / 不完整 / 易引发误解”的点。请以“正式归档版”标准完成返修，避免为后续 3.x 迭代留下隐患。

## 一、返修目标

本轮返修不是继续扩功能，而是完成以下目标：

1. 消除报告、截图、实际界面之间的不一致
2. 消除版本信息与更新历史之间的断层
3. 补齐外部验收视角下最容易引发质疑的证据缺口
4. 形成可正式归档的 3.0 首轮交付包

请注意：
- 本轮以“收口与校准”为主
- 不得顺手扩需求
- 不得破坏 2.9 工程底座与 3.0 已完成功能
- 所有改动必须最小化、可验证、可回滚

---

## 二、必须修正的问题（P0）

### P0-1 文案与截图不一致

当前验收报告写明：
- “愿意认识”改为“进入首聊”

但当前提交截图中的匹配结果页主按钮仍显示：
- “愿意认识”

这会导致外部验收时出现“报告描述与实际 UI 不一致”的问题。

请执行以下动作：
1. 核查真实当前代码中的最终文案到底是什么
2. 若最终实现确为“进入首聊”，则更新截图证据
3. 若最终实现仍为“愿意认识”，则修改验收报告中的描述
4. 确保以下三者完全一致：
   - 代码实际 UI
   - 验收报告文字
   - 归档截图

验收标准：
- 不再出现“报告说 A，截图显示 B”的情况

---

### P0-2 版本号与更新历史不同步

当前“关于与更新”页面中：
- 当前版本已显示 `0.02.09`

但更新历史中仍主要停留在 `0.02.08`，缺少 0.02.09 对应的明确记录。

这会造成以下隐患：
- 外部读者无法判断 0.02.09 到底改了什么
- 版本号升级缺乏 changelog 支撑
- 后续 3.x 回看时容易出现版本追踪断层

请执行以下动作：
1. 在“关于与更新”页补齐 `0.02.09` 的更新说明
2. 文案必须准确反映 3.0 首轮已完成范围，避免夸大
3. 推荐口径：
   - 3.0 首轮：完成首页关键路径优化、匹配解释升级、首聊引导与 telemetry 收口
4. 确保以下三者一致：
   - 页面版本号
   - 页面更新历史
   - 验收报告版本描述

验收标准：
- 0.02.09 的“版本号—更新内容—验收报告”三处一致

---

### P0-3 消息页证据不足

当前报告说明：
- 消息页真实账号为空态
- acceptance smoke 已覆盖 mock conversations

这在技术上成立，但从外部验收视角仍存在一个问题：
当前证据更能证明“空态正确”，但较弱地证明“首聊 / 会话恢复 / 列表可用”这些 3.0 目标。

请执行以下动作：
1. 保留真实账号空态截图
2. 额外补一张以下二选一截图作为辅助证据：
   - mock 会话列表示意图
   - 首聊引导 / 非空会话页截图
3. 在验收报告中明确区分：
   - 真实账号数据态证据
   - mock / acceptance smoke 场景证据

验收标准：
- 外部读者能清楚看出：
  - 空态已验证
  - 非空态 / 首聊态也有验证证据
  - 二者不是混在一起表述

- 这是 **3.0 首轮阶段性交付**
- 不是 **3.0 全版本结项**

- “3.0 已完成阶段 0 与 Batch-1 ~ Batch-4.2 的首轮实现，当前交付包可作为第一阶段外部验收材料。”
