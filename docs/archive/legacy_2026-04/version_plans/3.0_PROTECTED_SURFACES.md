# EliteSync 3.0 Protected Surfaces

更新时间：2026-04-08

当前基线版本：`0.02.09 / 209`

## 1. 保护面总原则

1. 继承 2.9 所有保护面，不降级。
2. 3.0 新增体验优化必须在保护面约束内进行。
3. 任何触碰保护面的改动都必须具备：
   - 最小回归验证
   - 明确回滚点
   - 可审计证据

---

## 2. 保护面清单（3.0）

## 2.1 注册 / 登录主链路
- 范围：登录态恢复、注册登录入口与状态跳转
- 敏感原因：链路一旦受损会导致全局不可用
- 允许改动：文案、引导提示、非语义型 UI 调整
- 必做检查：登录成功、登录态恢复、首次进入主界面可达

## 2.2 问卷提交流程
- 范围：问卷继续入口、提交与完成态引导
- 敏感原因：影响新用户转化与匹配可用输入
- 允许改动：CTA 与提示优化，不改提交流程语义
- 必做检查：继续问卷可达、提交成功、完成态回读一致

## 2.3 资料保存与画像重算链路
- 范围：`POST /api/v1/profile/basic`、`GET /api/v1/profile/astro/summary`、`GET /api/v1/profile/astro/chart`、`GET /api/v1/geo/places`
- 敏感原因：canonical 真源核心链路
- 允许改动：前端提示与空态优化、超时兜底展示
- 禁止改动：让本地缓存覆盖 canonical 真值
- 必做检查：保存成功、summary/chart 回读一致、弱网下可恢复

## 2.4 每周匹配主流程
- 范围：匹配入口、结果页、解释页
- 敏感原因：核心价值链路
- 允许改动：解释层结构和行动引导（展示层）
- 禁止改动：底层匹配主分语义
- 必做检查：匹配可达、解释可读

## 2.5 双向确认主流程
- 范围：匹配后的双向确认动作与状态流转
- 敏感原因：影响“查看匹配 -> 进入会话”的关键转化
- 允许改动：按钮文案、引导提示、状态说明
- 禁止改动：确认状态语义与后端判定规则
- 必做检查：双向确认可达、状态可回读、与会话可达性一致

## 2.6 消息列表与聊天主链路
- 范围：会话列表、聊天页、消息发送/读取
- 敏感原因：首聊转化与留存关键路径
- 允许改动：首聊引导文案、空态和失败恢复提示
- 禁止改动：绕过治理拦截与会话权限约束
- 必做检查：进入会话、首条消息发送、弱网恢复可见、被拉黑场景不可发送

## 2.7 举报 / 拉黑 / 账号状态反馈链路
- 范围：举报、拉黑、申诉与用户治理状态反馈
- 敏感原因：安全合规与治理闭环
- 允许改动：入口文案与状态提示优化
- 禁止改动：治理状态字段语义与处理流程绕开
- 必做检查：举报/拉黑可提交，后台可见且状态可追踪

## 2.8 后台审核与用户治理链路
- 范围：`AdminDashboardPage`、`AdminModerationPage`、`AdminVerificationPage`、`AdminUsersPage`、`AdminReportDetailPage`
- 敏感原因：运营治理和风控执行面
- 允许改动：筛选、提示、展示优化
- 禁止改动：无审计证据的治理逻辑修改
- 必做检查：关键后台页可打开，核心列表可读取，非管理员访问被拒绝

## 2.9 版本检查 / 健康检查链路
- 范围：`GET /api/v1/app/version/check`、`/api/v1/app/health`
- 敏感原因：发布可用性和可观测性基础
- 允许改动：展示层提示、健康卡 UI
- 禁止改动：版本口径不一致、健康检查被误当第二真源
- 必做检查：接口可达、版本号与发布口径一致

## 2.10 发布门禁与回滚体系
- 范围：smoke/regression/release gate/rollback 脚本与清单
- 敏感原因：生产安全底线
- 允许改动：检查项增补和证据模板增强
- 禁止改动：绕过门禁直接发布
- 必做检查：脚本可执行、回滚点可用、证据留存完整

## 3. 检查映射表（Smoke / Regression / Rollback）

| 保护面 | 最小 Smoke | 最小 Regression | Rollback 点 |
|---|---|---|---|
| 注册/登录 | 登录成功并进入首页 | 登录态恢复与重进验证 | 回退认证相关改动提交 |
| 问卷流程 | 问卷入口与继续可达 | 提交后状态一致 | 回退问卷引导改动 |
| 资料与画像 | profile/basic 保存成功 | summary/chart 回读 + 弱网兜底 | 回退资料/画像链路改动 |
| 匹配主流程 | 匹配入口与结果可达 | 解释可读 | 回退匹配展示层改动 |
| 双向确认主流程 | 双向确认动作可提交 | 确认状态回读一致 + 会话可达性一致 | 回退确认引导与状态展示改动 |
| 聊天主链路 | 会话可进入、可发送消息 | 弱网/失败恢复提示有效 + 拉黑场景发送受限有效 | 回退聊天引导改动 |
| 治理链路 | 举报/拉黑入口可达 | 后台可见并可追踪 | 回退治理入口改动 |
| 后台治理链路 | 管理后台关键页可达 | 非管理员访问受限 + 关键列表可读取 | 回退后台权限与展示改动 |
| 版本/健康检查 | health/version 接口可达 | 口径与当前版本一致 | 回退版本检查相关改动 |
| 发布门禁 | smoke 执行成功 | regression/release gate 通过 | 执行回滚脚本到前一稳定点 |

---

## 4. 保护面变更审批规则

1. 触碰保护面必须先写出“允许改动范围”和“不改范围”。
2. 变更后必须同时提供：
   - 执行证据
   - 未验证项
   - 剩余风险
3. 任一保护面回归失败，阻断进入下一批次。
