# 4.9 截图验收说明

更新时间：2026-04-29

## 验收目标

本页用于说明 4.9 UI baseline、smoke 与 release gate 的截图证据如何理解与验收。

## 已纳入证据的页面

- 首页
- 消息列表
- 聊天页
- 通知中心
- 匹配页
- 我的页
- 设置页
- 版本中心
- RTC 通话页
- RTC 音频帧现场证据

## 验收原则

1. 截图用于证明现代 UI 基线仍在，不用于冒充功能完成。
2. 通话页是否连通，以现场联测与 call state / audio frame 证据为主。
3. 若截图时机未完整捕捉到音频帧，优先信任现场联测确认。
4. 通知中心 phone 侧当前仍有 observation，首页通知卡片仅算入口级证据，不作为独立页通过。
5. 数据库正式演练已完成，截图层面不再把它当作阻断项。

## 结论

4.9 的截图证据已经足以支持：

- 现代 UI baseline 仍存在
- RTC 通话入口仍可达
- 双端 smoke 的主页面证据已齐
- 数据库正式演练已完成

若后续需要更完整的 runtime state 证据，可再补一轮独立通话截图，但这不是当前验收阻断项。
