# EliteSync 3.8 已知偏差

更新时间：2026-04-16  
版本状态：stage 4 归档版

## 1. 说明

本文件记录 3.8 中已经明确知道、但不应当被误判成 bug 的差异或偏差。

所有条目都必须按以下原则处理：

- 只属于 derived-only / display-only / advanced-context
- 不回写 canonical truth
- 不作为服务端真值变化证据

## 2. 已知偏差清单

### 2.1 路线模板与样例密度差异

- baseline 与 dense-modern 的可见点位 / 相位数值不同
- 这种差异来自路线模板与样例密度，不是 truth 层变化
- 现代路线在 dense-modern 上会保留更多扩展相位，这是预期结果

### 2.2 工作台参数仍是解释层口径

- 黄道制、宫位制、相位密度、容许度、点位范围都属于工作台口径
- 当前 stage 4 的工作台变化不会写回 canonical truth
- 后续若引擎升级接入新口径，必须单独做架构审查与回归

### 2.3 校准报告不是算法真值

- `3.8_CALIBRATION_REPORT.md` 只用于校准、偏差归档与回归锚定
- 报告里的差异是预期 display delta
- 不应把报告中的数值直接当作服务端真值变化

## 3. 归档结论

- 已知偏差已与校准样例绑定
- 这些差异不会阻断 3.8 进入 stage 5
- 后续回归时若看到同类数值差异，应先对照本文件，再决定是否升级为 bug
