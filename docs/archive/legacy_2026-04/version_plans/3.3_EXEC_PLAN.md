## 5. 阶段说明

### 阶段 0：范围冻结与风险评审

目标：
- 先确认 3.3 是“显示链路深化 + 设置中心完善版”，不是算法升级版

输出：
- `3.3_EXEC_PLAN.md`
- `3.3_RISK_REVIEW.md`

### 阶段 1：资料与画像真值链路收口

目标：
- 让“出生资料修改 -> 服务端重算 -> 页面刷新 -> 本地盘面重绘”可见、可测、可解释

关键点：
- 资料编辑页说明统一
- 资料更新失败 / 重算失败反馈明确
- 资料页 / 画像页 / 盘面页口径一致

### 阶段 2：星盘显示深化

目标：
- 在不动 canonical 算法的前提下，把本地盘面显示做稳、做清楚、做可恢复

关键点：
- 盘面元素开关稳定生效
- 恢复默认与预设档位可靠
- 盘面说明区与图层说明区不互相干扰

### 阶段 3：设置中心与关于页正式化

目标：
- 把设置、账号安全、隐私、通知、关于与更新整理成正式产品结构

关键点：
- 首页结构清楚
- 入口层级清楚
- 版本信息与更新说明统一

### 阶段 4：版本中心与口径统一

目标：
- 收口版本、下载、更新历史、服务状态、统计口径

关键点：
- 同一版本号在多个页面不冲突
- Admin 统计卡片不再误导

### 阶段 5：回归、验收与归档

目标：
- 形成正式交接包，而不是零散改动

必交文件：
- `3.3_ACCEPTANCE_REPORT.md`
- `3.3_HANDOFF_FINAL_YYYYMMDD.md`
