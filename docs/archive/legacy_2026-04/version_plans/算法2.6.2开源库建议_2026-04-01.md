# 可复用开源件清单（按八字 / 星盘 / 紫微 / Flutter 分组）

> 结论先说：
> **没有一套成熟到可直接商用落地、同时覆盖“八字 + 星盘 + 紫微 + Flutter UI”的全家桶。**
> 现在最成熟的是 **紫微斗数显示链路**，其次是 **西占星盘绘制链路**；**八字更多是计算库，不是成熟显示组件**。([GitHub][1])

---

## 一、八字（Bazi）

### 1. `6tail/lunar-php`

* **类型**：计算库
* **适用**：PHP / Laravel 后端
* **许可**：MIT
* **能力**：支持干支、生肖、节气、八字、五行、十神、星座等，适合做你们现有服务端 canonical 结果源。
* **建议定位**：继续作为**八字计算与字段输出主库**，前端自行做“四柱主面板 + 五行图 + 标签摘要”。
* **不建议**：把它当成前端可直接复用的显示组件。([GitHub][2])

### 2. `cantian-ai/bazi-mcp`

* **类型**：服务型/工具型项目
* **适用**：八字信息生成、对接式能力参考
* **许可**：ISC
* **能力**：定位是 Bazi MCP Server，强调“八字信息与洞察”的服务能力。
* **建议定位**：适合做**接口设计参考**或对照输出，不适合直接当你们 APP 的显示组件。([GitHub][3])

### 3. `Wolke/bazi-mingli`

* **类型**：技能/知识型项目
* **适用**：概念、术语、解释逻辑参考
* **许可**：需你们本地再核具体 LICENSE 文本
* **建议定位**：可参考其术语体系与命理解释结构。
* **不建议**：直接视为成熟 UI 组件或直接嵌入产品。([GitHub][4])

### 八字分组结论

* **能复用的核心**：计算层
* **不能指望现成拿来用的**：完整显示层
* **你们当前最稳方案**：`lunar-php` 出 canonical，Flutter 自己做四柱表、五行雷达/柱图、关系摘要。([GitHub][2])

## 二、星盘（西占 / Natal Chart）

### 1. `AstroDraw/AstroChart`

* **类型**：显示库
* **适用**：Web / SVG 绘制
* **许可**：MIT
* **能力**：专门用于生成 SVG astrology chart，仓库明确写了“只负责显示，不计算行星位置”。
* **建议定位**：非常适合做**星盘 UI 参考**、后台原型、Web 端盘面输出。
* **不建议**：直接拿来替代你们 Flutter 客户端最终 UI。([GitHub][5])

### 2. `hoishing/natal`

* **类型**：显示 + 计算整合型库
* **适用**：Python / SVG 输出
* **能力**：支持 birth chart、transit chart、synastry chart、solar return chart，并且高度可配置。
* **注意**：它依赖 Swiss Ephemeris 这条线，许可合规仍需你们谨慎处理。
* **建议定位**：适合做**后端生成 SVG 星盘**、研究验证或内部工具。([GitHub][6])

### 星盘分组结论

* **能复用的核心**：轮盘绘制思路、SVG 输出能力
* **最适合你们的用途**：把它们当作**盘面结构参考**，Flutter 端用 `CustomPaint` 重绘。
* **不建议**：在当前闭源商用前提下，轻率把 Swiss 依赖链直接做成生产客户端方案。([GitHub][5])

---

## 三、紫微斗数（Zi Wei Dou Shu）

### 1. `SylarLong/iztro`

* **类型**：核心数据/排盘库
* **适用**：JavaScript / TypeScript
* **许可**：MIT
* **能力**：支持生日、出生时间、性别输入，输出 12 宫、生肖、星座、四柱、运限、流耀等结构化信息。
* **建议定位**：适合做**紫微服务端/中间层 canonical 逻辑参考**。([GitHub][7])

### 2. `SylarLong/react-iztro`

* **类型**：显示组件
* **适用**：React / Web
* **许可**：MIT
* **能力**：这是目前我看到最成熟的开源紫微显示组件之一，明确支持完整紫微斗数星盘、主星/辅星/杂耀/四化/流耀、重要星耀高亮、运限指示等。
* **建议定位**：这是你们紫微 UI 最值得参考的**第一优先级开源件**。
* **不建议**：直接拿来做 Flutter 成品 UI，但非常适合当视觉与布局蓝本。([GitHub][1])

### 3. `EdwinXiang/dart_iztro`

* **类型**：Flutter/Dart 侧计算库
* **适用**：Flutter
* **许可**：仓库需你们本地再核 LICENSE，但 README 明确它源自 `iztro`
* **能力**：支持紫微斗数和八字计算、真太阳时、地理位置、多平台。
* **建议定位**：适合做**Flutter 侧辅助能力或原型验证**。
* **不建议**：直接让客户端成为 canonical 真源。([GitHub][8])

### 紫微分组结论

* **最成熟的可复用显示链路**：`iztro` + `react-iztro`
* **Flutter 最值得看的玄学专用库**：`dart_iztro`
* **你们当前最优策略**：
  服务端继续输出 canonical 紫微结果，Flutter 参考 `react-iztro` 做 12 宫盘 UI，自身不要承担真值计算。([GitHub][1])

## 四、Flutter 通用基础件

### 1. `fl_chart`

* **类型**：通用图表库
* **适用**：Flutter
* **许可**：MIT
* **能力**：支持 line / bar / pie / scatter / radar / candlestick。
* **建议定位**：

  * 五行雷达图
  * 五行柱状图
  * 资料完整度环图
  * 模块分数图
* **不建议**：用它硬画星盘轮盘或紫微宫盘主视觉。([GitHub][9])

### 2. `CustomPaint + CustomPainter`

* **类型**：Flutter 原生自定义绘制机制
* **适用**：复杂盘面
* **建议定位**：

  * 星盘轮盘
  * 紫微 12 宫盘
  * 八字四柱结构面板中的特殊连线或高亮
* **原因**：目前并没有成熟到足够产品级的 Flutter 玄学显示组件，复杂主视觉自己画更稳。
* **说明**：这条是工程建议，不是外部库条目。([GitHub][8])

## 五、可直接采用的推荐组合

### 方案 A：你们当前最稳的组合

* **八字**：`lunar-php`
* **星盘**：服务端现有 `western_lite` + 参考 `AstroChart` 轮盘结构
* **紫微**：服务端紫微 canonical + 参考 `react-iztro`
* **Flutter 图形**：`fl_chart` + `CustomPaint`
  这套组合最符合你们现在的 Laravel + Flutter 架构。([GitHub][2])

### 方案 B：如果以后要补 Web 管理端 / H5 原型

* **星盘显示**：`AstroChart`
* **紫微显示**：`react-iztro`
* **八字仍走服务端 canonical 输出**
  这套方案在 Web 端能最快出效果。([GitHub][5])

## 六、我的最终建议

### 优先级 1：立即可借鉴

* `react-iztro`
* `iztro`
* `fl_chart` ([GitHub][1])

### 优先级 2：适合研究 / 后端原型

* `AstroChart`
* `hoishing/natal`
* `dart_iztro` ([GitHub][5])

### 优先级 3：适合当计算参考，不适合直接当显示组件

* `lunar-php`
* `bazi-mcp`
* `bazi-mingli` ([GitHub][2])

---

## 七、一句话结论

**紫微有现成可借的显示组件，星盘有现成可借的绘制库，八字主要靠计算库 + 自研显示；Flutter 端最现实的路线仍然是“通用图表库 + 自定义盘面绘制”。** ([GitHub][1])
