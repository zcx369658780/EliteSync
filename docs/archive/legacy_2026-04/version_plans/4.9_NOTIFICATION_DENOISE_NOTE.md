# 4.9 通知中心工程 slug 降噪说明

更新时间：2026-04-28

## 背景

4.8 / 4.7 复验中，通知中心仍会在用户面显示部分工程 slug，例如：

- `rtc_call_invite`
- `rtc_call_accepted`
- `rtc_call_ended`
- `rtc_call_missed`

这不影响通知主链回流，但会降低用户面产品感，也不符合 4.9 的测试前治理目标。

## 处理原则

- 工程 type 继续保留在 debug / log / telemetry；
- 用户界面只展示自然文案；
- 通知回流、已读、阅读状态、路由跳转都不改；
- 旧通知数据仍需正常打开，不得因降噪而丢失行为。

## 当前实现

通知中心展示层已增加 kind → 用户面标签映射：

- `message` → `新消息`
- `status_like` → `动态被喜欢`
- `status_comment` → `动态有新回复`
- `match_like` → `新提示`
- `match_success` → `新匹配`
- `rtc_call_invite` → `语音来电`
- `rtc_call_accepted` → `通话已接听`
- `rtc_call_rejected` → `通话已拒绝`
- `rtc_call_missed` → `未接来电`
- `rtc_call_ended` → `通话已结束`
- 其他未知值 → `新通知`

对应实现文件：

- [`apps/flutter_elitesync_module/lib/features/notification/presentation/pages/notification_center_page.dart`](D:/EliteSync/apps/flutter_elitesync_module/lib/features/notification/presentation/pages/notification_center_page.dart)

## 不做什么

- 不重写 `GET /api/v1/notifications`
- 不改通知表结构
- 不重写 read / read-all
- 不把通知中心扩成营销推送系统
- 不让工程 slug 直接显示给普通用户

## 验收口径

通知中心主界面应只显示自然文案，工程 slug 仅保留在 telemetry / debug / 后端原始字段中。
