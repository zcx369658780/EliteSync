# EliteSync 3.3 风险评审

更新时间：2026-04-11

## 高风险点

- 资料修改后重算链路
- canonical 真值与本地偏好的边界
- 星盘本地绘制与盘面设置联动
- 设置中心入口重构
- 关于与更新页版本口径
- Admin 统计显示口径
- 版本检查与下载口径
- 任何误把显示需求升级成算法需求的改动

## 高风险模块

### 1. 出生资料编辑与重算
- 风险点：
  - 修改后未触发服务端重算
  - 重算失败未给出明确反馈
  - 前端缓存覆盖 canonical 真值
- 风险等级：高

### 2. 星盘显示层
- 风险点：
  - 本地 SVG 开关与真实显示不一致
  - 恢复默认失效
  - 预设档位与单项开关相互覆盖
- 风险等级：高

### 3. 设置中心与关于页
- 风险点：
  - 入口重构导致旧功能不可达
  - 版本 / 更新 / 服务状态口径不一致
  - 文案把显示偏好写成真值
- 风险等级：中高

### 4. Admin 统计与治理口径
- 风险点：
  - 测试账号 / 合成用户 / 普通用户口径混淆
  - 统计卡片重叠展示被误读
  - 页面解释文案不足
- 风险等级：中高

### 5. 版本检查与下载
- 风险点：
  - 版本号 / changelog / about / 下载地址不同步
  - 发布脚本与远端环境基线不一致
  - alpha 后缀排序回归
- 风险等级：高

## 兼容性提示

- canonical 真值必须继续由服务端决定
- 前端本地偏好只影响显示
- 星盘设置页不能演变成算法调参面板
- 资料编辑页必须兼容旧数据
- 版本中心必须兼容 0.03.02a / 0.03.02 历史口径
- Admin 统计解释必须继续保留“可重叠”说明

## 结论

3.3 属于典型跨资料、显示、设置、版本口径的版本，必须先冻结边界，再推进实施，不能按普通 UI 微调对待。
