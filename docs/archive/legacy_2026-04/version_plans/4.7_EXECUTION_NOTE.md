# 4.7 执行记录

更新时间：2026-04-26

## 执行目标

`4.7` 作为 `测试前稳定化与质量门禁版`，目标不是扩新功能，而是把现有现代 UI、回滚政策、baseline evidence 与 smoke / regression 门禁固定下来，避免再次发生 UI 回退和跨层恢复事故。

## 本轮已完成事项

### 1. UI protected surfaces 落地

- 新增 `docs/PROTECTED_UI_SURFACES.md`
- 明确现代 UI 作为保护面
- 涵盖主导航、首页、消息、聊天、通知、匹配、我的、设置、版本中心、状态流以及星空视觉基线

### 2. 回滚与恢复政策落地

- 新增 `docs/runbooks/ROLLBACK_AND_RECOVERY_POLICY.md`
- 明确禁止 repo 级回滚
- 仅允许路径级、文件级、最小范围恢复
- 要求恢复前做安全快照

### 3. UI baseline evidence 补齐

- 补齐手机与模拟器的现代 UI 证据
- 更新 `docs/version_plans/4.7_UI_BASELINE_EVIDENCE_INDEX.md`
- 证据覆盖：
  - 首页
  - 消息列表
  - 聊天页
  - 通知中心
  - 匹配页
  - 状态流 / 发布状态页
  - 我的 / 设置 / 退出登录入口
  - 版本中心

### 4. UI reversion 事故修复说明落地

- 新增 `docs/version_plans/4.7_UI_REVERSION_FIX_NOTE.md`
- 明确事故原因是跨层恢复时未保护现代 UI 表面
- 明确未来恢复必须分层、路径级、最小范围执行

### 5. 回归与门禁补齐

- 更新 `docs/version_plans/4.7_REGRESSION_CHECKLIST.md`
- 新增 UI Baseline Regression 条目
- 新增 release gate / baseline guard 说明

## 当前状态判断

`4.7` 的工程目标已经从“继续修功能”转为“稳定化与门禁收口”。

当前可认定的事实是：

- 现代 UI 基线已补齐主要页面证据
- protected surfaces 与 rollback policy 已写入项目文档
- 4.7 的 smoke / regression 口径已明确
- 4.7 的验收材料链可以进入顾问复核

## 仍保留的观察项

- 独立的 live-call runtime state 页面仍可作为补充证据，不影响当前门禁收口
- 若后续跨层恢复再次发生，必须先走 blocker report，再做最小恢复
