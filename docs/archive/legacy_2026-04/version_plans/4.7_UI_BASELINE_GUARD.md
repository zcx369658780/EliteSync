# 4.7 UI Baseline Guard

## Goal

Prevent modern UI regression while stabilizing build, backend, RTC, and media blockers.

## Source of truth

- `docs/PROTECTED_UI_SURFACES.md`
- `docs/runbooks/ROLLBACK_AND_RECOVERY_POLICY.md`
- `docs/4.7_UI_REVERSION_REPORT.md`

The current modern UI is a protected baseline. Recovery work may not silently revert:

- main navigation / bottom tabs
- home page layout
- messages / chat entry placement
- notifications center layout
- match / profile / questionnaire / version center entry placement
- starry background and modern card spacing

## Mandatory guardrails

1. Use path-level recovery only.
2. Never use repo-level rollback to fix a cross-layer blocker.
3. Before any recovery, create a safety checkpoint.
4. After any recovery, diff-check all protected UI surfaces.
5. If a change touches a protected surface, treat it as a UI task and stop for explicit approval.
6. Do not restore any file under `docs/PROTECTED_UI_SURFACES.md` without explicit user approval.

## UI regression requirement

Any 4.7 smoke / regression run must include UI baseline regression checks for:

- tab layout
- home page
- messages page
- chat entry
- notifications center
- settings / logout discoverability
- starry background / modern card / spacing
- no debug / demo / scaffold / internal / mock text regression
