# EliteSync 3.8 差距矩阵

更新时间：2026-04-16  
版本状态：stage 0 拆解

## 1. 目的

本矩阵用于把 `3.7` 的现有能力与 `3.8` 目标能力对齐，明确当前差距、需要补的证据与优先级。

## 2. 差距表

| 目标项 | 3.7 当前状态 | 3.8 需要达到 | 当前差距 | 需要的证据 | 优先级 |
|---|---|---|---|---|---|
| 参数工作台深化 | 只有 MVP 级参数工作台 | 专业工作台首版 | 宫位制、黄道制、相位密度、点位集不够完整 | 参数面板截图、DTO、测试 | P0 |
| 参数解读弹窗 | 仅有基础说明卡 | 参数 / 星体 / 相位 / 主星 / 飞宫可点开阅读 | 缺统一解读块与弹窗机制 | 解读弹窗截图、builder 测试 | P0 |
| 主星 / 宫主星 / 飞宫解释 | 只有碎片化摘要 | 结构化摘要首版 | 字段来源与展示样式未统一 | 解释摘要、字段映射、测试 | P0 |
| 校准样例 | 3.7 主要是示例与演示页 | 正式样例集 + 校准报告 | 未把参考样例纳入工程闭环 | CALIBRATION_REPORT、对照截图 | P0 |
| 视觉 / 交互回归 | 有基础星盘阅读性优化 | 体验短板正式收口 | 字体、标签避让、日期输入仍待系统化 | UI 回归清单、截图 | P0 |
| 桌面 / 宽屏预演 | 未正式纳入 | 至少形成预演结构 | 无桌面布局验证 | 大屏截图或草图 | P1 |
| 小行星 / 虚点解释 | 未正式纳入 | 第一批骨架 | 缺字段与模板 | 解释骨架、测试 | P1 |

## 3. 已有能力与可复用基线

- `chart_data`、`engine_info`、`metadata` 仍可作为主基线
- `route_mode`、`pair_context`、`time_context`、`return_context`、`comparison_context` 的 advanced-context 设计可复用
- `3.7` 的高级预览页、演示页、样例矩阵、已知偏差可作为显示层 / 归档层参考
- 资料真值链路、设置中心、版本中心和发布门禁可直接复用

## 4. 必须补齐的工程证据

1. 参数工作台的布局与字段截图
2. 单项解读弹窗截图
3. 主星 / 飞宫解释摘要截图
4. 正式校准样例对照截图
5. `CALIBRATION_REPORT.md`
6. UI / 交互回归清单
7. Claude / Gemini 审查留痕

## 5. 归档判定

3.8 只有在以下内容齐备后，才可从“阶段推进”升级为“正式归档”：

- P0 能力完成
- 差距矩阵闭环
- 风险评审闭环
- Claude / Gemini 留痕齐全
- 校准报告与回归清单齐全
