## 5. 阶段说明

### 阶段 0：范围冻结与风险评审

目标：
- 先确认 3.4 是“体验精修 + 正式化收口 + Beta 运营准备版”，不是算法升级版

输出：
- `3.4_EXEC_PLAN.md`
- `3.4_RISK_REVIEW.md`
- `3.4_PROTECTED_SURFACES.md`

### 阶段 1：页面口径统一与资料编辑链路收口

目标：
- 让资料编辑、保存反馈、重算提示、版本中心口径更一致、更易理解

关键点：
- 保存后反馈更明确
- 重算失败 / 慢网提示更清楚
- 当前版本、更新历史、版本检查口径统一

### 阶段 2：星盘阅读性精修

目标：
- 在不改 canonical 算法的前提下，把本地盘面显示做得更易读、更可解释

关键点：
- 标签拥挤与层次关系更合理
- 预设档位文案更直观
- 图层说明与显示偏好边界更清楚

### 阶段 3：设置中心与 Beta 运营准备

目标：
- 继续把设置中心做成熟，并为真实 Beta 运维做准备

关键点：
- 设置页分组与说明收口
- health / version / check 复核
- 弱网首进验证
- 最小回滚检查

### 阶段 4：回归、验收与归档

目标：
- 形成正式交接包，避免零散交付

必交文件：

状态：

- 关键页面截图已补齐
