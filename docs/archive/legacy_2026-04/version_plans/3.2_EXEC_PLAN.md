# EliteSync 3.2 执行计划

更新时间：2026-04-10
适用对象：Codex / Gemini-MCP / Claude-mcp / 项目负责人
版本状态：草案 v1（冻结件）

## 当前执行状态

- 阶段 0 已完成：冻结边界、风险拆解、保护面确认、回归清单成型
- 阶段 1 已启动：测试账号体系底座、批次审计字段、管理员用户列表客观侧面
- 当前已落地的最小证据：`app:dev:sync-account-tier`、`app:dev:synthetic-users`、Admin 用户列表批次筛选、账号类型筛选、客观元信息展示
- 玄学首批可见修正：八字页底部技术参数去掉重复“精度”误导行，当前保留“位置来源 / 引擎 / 置信”三段，避免同值重复造成验收歧义
- 玄学术语进一步收口：星盘 / 紫微 / 八字 / 盘面设置页统一将 `precision` 类字段对外展示为“版本”或“可信度”，避免把版本、模式、置信度混叫成“精度”导致验收歧义
- 广场轻分层已启动：首页状态广场预览与广场页均开始展示作者层级与可见层级标签，区分公开层 / 广场层 / 私密层，后续继续补可见性治理与审核视图
- 匹配与首聊转化已开始补强：匹配结果页的行动建议已改成可点击的开场草稿卡，点击后会预填聊天输入框并跳转会话页；聊天页顶部破冰卡也已改成可点建议，便于开场与沉默后恢复接话
- 当前已生成阶段性验收草稿 `3.2_ACCEPTANCE_REPORT.md`，但 Flutter 本机工具链仍不稳定，正式归档前需要在可用环境补跑 acceptance smoke / 截图验收

## 1. 版本定位

3.2 = 匹配与互动转化优化 + 玄学可信度修复 + 测试运营增强 + 广场轻分层版

## 2. 执行顺序

1. 测试账号体系底座
2. 账号可观测与管理
3. 玄学 correctness 修复
4. 匹配与首聊转化
5. 广场轻分层
6. 回归与验收归档

## 3. P0 范围

- 建立正式测试账号生成与管理体系
- 默认支持 1000 个 synthetic 测试账号池
- 支持按批次扩展到 10000 个 synthetic 测试账号
- 保留 20-50 个高质量人工 test 账号
- 测试账号资料完整，便于匹配、广场、聊天与玄学联调
- 修复玄学排盘 correctness 与 readability 的 P0 问题
- 强化匹配后的首聊、恢复、持续互动转化
- 广场做轻量分层与审核/可见性增强
- 形成完整阶段验收与最终归档包

## 4. P1 范围

- 关系进度条 / 熟悉度 / 轻量互动任务
- 更细的广场频道标签与排序策略
- 更细的管理查询与批次统计面板
- 更丰富的 synthetic 行为脚本模板

## 5. 不纳入主线

- 全量专业占星模式一次性完整扩张
- 校园垂直化战略大转向
- 重直播 / 重打赏 / 重语音房模式
- AI 虚拟恋人主产品化

## 6. 阶段 0 冻结边界

- canonical 数据只能由服务端决定
- 前端缓存不能成为第二真源
- synthetic/test/normal 必须严格分层
- synthetic 默认 `exclude_from_metrics = true`
- 3.1 已通过能力不得被破坏
- 任何迁移 / 配置 / 批次变更都要配套回滚点
