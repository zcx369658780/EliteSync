# 4.6F LiveKit 真语音联测 Runbook

更新时间：2026-04-23

## 目的

验证 4.6F 的真语音链路是否可在两端设备上完成：

1. 发起通话
2. 对端收到来电
3. 接听后加入同一 LiveKit 房间
4. 双方可听到语音
5. 挂断后双方安全退出

## 前置条件

- 阿里云后端已部署 4.6F 代码
- LiveKit 服务已可访问
- 后端环境变量已配置：
  - `LIVEKIT_ENABLED=true`
  - `LIVEKIT_URL`
  - `LIVEKIT_API_KEY`
  - `LIVEKIT_API_SECRET`
  - `LIVEKIT_TOKEN_TTL_SECONDS`
- Flutter 宿主包已启用 `useLiveKitRtc=true`
- 两台设备分别登录有效烟测账号

默认烟测账号：

- `17094346566` / `SmokeUser`
- `13772423130` / `test1`

## 预期代码链路

- `POST /api/v1/rtc/calls` 创建通话会话
- `GET /api/v1/rtc/calls/{callId}/livekit` 返回会话参与者专用 join 信息
- Flutter 通话页在 `connecting / in_call` 时尝试连接 LiveKit 房间
- 通话终态时执行 `leave()`

## 联测步骤

### 1. 发起端

1. 打开发起端 app
2. 进入消息页并打开目标会话
3. 点击语音通话
4. 确认通话状态进入 `calling`

### 2. 接听端

1. 打开接听端 app
2. 等待来电页自动拉起
3. 点击接听
4. 确认状态进入 `connecting` / `in_call`

### 3. 音频验证

1. 发起端说话
2. 接听端确认能听到声音
3. 接听端说话
4. 发起端确认能听到声音

### 4. 挂断验证

1. 任一端点击挂断
2. 另一端应同步进入通话结束页
3. 房间应自动离开

## 成功标准

- 双方能进入同一通话房间
- 双方麦克风可正常工作
- 双方能够互相听到声音
- 终态退出后不再占线

## 失败排查

- 若接听页不弹出，优先检查：
  - 通知中心 / RTC 会话直扫链路
  - 后端 `rtc_call_invite` 是否写入
- 若能进入页面但没有声音，优先检查：
  - `LIVEKIT_*` 环境变量
  - 麦克风权限
  - Room token 的 room / identity 是否一致
- 若持续 busy，优先检查：
  - 旧会话是否已超时回收
  - 两端是否都执行了 leave
