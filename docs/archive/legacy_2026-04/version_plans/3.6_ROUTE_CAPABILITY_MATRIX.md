## 1. 基线说明

3.5 之后，当前工作台的默认口径可以视为：

- `zodiacMode`: `tropical`
- `houseSystem`: `whole`
- `aspectMode`: `standard`
- `orbPreset`: `standard`
- `pointMode`: `full`

3.6 的路线目标是在这个基础上把：

- `standard`
- `classical`
- `modern`

三条路线做成明确、可比较、可回退的能力集。

## 4. 当前字段角色

| 字段 / 模块 | 角色 | 3.6 要求 |
|---|---|---|
| `chart_data` | canonical 读取输入 / 本地绘制核心 | 继续兼容，不可断旧字段 |
| `engine_info` | 引擎来源 / 版本 / 路线元信息 | 仅增量扩展 |
| `metadata` | schema / field roles / route 标记 | 仅增量扩展 |
| `astro_chart_preferences_v1` | 本地显示偏好 | 仍然 display-only |
| `route_mode` | 路线上下文 | 不能升级成真值字段 |

---

## 5. 差异归档要求

3.6 的路线能力矩阵必须最终落成可归档材料：

- `ROUTE_CAPABILITY_MATRIX`
- `ROUTE_PARITY_REPORT`
- `KNOWN_DEVIATIONS`
- `MULTI-ROUTE_SAMPLE_SET`

这些材料的用途是：

1. 区分“路线能力”与“canonical 真值”
2. 区分“展示差异”与“计算差异”
3. 为 3.7 预留路线升级的稳定基线
