## 2. 回归清单

### 2.1 资料与画像链路

- `POST /api/v1/profile/basic` 保存后是否仍会返回重算后的快照
- 编辑出生时间 / 出生地后，是否仍能触发重算并刷新
- `astro/summary` 与 `astro/chart` 是否仍能读到最新真值

### 2.2 星盘展示链路

- 星盘页是否仍能本地绘制
- `chart_data` 缓存 / 兜底是否仍可用
- 盘面设置是否仍只影响本地展示
- 设置开关是否不会改写 canonical 真值

### 2.3 引擎与 schema

- 新增 `engine_info` 是否只是附加字段
- 旧缓存没有 `engine_info` 时是否仍可显示
- 旧客户端是否可以忽略新增元信息
- `chart_data` 旧 key 是否仍然可读

### 2.4 旧入口保护

- 资料主页是否仍可进入
- 资料编辑页是否仍可进入
- 星盘主页面是否仍可进入
- 设置中心是否仍可进入
- 版本中心是否仍可进入

---

## 3. 手工验收步骤

1. 打开一个固定样例账号。
2. 进入资料编辑页，修改出生时间或出生地。
3. 保存后确认页面重算并刷新。
4. 打开星盘页，确认盘面仍正常绘制。
5. 切换参数面板，确认仅影响本地显示。
6. 关闭应用并重新打开，确认缓存与回读正常。
7. 切换到设置中心和版本中心，确认旧入口可达。
8. 若 UI 已补参数面板，切换预设后确认盘面变化可见。

---

## 4. 边界条件

必须覆盖：

- 缺少出生时间
- 缺少出生地
- 经度 / 纬度缺失
- 时区默认值回退
- 旧缓存只有 `chart_data` 没有 `engine_info`
- 新旧响应字段共存
- 重复切换参数预设
- 弱网下首进

---

## 5. 失败后的补测建议

如果任一项失败，先补测：

1. 后端 astro 测试
2. Flutter DTO / builder 测试
3. 星盘页渲染测试
4. 参数面板 provider / preset 测试
5. acceptance smoke

如果失败点与 schema 漂移有关，优先确认：

- 是否删了旧字段
- 是否把真值字段和显示字段混在一起
- 是否破坏了缓存兜底
