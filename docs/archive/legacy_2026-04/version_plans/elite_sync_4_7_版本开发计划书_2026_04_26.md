# EliteSync 4.7 版本开发计划书

发布日期：2026-04-26

## 1. 版本定位

4.7 是 `测试前稳定化与质量门禁版`。

本版本不扩新业务功能，目标是把 4.0 - 4.6P 已完成的主链固化为：

- 可测试
- 可复验
- 可恢复
- 可保护 UI 基线
- 可稳定交付

## 2. 当前版本目标

1. 防止 UI 回退再次发生。
2. 固定现代 UI 为 protected surface。
3. 把跨层恢复流程改造成分层、路径级、最小范围恢复。
4. 让 smoke / regression / release gate 的口径固定化。
5. 为后续版本建立可复用的 blocker report / rollback policy / baseline evidence 流程。

## 3. 本版本不做什么

- 不做新业务功能扩张
- 不做新 RTC 架构
- 不做多人通话 / 直播 / 视频平台化
- 不做 repo 级回滚恢复
- 不做未经批准的 UI 复位

## 4. 本版本必须先完成的门禁文档

- `docs/PROTECTED_UI_SURFACES.md`
- `docs/runbooks/ROLLBACK_AND_RECOVERY_POLICY.md`
- `docs/version_plans/4.7_UI_BASELINE_GUARD.md`
- `docs/version_plans/4.7_UI_REVERSION_FIX_NOTE.md`
- `docs/version_plans/4.7_UI_BASELINE_EVIDENCE_INDEX.md`
- `docs/version_plans/4.7_REGRESSION_CHECKLIST.md`

## 5. 核心执行顺序

1. 冻结 UI protected surfaces
2. 确认回滚 / 恢复政策
3. 补齐 UI baseline evidence index
4. 运行 4.7 smoke
5. 运行 UI Baseline Regression
6. 通过 release gate 后再考虑后续版本

## 6. 版本门禁原则

- 任何恢复操作先 checkpoint
- 任何保护面恢复都必须先确认路径
- 任何跨层 blocker 先写 blocker report
- 任何 UI 变化都必须先确认是否属于 protected surface

## 7. 当前判断

4.7 当前的工作重心不是功能实现，而是：

- 保护现代 UI
- 固化恢复边界
- 防止再发生 repo 级回滚
- 补齐证据索引与回归清单
