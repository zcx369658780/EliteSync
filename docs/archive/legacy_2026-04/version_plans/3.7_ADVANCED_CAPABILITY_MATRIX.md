# EliteSync 3.7 高级能力矩阵

更新时间：2026-04-12  
适用阶段：3.7 stage 3

## 1. 高级能力清单

### 1.1 关系维度

- 合盘（Synastry）
- 对比盘（Bi-wheel / Comparison）

### 1.2 时间维度

- 行运（Transit）
- 返照（月返优先，必要时降级日返）

### 1.3 高级解读层

- 关系类型摘要
- 关键相位摘要
- 时间窗口摘要
- 主题摘要
- 可见边界提示

## 2. 字段角色定义

| 字段 / 模块 | 角色 | 3.7 约束 |
|---|---|---|---|
| `chart_data` | canonical 读取输入 / 现有本命盘核心 | 继续兼容，不可破坏 |
| `engine_info` | 引擎来源 / 版本 / 路线 / 高级能力元信息 | 只增量扩展 |
| `metadata` | schema / field roles / context 标记 | 只增量扩展 |
| `pair_context` | 双人上下文 | advanced-context |
| `time_context` | 时间上下文 | advanced-context |
| `advanced_summary` | 高级摘要字段 | derived-only / display-only |
| `route_mode` | 路线展示上下文 | display-only |

## 3. 能力交付标准

### 3.1 合盘 / 对比盘

- 能切换双人样例
- 能展示双人基础信息
- 能展示关键相位和最小摘要
- 能清楚说明结果是 derived-only

### 3.2 行运

- 能展示当前时间窗口
- 能展示与本命盘的关键相位差异
- 能给出最小时间摘要卡
- 能清楚说明结果是 advanced derived

### 3.3 返照

- 只做一项最小可用能力
- 优先月返，必要时降级日返
- 与本命盘 truth 明确分层

## 4. 回归样例方向

必须建立三类样例：

1. 单人时间维度样例
2. 双人合盘样例
3. 双人对比盘样例

每类样例至少覆盖：

- 输出计数稳定性
- 关键摘要稳定性
- 已知偏差归档

## 5. 不允许的事情

- 不允许把高级结果回写为 canonical truth
- 不允许删除旧字段
- 不允许把高级页面做成完全独立的专业后台而失去可读性

## 6. stage 2 状态

- `pair` / `transit` / `return` 的后端 scaffold 已落地。
- Flutter 侧已接入高级预览卡、scaffold 请求与高级详情页。
- 当前返回结构仍保持 derived-only / display-only / advanced-context 口径。
- 接下来 UI 层只允许消费这些上下文，不允许重写 truth 层。
