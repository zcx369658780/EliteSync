# EliteSync 2.8 执行计划（信任安全与运营后台版）

更新时间：2026-04-06  
适用版本：`2.8`  
当前基线版本：`0.02.06`（2.7 已结项）

## 版本定位
2.8 的目标不是继续增强前台体验，而是补齐真实用户接入前必须具备的治理能力：举报、拉黑、封禁、申诉、认证人工审核、运营后台与核心指标。

## P0 范围
- 举报 / 拉黑 / 封禁 / 申诉闭环
- 认证状态与人工审核占位
- 运营后台 MVP：用户 / 举报 / 审核 / 看板
- 最小事件埋点
- 更新 smoke / regression / rollback 文档

## P1 范围
- 头像 / 昵称 / 简介最小审核状态
- 反馈数据后台可见
- 操作日志与备注增强
- 用户搜索与筛选增强

## 不做事项
- 不大改前台匹配主流程
- 不重构聊天底层
- 不重做问卷体系
- 不大改数据库核心 schema
- 不推进大规模商业化付费
- 不把 2.8 做成全量后台系统

## 推荐执行顺序
1. 定义举报 / 审核 / 封禁 / 申诉状态机
2. 定义后台 IA 与最小页面结构
3. 接前台举报 / 拉黑入口
4. 接后台举报与审核处理
5. 接最小事件埋点与看板
6. 更新 release / regression / rollback 文档

## 成功标准
- 用户能举报和拉黑
- 后台能查看并处理举报
- 认证状态可展示且人工审核可执行
- 后台能看到最小用户 / 举报 / 审核 / 看板信息
- 最小事件埋点可用
- 2.6.4 门禁与 2.7 主链路没有倒退
